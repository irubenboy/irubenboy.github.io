    1 Funciones¶

    Introducción¶

    Una subrutina o subprograma, como idea general, se presenta como un subalgoritmo que forma parte del algoritmo principal, el cual permite resolver una tarea específica.

    Podemos distinguir tres términos que poseen diferencias:

    • Función: conjunto de instrucciones que devuelven un resultado.
    • Procedimiento: conjunto de instrucciones que se ejecutan sin devolver ningún resultado.
    • Método: función o procedimiento que pertenece a un objeto.

    Dichas funciones se pueden utilizar desde muchos sitios diferentes, por lo que de manera general, no se suelen poner mensajes en consola en las funciones ya que puede ser que dichos mensajes no interesen en todos los sitios donde se utilice dicha función. A no ser que la función se haya creado específicamente para dar mensajes informativos en consola.

    Declaración¶

    Existen diferentes maneras de crear una función, pero la forma estándar y longeva es la siguiente:

    function functionName([param1, param2, param3, ..., paramN]){
    // instrucciones
    [return value]
}

    Para declarar una función de forma estándar es necesario hacer uso de la palabra reservada function seguido del nombre de la función (functionName) y paréntesis. Dentro de estos paréntesis se pueden declarar los parámetros necesarios para usar dentro de la función. Los parámetros son variables que se necesitan usar dentro de las funciones y que en cada llamada de la función tiene valores diferentes. Una función puede recibir ningún parámetro.

    Las funciones, además, pueden retornar un valor que puede ser usada en variables o en otras funciones. Si una función no retorna ningún valor, estaríamos hablando técnicamente de un procedimiento.

    Veamos un ejemplo de una función que recibe dos números, y retorne la suma de ambos:

    function suma(sumando1, sumando2){
    return sumando1 + sumando2
}

    Ahora, veamos un ejemplo de una función con dos retornos. En este caso, en el momento que retorne una de los dos valores, abandona el flujo de la función. Una función aunque tenga varios return solo puede retornar un valor:

    function isPair(number){
    if(number % 2 === 0){
        return 'es par'
    }

    return 'es impar'
}

    En el ejemplo anterior, si el número obtenido como parámetro es par, retorna el valor de es par. Por el contrario, si el número es impar, no entra en la condición y obvia el primer return, por lo que retornaría es impar. Veamos un ejemplo de procedimiento:

    function sayHi(){
    console.log('Hi')
}

    Recuerda

    En este punto, es bueno recordar el ámbito de vida de una variable, y recordar la diferencia entre ámbito de vida local y ámbito de vida global, tal y como lo vimos en el tema dos

    Llamada¶

    Para realizar la llamada de una función, se usa la siguiente estructura: functionName([arg1, arg2, ..., argN]). Se le puede indicar los argumentos que reciba.

    La diferente entre argumento y parámetro es que un parámetro es una variable que se define a la hora de declarar la función y un argumento es el valor del parámetro a la hora de la llamada de la función.

    Warning

    Si una función no recibe parámetros pero en su llamada le pasamos argumentos, estos serán omitidos y Javascript no lanzaría error. Lo mismo ocurriría si recibe X parámetros, y se le pasa más argumentos de los deseados:

    function sayHi(){
    console.log('Hi!')
}

sayHi('Hola mundo')

    Lo mismo ocurre, en el caso contario, si una función define un parámetro y este no es recibido como argumento, se entiende que el valor de ese parámetro es undefined:

    function sayHi(msg){
    console.log(msg)
}

sayHi()

    En el siguiente ejemplo, podemos comprobar mejor la diferencia entre procedimiento y función:

    function sum(num1, num2){
    return num1 + num2
}

function show(msg){
    console.log(msg)
}

let sumValue = sum(2, 4)

show(sumValue)

    Como vemos en el ejemplo, la función sum retorna la suma de los dos parámetros que recibe, por lo que el valor retornado puede ser usado en otra variable o como argumento de otra función. Mientras que, la función show es un procedimiento, ya que no retorna nada, y no puede ser usada en ninguna variable. Sin embargo, si por error es usada, el valor de dicha variable será undefined ya que el procedimiento no retorna ningún valor, pero podemos entender que por defecto retorna undefined.

    Parámetros predeterminados¶

    Como hemos dicho anteriormente, si un parámetro no es pasado como argumento en una función su valor predeterminado es undefined. Sin embargo, algunos lenguajes como Javascript permite definir un valor predeterminado para dichos parámetros para el caso de que no sean usados:

    function sayHi(msg = 'Hi'){
    console.log(msg)
}

sayHi()

    En el ejemplo anterior, como la llamada a la función no recibe parámetros, mostrara por consola el valor predeterminado Hi.

    Warning

    En versiones anteriores, no eran soportados los parámetros predeterminados. Entonces, se usa un condicional para comprobar si el valor era undefined y se indicada un valor:

    function sayHi(msg){
    if(msg === undefined){
        msg = 'Hi'
    }

    console.log(msg)
}

    O se usaba el operador ||:

    function sayHi(msg){
    msg = msg || 'Hi'
    console.log(msg)
}

    Nomenclatura¶

    Las funciones son acciones, por lo que se recomienda que su nombre sea un verbo. Debe ser breve, lo más preciso posible y describir lo que hace la función, para que alguien que lea el código obtenga una indicación de lo que hace la función.

    Es una práctica generalizada comenzar una función con un prefijo verbal que describe vagamente la acción. Debe haber un acuerdo dentro del equipo sobre el significado de los prefijos.

    Por ejemplo, funciones que comienzan con show usualmente muestran algo.

    Funciones que comienza con…

    • get… – devuelven un valor
    • calc… – calculan algo
    • create… – crean algo
    • check… – revisan algo y devuelven un boolean, etc.

    Ejemplos de este tipo de nombres:

    showMessage(..)     // muestra un mensaje
getAge(..)          // devuelve la edad (la obtiene de alguna manera)
calcSum(..)         // calcula una suma y devuelve el resultado
createForm(..)      // crea un formulario (y usualmente lo devuelve)
checkPermission(..) // revisa permisos, y devuelve true/false

    Tips

    Es altamente recomendable que una función solo tenga un objeto, es decir, una acción. Una función debe hacer exactamente lo que sugiere su nombre, no más.

    Dos acciones independientes por lo general merecen dos funciones, incluso si generalmente se convocan juntas (en ese caso, podemos hacer una tercera función que llame a esas dos).

    Algunos ejemplos de cómo se rompen estas reglas:

    • getAge – está mal que muestre una alert o un log con la edad (solo debe obtenerla).
    • createForm – está mal que modifique el documento agregándole el form (solo debe crearlo y devolverlo).
    • checkPermission – está mal que muestre el mensaje acceso otorgado/denegado(solo debe realizar la verificación y devolver el resultado).

    En estos ejemplos asumimos los significados comunes de los prefijos. Tú y tu equipo pueden acordar significados diferentes, aunque usualmente no muy diferente. En cualquier caso, debe haber una compromiso firme de lo que significa un prefijo, de lo que una función con prefijo puede y no puede hacer. Todas las funciones con el mismo prefijo deben obedecer las reglas. Y el equipo debe compartir ese conocimiento.

    También se considera que una buena práctica que una función sea lo más breve posible. En caso de que una función sea demasiado larga, se debería revisar y comprobar que el código no se puede dividir en más funciones. De esta forma, se obtiene una serie de ventajas, como una mayor legibilidad, más facilidad a la hora de probar y depurar.

    Recursividad¶

    La recursividad es un patrón de programación que es útil en situaciones en las que una tarea puede dividirse naturalmente en varias tareas del mismo tipo, pero más simples. O cuando una tarea se puede simplificar en una acción fácil más una variante más simple de la misma tarea. O, como veremos pronto, tratar con ciertas estructuras de datos.

    Sabemos que cuando una función resuelve una tarea, en el proceso puede llamar a muchas otras funciones. Un caso particular de esto se da cuando una función se llama a sí misma. Esto es lo que se llama recursividad.

    Esta técnica es peligrosa ya que se pueden generar fácilmente llamadas infinitas (la función se llama a sí misma, tras la llamada se vuelve a llamar a sí misma, y así sucesivamente sin freno ni control). Por lo tanto, es muy importante tener en cuenta cuándo la función debe dejar de llamarse.

    Hay que ser muy cauteloso cuando se utiliza la recursividad, pero permite soluciones muy originales y abre la posibilidad de solucionar problemas muy complejos.

    Veamos un ejemplo sin recursividad:

    function multiplicate(num1, num2){
    let result = 0

    for(let i = 1; i <= num2; i++){
        result += num1
    }

    return result
}

    En ele ejemplo anterior observamos como se realiza la multiplicación de dos números realizando suma. En este caso, se está utilizando la iteración. Le sumamos a la variable result el valor del argumento num1 tantas veces como se indica en num2. Ahora bien, cuando hacemos una multiplicación como ejemplo 2*3, ¿no es lo mismo que 2+2*2?, o lo que es lo mismo, 2 + 2 + 2 * 1. En este caso, el indice que indica el número de veces a sumar se va reduciendo. En este caso, estamos pensando haciendo uso de la recursividad, ya que cada vez que hacemos una multiplicación estaríamos llamando a nuestra función.

    function multiplicate(num1, num2){
    return num1 + multiplicate(num1, num2 - 1)
}

    Sin embargo, el ejemplo anterior da lugar a que sea infinito, ya que no se controla cual debe ser la última recursión, que en este caso sería cuando num2 valga 1. Esta sentencia de control, se llama caso base es necesaria en la funciones recursivas, ya que indican la última ejecución de la recursividad.

    function multiplicate(num1, num2){
    if(num2 === 1){
        return num1
    }

    return num1 + multiplicate(num1, num2 - 1)
}

    Si se desea hacer la multiplicación 2 * 3:

    1. En la ejecución inicial no se ejecuta el bloque if si no que se ejecuta 2 + multiplicate(2, 2)
    2. En la siguiente, sigue sin entrar en el if, por lo que se ejecuta 2 + multiplicate(2, 1)
    3. En esta ejecución retornaría 2, ya que num2 vale 1 y entraría en el if.
    4. Una vez retornado 2, podemos completar la ejecución del paso 2, retornando 2 + 2, que sería 4.
    5. Por último, usaríamos dicho valor, en la ejecución del paso 1, 2 + 4, dando como resultado 6.

    El número máximo de llamadas anidadas (incluida la primera) se llama profundidad de recursividad. En nuestro caso, será exactamente num2.

    La profundidad máxima de recursividad está limitada por el motor de JavaScript. Podemos confiar en que sea 10 000; algunos motores permiten más, pero 100 000 probablemente esté fuera del límite para la mayoría de ellos. Hay optimizaciones automáticas que ayudan a aliviar esto (“optimizaciones de llamadas de cola”), pero aún no tienen soporte en todas partes y funcionan solo en casos simples.

    Eso limita la aplicación de la recursividad, pero sigue siendo muy amplia. Hay muchas tareas donde la forma recursiva de pensar proporciona un código más simple y fácil de mantener.

    Recursividad vs iteración¶

    Ambas implican sentencias repetitivas hasta llegar a una determinada condición, por lo que ambas pueden generar programas que no finalizan si la condición nunca se cumple. En el caso de la iteración es una condición la que permite determinar el final, la recursividad lo que hace es ir simplificando el problema hasta generar una llamada a la función que devuelva un valor y no se vuelva a llamar. Para un ordenador es más costosa la recursividad ya que implicar realizar muchas llamadas a funciones, es decir, es más rápida la solución iterativa. Entonces, ¿por qué elegir recursividad? La recursividad se utiliza sólo sí:

    • No encontramos la solución iterativa a un problema.
    • El código es mucho más claro en su versión recursiva.

    Pila¶

    Una pila (stack) es una lista ordenada o estructura de datos que permite almacenar y recuperar datos, el modo de acceso a sus elementos es de tipo LIFO (del inglés Last In, First Out, último en entrar, primero en salir) de supuestos en el área de informática debido a su simplicidad y capacidad de dar respuesta a numerosos procesos.

    Para el manejo de los datos cuenta con dos operaciones básicas: apilar (push), que coloca un objeto en la pila, y su operación inversa, desapilar (pop), que retira el último elemento apilado.

    En cada momento sólo se tiene acceso a la parte superior de la pila, es decir, al último objeto apilado (denominado TOS, Top of Stack). La operación desapilar permite la obtención de este elemento, que es retirado de la pila permitiendo el acceso al anterior (apilado con anterioridad), que pasa a ser el último, el nuevo TOS.

    Para las llamadas entre funciones, se utiliza una estructura de tipo pila: supongamos que se está procesando una función y en su interior llama a otra función. La función se abandona para procesar la función de la llamada, pero antes se almacena en una pila la dirección que apunta a la función. Ahora supongamos que esa nueva función llama a su vez a otra función. Igualmente, se almacena su dirección, se abandona y se atiende la petición. Así en tantos casos como existan peticiones. La ventaja de la pila es que no requiere definir ninguna estructura de control ni conocer las veces que el programa estará saltando entre funciones para después retomarlas, con la única limitación de la capacidad de almacenamiento de la pila. Conforme se van cerrando las funciones, se van rescatando las funciones precedentes mediante sus direcciones almacenadas en la pila y se va concluyendo su proceso, esto hasta llegar a la primera.

    En el caso de una función recursiva, esto es posible implementarlo con sencillez mediante una pila. La función se llama a sí misma tantas veces como sea necesario hasta que el resultado de la función cumpla la condición de retorno; entonces, todas las funciones abiertas van completando su proceso en cascada. No se necesita saber cuantas veces se anidará y, por tanto, tampoco cuando se cumplirá la condición, con la única limitación de la capacidad de la pila. De sobrepasarse ese límite, normalmente porque se entra en un bucle sin final, se produce el error de desbordamiento de la pila (stack overflow).

    Supongamos que tenemos la función multiplicate(2, 3), en la primera llamada, volvemos a llamar el valor multiplicate(2, 2) poniendo esta llamada en la primera posición de la pila. La llamada de multiplicate(2, 2) se ejecuta y pasa a llamar multiplicate(2, 1), apilando esta llamada al comienzo de la pila. Al ejecutar multiplicate(2, 1) retorna 1, por lo que se desapila de la pila y con el valor retornado va operando las llamadas de la pila multiplicate(2, 2) la de multiplicate(2, 3).

  if (age > 18) {
    return true;
  } else {
    // ...
    return confirm('¿Tus padres te permitieron?');
  }
