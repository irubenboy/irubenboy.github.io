    2 Referencias¶

    Introducción¶

    Una de las diferencias entre objetos y primitivos es que los objetos son almacenados y copiados por referencia, son almacenados y copiados por valor completo

    Una referencia en cualquier lenguaje de programación hace alusión a la posición en memoria RAM que tiene una variable. Una variable es conceptualmente un recipiente donde se almacena un dato.

    Una variable es un lugar en la memoria donde se guarda un dato. Para ser exacto, este lugar en la memoria es la Pila o Stack. Pero cuando se crea un objeto, el objeto se guarda en una parte de la memoria llamada Heap.

    Stack es una sección de memoria que contiene métodos, variables locales y variables de referencia. La memoria de pila siempre se referencia en el orden de último en entrar primero en salir. Las variables locales se crean en la pila.

    Heap es una sección de memoria que contiene objetos y también puede contener variables de referencia. Las propiedades se crean en el heap.

    Representación en memoria de tipos primitivos¶

    Veamos paso a paso un ejemplo:

    • Cuando se declara una variable de un tipo primitivo, el compilador reserva un área de memoria para ella:

      const i, j

    • Cuando se asigna un valor, éste es escrito en el área reservada:

      i = 16
j = 3

    • La asignación entre variables significa copiar el contenido de una variable en la otra:

      i = j;

    • La comparación entre variables compara los contenidos de las mismas:

      i == j; // true porque 3 es igual a 3

    const i = 10, j = 20

console.log("Valor de la variable i:", i)
console.log("Valor de la variable j:", j)

i = j

console.log("Valor de la variable i:", i)
console.log("Valor de la variable j:", j)

console.log(`i y j${i == j ? "" : " no"} tiene el mismo contenido`)

    Representación en memoria de objetos¶

    Supongamos que tenemos una objeto Complex con dos atributos: r, i.

    • Cuando se declara una variable de un objeto, el compilador reserva un área de memoria para ella. Mientras no se cree ningún objeto con el operador {}, lo que contendrá será undefined. La palabra reservada undefined indica que una variable que referencia a un objeto se encuentra de momento sin referenciar a ninguno:

      const c1

    • Cuando se crea el objeto llamando con el operador {}, lo que se guarda en la variable complex1 es la dirección de memoria donde se ha almacenado el objeto en el heap, es decir, la diferencia entre los tipos primitivos y los objetos es que en los primitivos la variable almacena el valor y en los objetos, la variable almacena la dirección de memoria donde se encuentra el objeto.

      c1 = {
    r: 7.2,
    i: 2.4
}

    • La asignación entre variables de objetos significa copiar la dirección de memoria donde se encuentra el objeto:

      const c2 = c1;

      En este caso, el valor de la variable c2 es 0x2A18, por lo tanto, ambas variables apuntan al mismo objeto. Si una de ellas modifica algún valor del objeto, también le afectará a la otra variable. Por ejemplo, si a c2 le cambiamos el valor de r a 9.7, si consultamos el valor de r de c1, también valdrá 9.7, en lugar de 7.2.

    • La comparación entre referencias no compara los contenidos de los objetos sino las direcciones de memoria, es decir, si apuntan al mismo sitio.

      c2 == c1 // (1)!
const c3 = new Complex(7.2, 2.4)
c3 == c1 // (2)!
      1. true porque apuntan al mismo sitio
      2. false porque aunque tengan los mismos valores, no apuntan al mismo sitio

      Para comparar el contenido de los objetos, se debe utilizar el operador ===:

      c3 === c1 // (1)!
      1. true porque los contenidos de los objetos son los mismos
    const c1 = {r: 7.2, i: 2.3}
const c2 = c1

console.log("C1 -> r: " c1.r, "i: ", c1.i)
console.log("C2 -> r: " c2.r, "i: ", c2.i)

c1.r = 10
c2.j = 7.6

console.log("C1 -> r: " c1.r, "i: ", c1.i)
console.log("C2 -> r: " c2.r, "i: ", c2.i)

const c3 = {r: 10, i: 7.6}
console.log("C2 -> r: " c3.r, "i: ", c3.i)

console.log("c1 == c2 -> ", c1 == c2)
console.log("c1 == c3 -> ", c1 == c3)
console.log("c1 === c3 -> ", c1 === c3)

    Garbage Collector¶

    Garbage Collector (recolector de basura) es un programa que se ejecuta en el motor Javascript que elimina los objetos que ya no están siendo utilizados por una aplicación. Es una forma de gestión automática de la memoria.

    Un objeto es elegible para el recolector de basura cuando deja de existir alguna referencia hacia él. Veamos algunos ejemplos.

    const c1 = { r: 7.2, i: 2.4 };
//...
c1 = null;
//...

    Existe un objeto referenciado por la variable c1. Cuando la variable c1 pierde la referencia porque se le asigna el null, se pierde cualquier forma de acceder al objeto, de modo que pasa a ser elegible para el recolector de basura.

    const c1 = {r: 7.2, i: 2.4};
const c2 = c1;
// ...
c1 = null;
// ...

    En este segundo ejemplo, el objeto no pasa a ser elegible para ser recolectado pues aunque la variable c1 haya perdido la referencia porque se le asigna el null, todavía existe una referencia hacia el objeto por la variable c2

    const c1 = {r: 7.2, i: 2.4};
//...
c1 = {r: 8.3, i: 2.7};
//...

    En este otro ejemplo, la variable c1 es reasignada, es decir, se le asigna otro objeto mediante el operador new, por lo tanto se pierde cualquier referencia al objeto creado al principio new Complex(7.2 , 2.4) por lo que dicho objeto pasa a ser elegible para el recolector de basura.

    Clonación¶

    El problema de la referencia, es que en ciertas ocasiones es necesario crear una copia de un objeto que se pueda manipular sin tener que manipular el objeto real.

    Una solución para este problema es iterar el objeto original e ir copiando los valores de forma primitiva en otro objeto:

    const obj = {
    name: "John",
    age: 17
}

const copy = {}

for(const key in obj){
    copy[key] =  obj[key]    
}

console.log(obj, copy) //{name: "John", age: 17}, {name: "John", age: 17}

copy.age = 20


console.log(obj, copy) //{name: "John", age: 17}, {name: "John", age: 20}

    También existe el método estático Object.assign(dest, ...sources) que copia el valor de sources en el objeto de destino dest.

    const user = { name: "John" };

const permissions1 = { canView: true };
const permissions2 = { canEdit: true };

Object.assign(user, permissions1, permissions2);

console.log(user.name); // John
console.log(user.canView); // true
console.log(user.canEdit); // true

    Si la propiedad ya existe, el método sobrescribe el valor:

    const user = { name: "John" }

Object.assign(user, { name: "Pete" })

alert(user.name) // ahora user = { name: "Pete" }

    Objetos anidados¶

    Hasta ahora, todos las propiedades de los objetos que hemos visto tenían un valor primitivo. Cierto es, que una propiedad de un objeto, también puede ser otro objeto:

    const user = {
  name: "John",
  sizes: {
    height: 182,
    width: 50
  }
}

console.log(user.sizes.height) // 182

    Si se le asigna con el método Object.assign(), el objeto interno no se copiará, si no que referenciará, por lo que volvemos a tener el mismo problema:

    const user = {
  name: "John",
  sizes: {
    height: 182,
    width: 50
  }
}

const clone = Object.assign({}, user)

alert( user.sizes === clone.sizes ) // true

user.sizes.width = 60
console.log(clone.sizes.width) // 60

    Para corregir esto, debemos hacer que user y clone sean objetos completamente separados, debemos usar un bucle que examine cada valor de user[key] y, si es un objeto, que replique su estructura también. Esto es conocido como “clonación profunda” o “clonación estructurada”. Existe un método structuredClone que implementa tal clonación profunda.

    El método structuredClone(obj) clona el objeto pasado como argumento con todas sus propiedades anidadas, es decir, hace una copia profunda.

    const user = {
  name: "John",
  sizes: {
    height: 182,
    width: 50
  }
}

const clone = structuredClone(user)

console.log(user.sizes === clone.sizes) // false

user.sizes.width = 60
console.log(clone.sizes.width) // 50
