    7 Conversión de tipos¶

    Introducción¶

    La mayoría de las veces, los operados y las funciones realizan conversiones automáticas de los tipos que reciben por el tipo que requieren. Por ejemplo, cualquier valor pasado al método log del console es convertido a cadena.

    También, en ciertas ocasiones, es necesario realizar dicha conversión de forma explícita. Existen diferentes funciones para realizar dicha conversión.

    Conversión explícita a cadena¶

    Vamos a convertir un valor a cadena cuando necesitamos que dicho valor sea usado como tal. Es una de las conversiones más sencillas, ya que sea obtiene el mismo valor pero con comillas. Por ejemplo, 2 sería "2", undefined "undefined", false "false".

    Para convertir un valor a una cadena podemos hacer uso de la función String(value)

    const number = 3
const numberString = String(number)

    También podemos usar las backticks introduciendo el valor a convertir dentro del ${...}:

    const number = 3
const numberString = `${number}`

console.log(typeof numberString) // string

    Conversión explícita a número¶

    La conversión numérica ocurre en funciones matemáticas y expresiones. Por ejemplo, al hacer "3" / "3", las cadenas se convierten automáticamente en número. Sin embargo, si se desea hacer una conversión explícita, es necesario hacer uso de la función Number(value).

    Warning

    El único operador matemático que no realiza la conversión de cadenas automática a número es el operador de adición +. Este operador se entiende en dicho caso, como un operador de concatenación de cadenas. Para poder hacer una operación de adición de una cadena y un número, es necesario hacer la conversión explícita de la cadena a número:

    console.log(3 + "3") // 33
console.log(3 + Number("3")) // 6

    Para pasar una cadena a número se sigue los siguientes pasos:

    1. Se eliminan los espacios de las cadenas. Por lo que, si tenemos una cadena `" 233 ", los espacios no afectarían a la conversión.
    2. Si la cadena resultante es vacía, se convierte al número 0.
    3. Si la cadena resultante es una cadena de dígitos (solo contiene números), se convierte a número.
    4. Si la cadena resultante es una cadena que contiene algún carácter distinto a un dígito, la conversión sería errónea y retornaría NaN.

    Existen cadenas especiales que pueden ser convertidos a números de base de decimal (base 10). Por ejemplo, para indicar que una cadena es una representación numérica octal, se debe usar 0o. 0x y 0b representan una cadena hexadecimal y binaria, respectivamente:

    console.log(Number("0o10")) // 8 (octal)
console.log(Number("2e1")) // 20 (notación exponencial)
console.log(Number("0b10100")) // 20 (binaria)
console.log(Number("0xF")) // 15 (hexadecimal)

    Por desgracia, el método Number no puede parsear unidades de medida a número:

    console.log('20px') // NaN

    Si se desea parsear una valor booleano a un número, será convertido a 0 si el valor del booleano es false, y a 1 si es true.

    console.log(Number(true)) // 1
console.log(Number(false)) // 0

    De la misma manera, el tipo undefined se convierte como NaN y el tipo null se convierte a 0.

    console.log(Number(null)) // 0
console.log(Number(undefined)) // 1

    Existen otros métodos para parsear un valor a número, parseInt(value) y parseDouble(value). Su funcionamiento es similar, pero en el primer caso realiza la conversión a número entero, y en el segundo a número decimal.

    Warning

    Los métodos parseInt y parseDouble solo sirven para pasar cadenas con dígitos. Cualquier otro tipo de valor (incluso las cadenas especiales para representar números de otras bases) retornaría NaN como resultado.

    console.log(parseInt(false)) // NaN
console.log(parseInt("0b1000")) // NaN

    Las funciones pueden recibir un segundo parámetro que indique la base a la que se desea convertir la cadena, sin necesidad de hacer el uso de cadenas especiales, como en el caso de Number(). Si no lo recibe, se usaría la base diez (decimal) por defecto.

    console.log(parseInt("101000", 2)) // 20
console.log(parseInt("010"), 8) // 8
console.log(parseInt("0xF"), 16) // 15

    Al usar este número con una cadena que comience con un número, convierte la cadena en dicho número ignorando el resto de la cadena. Por ello, este método, si permite parsear unidades:

    console.log(parseInt("2000 hola mundo")) // 2000
console.log(parseInt("20 hola 50")) //20
console.log(parseInt("0b10100")) // 0
console.log(parseInt("20em")) // 20

    Existe una manera más sencilla de realizar la conversión de manera explícita, haciendo uso del operador unario +. Al usar dicho operador delante del valor a convertir, se convierte de forma automática. Aunque si se usa en operaciones con varios operando, hay que tener en cuenta la precedencia, aunque dicho operador tiene mayor precedencia que los operadores matemáticos.

    console.log(+"3") //3
console.log(+false) //0
console.log(+true) // 1
console.log(+null) // 0
console.log(+undefined) //NaN
console.log(+"Hola") // NaN
console.log(+"0b10100") // 20
console.log(+"2e1") // 20

console.log(3 + +"4") // 7

    Conversión explícita a boolean¶

    Convertir a booleano es una de las operaciones de conversión más simples. Siguiendo las siguientes reglas:

    • Los valores vacíos, como undefined, 0, null, '', NaN se convierten a false.
    • El resto de valores se convierten a true.

    Para realizar la conversión explícita de los valores, se hace uso de la función Boolean(value):

    console.log(Boolean(1)) // true
console.log(Boolean(0)) // false

console.log("") // false
console.log("Hola") // true

    Warning

    Algunos lenguajes, como PHP, entiende la cadena "0" como false. Pero, en Javascript, cualquier cadena no vacía es siempre true. Al igual que ocurre con una cadena con solo espacios " "

    console.log(Boolean("0")) // true
console.log(Boolean(" ")) // true

    Existen otras formas de conversión implícitas a booleanos, como los operadores lógicos, las estructura if, etc.
