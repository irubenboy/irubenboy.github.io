    3 Variables y constantes¶

    Introducción¶

    A la largo del desarrollo de un script es necesario utilizar cierta información de forma repetida, por lo que es útil poder almacenar dicha información y poder usarlo en las partes del código en las que sean necesarias.

    En programación, existen las variables y las constantes para desempeñar dicha función.

    Variables¶

    Las variables son contenedores que sirven para almacenar los datos que utiliza un programa. Dicho más sencillamente, son nombres que asociamos a determinados datos. La realidad es que cada variable ocupa un espacio en la memoria RAM del ordenador para almacenar el dato al que se refiere. Es decir, cuando utilizamos el nombre de la variable realmente estamos haciendo referencia a un dato que está en memoria.

    En Javascript existen dos limitaciones para nombrar a una variable:

    1. El nombre únicamente puede incluir letras, dígitos o los símbolos $ y _. Ejemplo: $var, _var, var123.
    2. El primer carácter no puede ser un dígito. Ejemplo erróneos: 1var.

    Se recomienda hacer uso de nombres descriptivos pero breves. La elección de un nombre de variable debe ser mnemónico, es decir, diseñado para indicar al observador casual la intención de su uso. Por ejemplo, si queremos usar una variable para almacenar una edad, la llamaremos edad. Además se debería evitar los nombres de variables de un solo carácter excepto para las variables temporales usar y tirar. Los nombres comunes de las variables temporales son i, j, k, m, y n para enteros; c, d, y e para los caracteres.

    El estilo del nombre de las variables suele camelCase (o lowerCamelCase), es decir, las palabras van seguidas una detrás de otra, con cada inicial en mayúscula, excepto la primera que va en minúscula: miPrimeraVariable.

    Warning

    Es importante tener en cuenta la capitalización, ya que el mismo nombre escrito de formas diferentes da a lugar a dos variables distintas. manzana y MANZANA son dos variables distintas a pesar de tener el mismo nombre.

    Javascript permite escribir letras de cualquier alfabeto incluyendo letras del cirílico, logogramas chinos, etc., incluso nuestra letra ñ, pero no se recomienda su uso ya que existe una tradición internacional de utilizar inglés en el nombramiento de variables. Incluso si estamos escribiendo un script pequeño, este puede tener una larga vida por delante. Puede ser necesario que gente de otros países deba leerlo en algún momento.

    Otra prohibición a tener en cuenta a la hora de nombra variables es que no se pueden utilizar palabras que usa el mismo lenguaje. Estas palabras son denominadas como palabras reservadas. Por ejemplo, no se puede usar la palabra let como nombre de variable. En la siguiente web, puedes conocer el listado oficial de este tipo de palabras: https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Lexical_grammar#Keywords

    Declaración de una variable¶

    Para declarar una variable es necesario usar la palabra let seguida del nombre de la variable:

    let miPrimeraVariable

    Se puede inicializar una variable, o cambiar el valor de la misma haciendo uso del operador =:

    miPrimeraVariable = 10

    Además, se puede acceder a dicha variable usando su nombre:

    let message 

message = 'Hello world'

console.log(message)

    Javascript permite definir una variable, es decir, declarar e inicializar una variable en una única línea:

    let message = "hello world"

    También se puede definir y declarar varias variables en una sola línea, separadas por coma ,:

    let name = 'John Doe', age = 25, message = 'Hello world'

    Lo anterior, se podría hacer en múltiples línea, aunque no se recomienda:

    let name = 'John Doe', 
    age = 25, 
    message = 'Hello world'

    Tips

    Por norma general, se recomienda definir o declarar las variables en líneas diferentes con diferentes let. Aunque, es cierto que si son pocas variables y con información fácil de leer, se podría usar la forma múltiple lineal sin problema.

    Warning

    Declarar dos veces la misma variable produce un error de ejecución.

    let message = "This";


let message = "That"; // SyntaxError: 'message' ya fue declarado

    Funcionamiento de una variable¶

    Podemos entender una variable como una caja que se encarga de almacenar información relevante para nuestro código. Cuando una variable es declarada (no inicializada), estamos creando dicha caja, y una vez inicializada, almacenamos en la caja el valor correspondiente. Cuando una variable es modificada, su valor actual es eliminado de la caja y se inserta el nuevo valor en ella. Veamos un ejemplo:

    let message //(1)!

message = 'Hello world' //(2)!

message = 'Goodbye' //(3)!
    1. Se crea la caja para almacenar información.
    2. Se almacena el valor Hello World por primera vez en la caja.
    3. Se elimina el valor Hello World y se añade el nuevo valor Goodbye.

    Esto en realizar se refiere a que cuando creamos una variable, se reserva un espacio de memoria y al inicializarla se escribe el dato deseado en su interior. Cuando se modifica, ese dato se elimina y se sobreescribe el nuevo valor.

    Ámbito de vida de las variables¶

    Toda variable tiene un ámbito de vida. Esto es la parte del código en la que una variable se puede utilizar, que es en el bloque donde se ha declarado. De hecho las variables tienen un ciclo de vida:

    1. En la declaración se reserva el espacio necesario para que se puedan comenzar a utilizar (digamos que se avisa de su futura existencia)
    2. Se la asigna su primer valor (la variable nace)
    3. Se la utiliza en diversas sentencias. Cuando finaliza el bloque en el que fue declarada, la variable muere. Es decir, se libera el espacio que ocupa esa variable en memoria.
    4. Una vez que la variable ha sido eliminada, no se puede utilizar. Dicho de otro modo, no se puede utilizar una variable más allá del bloque en el que ha sido definida. Ejemplo:

    Podemos entender que una variable puede tener un ámbito de vida local o global. Podemos entender como una variable global, como aquella variable que puede ser accedida desde cualquier parte de nuestro script, es decir, la variable es declarada en el script y puede ser usada tanto dentro como fuera de cualquier bloque.

    let number = 10

console.log(number)

    Por el contrario, una variable local es una variable que se declara dentro de un bloque y solo puede ser usada en dicho bloque y nunca fuera de él:

    {
    let number = 10

    console.log(number)
}

console.log(number) // ❌ ReferenceError: number is not defined

    Warning

    Normalmente, debemos definir una variable antes de utilizarla. Pero, en los viejos tiempos, era técnicamente posible crear una variable simplemente asignando un valor sin utilizar let. Esto aún funciona si no hacemos uso de use strict en nuestros scripts para mantener la compatibilidad con scripts antiguos.

    num = 5; // se crea la variable "num" si no existe antes

alert(num); // 5

    Esto sería una mala práctica que se puede evitar haciendo del modo estricto:

    "use strict";

num = 5; // ❌ error: num no está definida

    Constantes¶

    Una constante es un valor que no puede ser modificado durante la ejecución de un programa, únicamente puede ser leído. Para crear una constante en Javascript se usa la palabra reservada const seguida del nombre de la variable y su inicialización:

    const miPrimeraConstante = 10

    Al intentar modificar el valor de una variable produciría un error:

    const miPrimeraConstante = 10

miPrimeraConstante = 20 // ❌ ERROR

    Tips

    Existe una buena práctica de escribir en mayúsculas y con guiones _, aquellas constantes que hacen referencia a ciertos datos que son difíciles de recordar, como por ejemplo el código hexadecimal de un color:

    const COLOR_RED = "#F00";
const COLOR_GREEN = "#0F0";
const COLOR_BLUE = "#00F";
const COLOR_ORANGE = "#FF7F00";

    De esta forma podemos acceder al color rojo de una manera más sencilla usando su nombre de constante.

    Se aconseja escribir en mayúsculas aquellas constantes cuyo valor se conocen antes la ejecución del programa y en minúsculas aquellas cuyo valor se deben calcular u obtener durante la ejecución:

    const NAME = 'Server name'

const div = /*...*/

    En el ejemplo anterior, la variable NAME se escribe en mayúsculas ya que es un valor que se conoce antes de ejecutar el script. Sin embargo, la variable div representa un elemento div del HTML por lo que no es un valor que se obtiene antes de la ejecución del programa, sino que, por el contrario debe ser obtenido una vez se haya ejecutado el programa.

    var¶

    ¡OJO!

    La información transcrita en el siguiente punto tiene como objetivo único INFORMAR sobre ciertos conocimientos antiguos de Javascript, por lo que NO se deberían llevar a la práctica.

    La palabra reservada var se utilizaba en versiones anteriores de Javascript y tiene una utilización bastante similar a la palabra reservada let. Sin embargo, declarar una variable con var tiene ciertos usos peligrosos que fueren limitados con la incorporación de let.

    Cuando declaramos una variable con let de forma global en nuestro script, dicha variable puede ser usada en cualquier parte de nuestro script, por lo que no se puede acceder a ella desde otros script (en caso de tener más de una etiqueta script en nuestro HTML, o diferentes módulos).

    <script>
    let number = 10
</script>
<script>
    'use strict'
    console.log(number) // ❌ ERROR
</script>

    Sin embargo, haciendo uso de la palabra var, esto si puede suceder, de forma que se puede declarar una variable en un script y poder usarlo en cualquier cualquier otro script, y en cualquier parte de nuestro código, incluso si se declara dentro de un bloque:

    <script>
    {
        var number = 10
    }
</script>
<script>
    'use strict'
    console.log(number)
</script>

    Lo que ocurre en el ejemplo, no es una buena práctica ya que exponemos a que nuestras variables sean modificadas desde cualquier parte, dentro o fuera de nuestro script.

    Otra de las malas prácticas que tiene el uso de var, es que permite redeclarar una variable, cosa que con el uso de let se ve limitado ya que lanza un error:

    let number = 10
let number = 20 // ❌ ERROR

var number = 10
var number = 20

    En este caso, se ignora la declaración y únicamente se cambia su valor.

    Otra de las malas prácticas que tiene su uso es el hoisting (elevamiento). Permite inicializar la variable incluso antes de su declaración, ya que no importa en el lugar que se declare porque siempre va a ser elevada al principio. Veamos un ejemplo:

    phrase = "Hello";

console.log(phrase);

var phrase;

    En el código anterior, permite inicializar la variable, y luego se declarar, aunque en realizar lo primero que se ejecuta es la declaración debido al elevamiento. Esto puede provocar una difícil legibilidad. El código anterior, se ejecutaría de esta forma:

    var phrase;

phrase = "Hello";

console.log(phrase);

    Ejercicios¶

    Ejercicio 1

    Crea un script que realice los siguientes pasos:

    1. Declara dos variables: admin y name.
    2. Asigna el valor John a la variable name.
    3. Copia el valor de name en admin.
    4. Muestra por consola el valor de la variable admin, en color verde.
    Ejercicio 2

    ¿Cómo nombrarías las siguientes variables?:

    • Nombre actual del usuario que ha iniciado sesión en nuestra web.
    • Nombre de nuestro planeta.
    • Tiempo transcurrido desde el comienzo de una tarea, hasta su finalización.
    • Fecha de nacimiento de un empleado.
    • Capacidad máxima de un disco duro.
    • Capacidad ocupada de un disco duro.
    • Capacidad sobrante de un disco duro.
    Ejercicio 3

    Dado el siguiente código, ¿cuál dirías que debería ir en mayúsculas? Justifica tu respuesta:

    const birthday = '18.04.1982';

const age = calculateAgeFromDate(birthday);
