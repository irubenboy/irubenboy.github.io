    4 Tipos de datos básicos¶

    Introducción¶

    Javascript es un lenguaje tipado débil, es decir, no te obliga a indicar el tipo de las variables a la hora de declararlas. Sin embargo, esto no quiere decir que una variable no sea de un tipo especificado y que es bueno conocer los tipos de datos con los que se va a trabajar.

    Javascript también es un lenguaje dinámicamente tipado. Esto quiere decir, que una variable puede ser declarada como un tipo, por ejemplo una cadena, y durante la ejecución del programa puede pasar a ser un número.

    let variable = "Hello"

variable = 123

    Aunque esto pueda parecer un gran acierto en Javascript, es cierto, que en ciertas ocasiones puede ser un problema, dependiendo de la utilidad que se le quiera dar a dicha variable. Por ejemplo, si queremos hacer una división, debemos disponer de dos números, si ahora bien, uno de esos números son introducidos por el usuario y éste introduce una cadena, la operación no sería válida. Por ello, es recomendable realizar comprobaciones de tipo cuando sea necesario

    Existen ocho tipos de datos básicos diferentes en Javascript: Number, BigInt, String, Boolean, null, undefined, Object y Symbol. Además, también existen otras estructuras de datos como las clases y los arrays.

    Number¶

    El tipo number representa los números en Javascript, tanto los números enteros, como los números de punto flotantes (decimales). El tipo number representa los números +(2⁵³+1) y -(2⁵³-1).

    let number = 2
let decimal = 1.60

    Además de los números comunes, existen los denominados valores numéricos especiales que pertenecen a este tipo y representan un valor matemático especial:

    • Infinity. Representa el infinito matemático (♾️). Es un número especial que es mayor que cualquier otro número. Haciendo uso del operador -, existirá el -Infinity que representa el infinito negativo, siendo un número más pequeño que cualquier otro. Este número puede ser obtenido al dividir un número entre 0.

    • NaN. Representa un error de cálculo. Es el resultado de un operación matemática incorrecta o indefinida. Por ejemplo, es el resultado de dividir una cadena a un número. Las siglas NaN significan Not a Number (No es número), por lo que indica que el resultado de un operación matemática que espera un número no retorna el tipo esperado. Cualquier operación que se haga con un operando de valor NaN, también devuelve un NaN, excepto NaN ** 0 = 1.

      Info

      Las operaciones matemáticas, en Javascript, son operaciones seguras. A diferencia de otros lenguajes, como en Java, cuando se realizan operaciones vulnerables (dividir por cero, dividir una cadena por un número) no lanza un error fatal, si no que por el contrario, si no que retorna un valor especial matemático como NaN o Infinity, salvaguardando así el flujo de la aplicación.

    BigInt¶

    EL tipo BigInt representan números enteros de tamaño superior a ±(2⁵³-1). Este tipo de número existe, porque cuando se intenta realizar una operación con number cuyo resultado quede fuera de dicho rango, habrá un error de precisión:

    console.log(9007199254740991 + 1); // 9007199254740992
console.log(9007199254740991 + 2); // 9007199254740992

    Para la mayoría de los propósitos, el rango de tipo number es suficiente, sin embargo a veces es necesario números realmente grandes, como para la criptografía o estudios científicos.

    BigInt fue agregado recientemente a Javascript para solventar dicho problema de precisión. Para indicar que un número es de tipo BigInt o no de tipo number se debe agregar una n al final de número:

    const bigInt = 1234567890123456789012345678901234567890n

    String¶

    El tipo de dato String representa una cadena de caracteres y debe colocarse entre comillas:

    const str = "Soy una cadena"

    En Javascript se puede inicializar una cadena haciendo uso de tres tipos de comillas diferentes:

    • Comillas dobles: "Hola".
    • Comillas simples: 'Hola'
    • Comillas invertidas (backticks): `Hola`

    Entre las comillas dobles y las comillas simples no existen diferencias entres ellas, simplemente sirven para mostrar literalmente el contenido de la cadena. Por ejemplo, si tengo la variable sum = "2+3" el valor de la variable es literalmente 2+3 y no 5, que es el resultado de la suma.

    Por otro lado, las comillas backticks son comillas de funcionalidad extendida, es decir, nos permiten incrustar variables y expresiones de tal formal que el valor de la cadena sea el valor de la variable o expresión. Para ello, la variable o la expresión debe estar envuelta en ${...}:

    let name = "John"
let lastName = "Doe"
let fullName = `${name} ${lastName}`

console.log(fullName) // "John Doe"

let number1 = 2
let number2 = 3
let result = `Result is ${number1 + number2}`

console.log(result) // "Result is 5"

    La expresión dentro de ${...} se evalúa y el resultado pasa a formar parte de la cadena. Podemos poner cualquier cosa ahí dentro: una variable como name, una expresión aritmética como 1 + 2, o algo más complejo.

    Warning

    El uso de ${...} solo se puede realizar con las backticks para el resto de comillas no funcionaria, ya que solo muestra el resultado literal:

    const name = "John"
const lastName = "Doe"
const fullName = "${name} ${lastName}"

console.log(fullName) // "${name} ${lastName}"

    Info

    En algunos lenguajes, hay un tipo especial carácter para un solo carácter. Por ejemplo, en el lenguaje C y en Java es char.

    En JavaScript no existe tal tipo. Sólo hay un tipo: string. Un string puede estar formado por un solo carácter, por ninguno, o por varios de ellos.

    Es importante conocer este dato, porque en ocasiones es necesario obtener solo un carácter y el usuario es libre de introducir una cadena más amplía, por lo que debe ser validado correctamente.

    Boolean¶

    El tipo boolean hace referencia a los valores true y false. Este tipo es usado comúnmente para almacenar valores afirmativos (true) o negativos (false). Por ejemplo, sí, correcto, verdadero serían formas de representar el valor true, mientras que no, incorrecto y falso para representar el valor false.

    const isLegalAge = true
const isUserLogged = false

    Los valores booleanos también son los resultados de realizar comparaciones, como veremos más adelante. Por ejemplo, 3 > 4 sería false ya que el número 3 no es mayor que el número 4.

    null¶

    En Javascript, el valor null hace referencia a un objeto inexistente. Sólo es un valor especial que representa nada, vacío o valor desconocido.

    const age = null;

    En el código anterior, la variable age tiene un valor vacío o desconocido por que aún no ha sido calculado o está en espera de ser introducido.

    undefined¶

    En Javascript, cuando una variable es declarada pero no definida se dice que es de tipo undefined que literalmente significa no definida.

    La diferencia con null, es que el tipo undefined quiere decir que la caja que va a contener el valor aún no ha sido definida como tal, mientras que con null, la caja ha sido definida pero está completamente vacía.

    Aunque se puede asignar a una variable el valor undefined es algo que por legibilidad de código no se recomienda. Para ello, se recomienda hacer uso del tipo null.

    Tanto undefined y null son tipos propios de Javascript.

    Ejercicios¶

    Ejercicio 4

    ¿Cuál es la salida del siguiente script?

    let name = "Ilya"

console.log( `Hola ${1}` )    
console.log( `Hola ${"name"}` )    
console.log( `Hola ${name}` )
