    5 Operadores¶

    Introducción¶

    Los operadores son símbolos que se utilizan para evaluar un expresión y así devuelva un resultado. Por ejemplo, el operador + sirve para evaluar una suma y que así devuelva un número. Los operandos son los valores que interactúan en la evaluación, es decir, las partes de la operación.

    Existen diferentes tipos de operadores, entre ellos:

    • Operadores unarios. Son operadores que solo necesitan un único operando.
    • Operadores binarios. Son operadores que necesitan dos operandos, como la suma.
    • Operadores ternarios. Son operadores que necesitan tres operandos.

    Operadores unarios¶

    En Javascript, los símbolos positivo + y negativos - son operadores unarios que evalúan el signo el operador. De esta forma podemos usar - para pasar un número positivo a negativo:

    console.log(+2) // positive number
console.log(-2) // negative number

    Operadores matemáticos¶

    En Javascript tenemos los siguientes operadores para poder realizar operaciones matemáticas:

    Operador Descripción Ejemplo de expresión Resultado del ejemplo
    + Suma 2.5 + 7.1 9.6
    - Resta 235.6 - 103.5 132.1
    * Multiplicación 1.2 * 1.1 1.32
    / División 0.050 / 0.02
    7 / 2    		 0.25
    3
    % Resto División entera 20 % 7
    14.5 % 2    		 6
    0.5
    ** Exponenciación 2 ** 2 4

    El operador exponenciación sirve para realizar potencias o raíces. Para realizar raíces es necesario que el exponente de la potencia sea una fracción. Por ejemplo, para calcular la raíz cuadrada de 4: 2 ** (1/2)

    Concatenar cadenas¶

    El operador binario + tiene un tercer uso, concatenar cadenas. Une una o más cadenas obteniendo como resultado una nueva cadena completa.

    const name = 'Jane'
const lastName = 'Doe'
const fullName = name + ' ' + lastName

console.log(fullName) // Jane Doe

    Hay que tener en cuenta que cuando se usa el operador de concatenación con números, estaríamos realizando una suma. Sin embargo, si uno de los dos operandos es de tipo cadena, aunque el otro sea un número o un booleano, siempre concatenará, incluso si la cadena es de un dígito:

    console.log(1 + "1") //11
console.log("1" + 2) //12

    En Javascript, si hay más de un operando y el operador es binario se va realizando la operación de dos en dos, según un orden que veremos posteriormente. De esta manera:

    console.log(2 + 2 + "2") // 4 + "2" = "42"
console.log("2" + 2 + 4) // "22" + 4 = "224"

    Warning

    El operador binario + es el único operador binario que trata a las cadenas dígitos (las cadenas que son numéricas) como cadenas y no los tratan como número. El resto de operadores matemáticos realizan la conversión de la cadena a número, y en caso de no ser posible el resultado de la operación sería NaN.

    console.log(2 - '1') // 1
console.log('6' / '2') // 3
console.log('hola mundo' / 2) //NaN

    Operador Asignación¶

    El operador de asignación = es un operador binario que sirve para asignar a una variable un valor, por lo que los operandos sería la variable el valor.

    let a = 10

    En Javascript, se permite encadenar varias asignaciones, de forma que se evalúen de derecha a izquierda:

    let a, b, c

a = b = c = 4

    En el ejemplo anterior, se realiza una asignación encadenada, de forma que primero se realiza la que esté más a la derecha (c = 4) y luego las restantes (b = c y c = a).

    Aquí una demostración de una asignación encadenada más compleja:

    let a = 1;
let b = 2;

let c = 3 - (a = b + 1);

console.log(a); // 3
console.log(c); // 0

    En el ejemplo anterior, primero se evalúa la asignación a = b + 1 y su resultado de utiliza en la siguiente asignación c = 3 - 3, por lo que el valor de la variable c acaba siendo 0.

    Aunque es cierto que Javascript te permite encadenar las asignaciones, no es algo que se recomienda fervientemente, por lo que es más legible hacerlo secuencialmente. Así el ejemplo anterior, sería más legible de esta forma:

    let a = 1;
let b = 2;

a = b + 1;

let c = 3 - a;

console.log(a); // 3
console.log(c); // 0

    Operadores combinados¶

    En ciertas ocasiones queremos asignarle a una variable un nuevo valor que depende de su valor anterior, por ejemplo, podemos incrementar 2 unidades.

    let number = 2

number = number + 2

console.log(number) // 4

    En Javascript (como en otros lenguajes), existe la posibilidad de combinar la operación y asignación en un solo operando, por ejemplo +=. De esta forma, el ejemplo anterior quedaría:

    let number = 2

number += 2

console.log(number) // 4

    Primero se indicaría el operador matemático y luego el de asignación, de está forma se evalúa primero la expresión (number + 2) y luego se realiza la asignación (number = 4).

    También se puede hacer uso de más de una operación matemática, prevaleciendo ésta siempre primero antes de la asignación. De esta forma:

    let n = 2;

n *= 3 + 5;

console.log(n); // 16

    En el ejemplo anterior, primero se evalúa la operación de suma (3 + 5) y luego se realiza la operación de multiplicación (2 * 8), y por último la asignación (16).

    Operadores incremento/decremento¶

    Los operadores incrementales y decrementales sirven para modificar el valor de las variables en un unidad. El operador incremento ++ modificar el valor de la variable en una unidad de forma incremental, mientras que el de decremento -- lo decrementa en una unidad.

    let count = 2;

count++; // 3

count--; // 2

    Importante

    Incremento/decremento sólo puede ser aplicado a variables. Intentar utilizarlo en un valor como 5++ dará un error.

    Estos operadores puede ser colocados antes o después de una variable, teniendo un comportamiento diferente. Esta diferente es notable durante la evaluación de una expresión. Si se colocan como sufijo (antes de la variable), el valor se verá incrementado/decrementado después de usar la variable en la expresión evaluada, mientras que si es usado como prefijo (después de la variable), el valor se vera incrementado/decrementado antes de usar la variable en la expresión.

    let number = 1

console.log(number++) // 1

console.log(number) //

    En el ejemplo anterior, tenemos una variable con un valor inicial a 1. Al hacer el incremento como sufijo, primero se usa el valor de la variable antes de incrementarse para mostrarla por pantalla, por lo que se mostrará 1, y luego se incrementará a 2.

    Ahora veamos una demostración, con un incremento como prefijo:

    let number = 1

console.log(++number) //2
console.log(number)

    En esta ocasión, primero se incrementa el valor y luego se muestra por pantalla.

    Veamos un ejemplo más complejo:

    let counter = 1

let expr = 2 ** counter++ // 2

let other = 2 ** ++otherCounter //8

    El valor de la variable expr es dos, ya que utiliza un incremento de sufijo, por lo que primero se usa el valor anterior 1 para realizar la operación 2 ** 1 y luego se incrementa a 2. Mientras que, la variable other tiene resultado 8 al usar un incremento de prefijo, por lo que primero se ha incrementado el valor (de 2 a 3) y luego se ha usado éste para la operación 2 ** 3.

    Operadores a nivel de bits¶

    Los operados a nivel de bits son operadores que tratan a los números como números enteros de 32 bits, trabajando con su representación binaria. Este tipo de operadores no es exclusivo de Javascript, sino que también son utilizados en otros lenguajes.

    Estos operadores se usan de forma muy puntual, simplemente cuando es necesario el uso de la representación de los números a un bajo nivel, pero aún así es bueno tener conocimientos de ellos.

    Podemos encontrar los siguientes operadores:

    • AND (&). Devuelve un 1 en las posiciones de bit dónde las posiciones de los operadores tienen un 1, por el contrario devuelven un 0.

      let x = 9 // 1001
let y = 12 // 1100

console.log(x & y) // 8 (1000)

    • OR (|). Devuelve un cero en las posiciones de bit dónde las posiciones de los dos operadores tienen un 0, por el contrario devuelven un 1.

      let x = 9 // 1001
let y = 12 // 1100

console.log(x | y) // 13 (1101)

    • XOR (^). Devuelve un cero en las posiciones dónde el bit es el mismo y un 1 dónde las posiciones son diferentes.

      let x = 9 // 1001
let y = 12 // 1100

console.log(x ^ y) // 5 (0101)

    • NOT (~). Es un operador unario que intercambia el valor, es decir si la posición tiene un valor de cero es cambiado por uno y viceversa.

      let x = 9 // 1001

console.log(~x) // 5 (0110)

    • LEFT SHIFT (<<). Es un operador binaria cuyo segundo operado indica el número de bits que se desplazarán del primer operado hacia la izquierda. En otras palabras, se añade tantos 0 como se indica al final del binario.

      let x = 2 // 0000010

console.log(x << 1) // 4 (00000100)

    • RIGHT SHIFT (>>). Es un operador binario cuyo segundo operado indica el número de bits que se desplazarán del primer operado hacia la derecha. En otras palabras, desaparece la cantidad de binarios indicada.

      let x = 2 // 0000010

console.log(x >> 1) // 1 (000001)

    • ZERO-FILL RIGHT SHIFT (>>>). Es un operador binario cuyo segundo operado indica el número de bits que se desplazarán del primer operado hacia la derecha, pero añadiendo 0 a la parte izquierda. En otras palabras, desaparece la cantidad de binarios indicada.

      let x = 2 // 0000010

console.log(x >>> 1) // 1 (0000001)

    Operador coma¶

    El operador coma , es uno de los operadores más raros e inusuales. A veces, es utilizado para escribir código más corto, entonces tenemos que saberlo para poder entender qué está pasando.

    El operador coma nos permite evaluar varias expresiones, dividiéndolas con una coma ,. Cada una de ellas es evaluada, pero sólo el resultado de la última es devuelto.

    Por ejemplo:

    let a = (1 + 2, 3 + 4);

console.log(a); // 7 (el resultado de 3 + 4)

    Aquí, se evalúa la primera expresión 1 + 2 y se desecha su resultado. Luego, se evalúa 3 + 4 y se devuelve como resultado.

    Info

    Tenga en cuenta que el operador coma tiene una precedencia muy baja, inferior a =, por lo que los paréntesis son importantes en el ejemplo anterior.

    Sin ellos: a = 1 + 2, 3 + 4 se evalúa primero el +, sumando los números a a = 3, 7, luego el operador de asignación = asigna a = 3, y el resto es ignorado. Es igual que (a = 1 + 2), 3 + 4.

    A veces, las personas lo usan en construcciones más complejas para poner varias acciones en una línea.

    Por ejemplo:

    for (a = 1, b = 3, c = a * b; a < 10; a++) {
 ...
}

    Tales trucos se usan en muchos frameworks de JavaScript. Por eso los estamos mencionando. Pero generalmente no mejoran la legibilidad del código, por lo que debemos pensar bien antes de usarlos.

    Operadores relaciones¶

    Los operados relaciones realizan comparaciones entre los datos obteniendo siempre un resultado booleano (true o false). Entre ellos, podemos encontrar:

    Operador Significado Ejemplo
    < Menor que 3 < 5
    > Mayor que 3 > 5
    <= Menor o igual que 3 <= 5
    >= Mayor o igual que 3 >= 5
    == Igual que 3 == 5
    != Distinto que 3 != 5

    Se pueden comparar números, o cadenas, siendo el algoritmo de comparación el siguiente:

    1. Compare el primer carácter de ambas cadenas.
    2. Si el primer carácter de la primera cadena es mayor (o menor) que el de la otra cadena, entonces la primera cadena es mayor (o menor) que la segunda. Hemos terminado.
    3. De lo contrario, si los primeros caracteres de ambas cadenas son los mismos, compare los segundos caracteres de la misma manera.
    4. Repita hasta el final de cada cadena.
    5. Si ambas cadenas tienen la misma longitud, entonces son iguales. De lo contrario, la cadena más larga es mayor.

    Por ejemplo, al comparar Z > A, solo es necesario hacer el primer paso. Sin embargo, al comparar Glow > Glee, es necesario más pasos:

    1. G es igual a G.
    2. l es igual a l.
    3. o es mayor que e, por lo que la primera cadena es mayor.

    ¡OJO!

    El algoritmo de comparación dado arriba es aproximadamente equivalente al utilizado de forma alfabética, pero no es exactamente el mismo.

    Por ejemplo, las mayúsculas importan. Una letra mayúscula A no es igual a la minúscula a. La a minúscula es mayor porque el carácter en minúscula tiene un índice mayor en la tabla de codificación interna que utiliza Javascript (Unicode)

    Para comparar entre distintos tipos, Javascript intenta realizar una parseo a números, en caso de que sea posible, si no lo transforma en NaN. Un NaN nunca es ni mayor, ni menor, ni igual a un número, por lo que una comparación con dicho tipo numérico daría siempre false. Un booleano se puede entender como un número binario, siendo true equivalente a 1y false a 0:

    console.log('3' > 4) // false
console.log('Hola' == 2) // false
console.log('Hola' != 2) // true
console.log(true > 0) // true

    Warning

    En Javascript el número 0 se puede entender en booleano como false, pero la cadena "0" se entiende como true, por lo que a la hora de compararlas hay que tenerlo en cuenta.

    Veamos otro ejemplo:

    console.log(0 == false) //true
console.log('' == false) // true

    Ambas comparaciones son true porque al pasar los datos no numéricos a números, se entienden como 0. Esto puede suponer un problema, porque a veces es conveniente comprobar si el contenido de dos variables es exactamente igual, y en este caso siempre daría true. Para ello hay un comparador de igualdad estricto que retornará true si y solo si los datos comparados son iguales en tipo y contenido:

    console.log(3 === '3') // false
console.log(0 === false) // false

    En los ejemplos anteriores, vemos que los datos son iguales en contenido (recuerda que false equivale a 0), pero no tienen el mismo tipo de dato, por lo que no serían iguales estrictos. Existe, por ende, el comparador de desigualdad estricto !==.

    El uso del operador de igual estricto es un mejor práctica que el uso del operador de igual básico, ya que el estricto es propenso a tener menos errores. Uno de los errores típicos es que al comparar null y undefined con el operador == éste devuelve true, pero con el operador === devuelve false.

    Warning

    Al hacer una comparación de cualquier tipo de dato con null o undefined ésta dará false, ya que su valor "numérico" es NaN:

    console.log(3 < null) // false
console.log(undefined > 5) // false

    Solo puede ser true si su valor es undefined o null y se compara de igualdad:

    let num

console.log(num === undefined)

    Operadores lógicos¶

    Los operadores lógicos son operadores binarios aplicados a cualquier tipo y que pueden devolver cualquier tipo:

    Operador OR (||)¶

    El operador OR (||) es un operador que en la programación clásica, retornará true si cualquiera de sus dos operandos es true:

    console.log(true || false) // true
console.log(false || true) // true
console.log(true || true) // true
console.log(false || false) // false

    En el ejemplo anterior, solo retorna false cuando se compara dos false. Si se intenta utilizar como operador otro tipo de dato, se parseará a booleano. En el ámbito de los números se entiende que 0 es false y que 1 es true. En el caso de las cadenas, se entiende como false una cadena vacía, y como true una no vacía. También se puede entender como true una variable distinta de null y undefined.

    console.log(1 || 0) // true
console.log('Hola' || '') // true

    Además de la funcionalidad clásica, en Javascript dicho operador, tiene una función extra. Se puede usar dicho operador para la asignación de una variable, de la siguiente forma: const result = value1 || value2 || value3. Su funcionamiento es simple:

    • Evalúa la expresión de izquierda a derecha.
    • Transforma cada operando en un valor booleano. Si el resultado de alguno es true, se detiene en el primer operando que da true, retornando el valor original del operando.
    • Si todos los operandos son false retorna el último operando.

    En otras palabras, una cadena de OR devuelve el primer valor verdadero o el último si ningún verdadero es encontrado.

    const result1 = 0 || '' || 7 // 7
const result2 = 3 || undefined || false // 3
const result1 = null || 'Hola' || 9 // 'Hola'
const result4 = null || undefined || '' || 0 // 0

    Uno de los usos más comunes de esta funcionalidad es poder definir una variable que depende de otros valores, pero en caso de los otros valores no estén definidos (ya sea porque sean null o undefined o cadena vacía, o incluso cero) podemos indicarle un valor por defecto.

    let name = ''
let firstName = null

const userName = name || firstName || 'Nombre de usuario'

    En el ejemplo anterior se evalúa las dos variables, en caso de que den false, se utiliza como valor de la variable userName la cadena Nombre de usuario.

    Operador AND (&&)¶

    El operador AND (&&) es un operador que en la programación clásica retornará true siempre y cuando todos los operandos devuelvan true y false si hay uno o más de uno que devuelve false.

    console.log(true && true); // true
console.log(false && true); // false
console.log(true && false); // false
console.log(false && false); // false

    Al igual que operador OR, en Javascript, se puede usar el operador AND para la asignación const result = value1 && value2 && value3. En este caso, el procedimiento es un ligeramente diferente:

    • Evalúa la expresión de izquierda a derecha.
    • Transforma cada operando en un valor booleano. Si el resultado de alguno es false, se detiene en el primer operando que da false, retornando el valor original del operando.
    • Si todos los operandos son true retorna el último operando.

    En otras palabras, AND retorna el primer valor falso o el último valor si ninguno fue encontrado. Las reglas anteriores son similares a las de OR. La diferencia es que AND retorna el primer valor falso mientras que OR retorna el primer valor verdadero.

    const result1 = 0 && '' && 7 // 0
const result2 = 3 && undefined && false // undefined
const result1 = null && 'Hola' && 9 // null
const result4 = 1 && 2 && true && 3 // 3

    Operador NOT (!)¶

    El operador NOT (!) es un operador unario que se encarga de convertir dicho operando en un booleano (en caso de que no lo sea) y devuelve su valor booleano contrario.

    console.log(!false) // true
console.log(!2) // false
console.log(!0) // true
console.log(!'Hola') // false
console.log(!'') // true
console.log(!undefined) // true

    Warning

    En Javascript los operadores AND, OR, NOT y NULLISH COALESCING son operadores corto-circuitos, es decir, van evaluando expresiones hasta que una de ellas obtiene un valor claro, dejando a las demás sin evaluar. Por ejemplo:

    const result = 10 || 0 || false

    En este ejemplo, el valor de la variable result es 10, pero el tiempo de ejecución es menor si se hubiese cambiado el orden, ya que al evaluar primero el 10, como se obtiene lo que se desea (una valor distinto de false) el resto de expresiones se dejan sin evaluar. Por eso, es importante el orden en el que se evalúan dichas expresiones. Veamos otro ejemplo:

    console.log(3 > 5 && 4 > 2)

    Ahora, se mostrará false, porque 3>5 retorna false por lo que no es necesario evaluar la siguiente expresión, ya que siendo la primera false el operando && retornará false sí o sí.

    Operador typeof¶

    El operador typeof es un operador unario de Javascript que devuelve el tipo de dato del operando. También se puede utilizar de la siguiente forma typeof(x) pero hay que tener en cuenta que typeof es un operando y no una función.

    console.log(typeof 'Hola') // string
console.log(typeof 4) // number

    Operador ternario¶

    El operador ternario es un operador con tres operando. El primer operando hace referencia a una condición que debe ser evaluada. Si esta condición es verdadera el operador retorna el segundo operando, mientras que si la condición es falsa el operador retorna el tercer operando. Su sintaxis es la siguiente:

    condition ? trueValue : falseValue

    Veamos un ejemplo:

    const result = age > 18 ? 'Mayor' : 'Menor'

    Podemos tener una secuencia de operadores ternario si queremos evaluar más de una posibilidad:

    let message = age < 3 
        ? '¡Hola, bebé!' 
        : age < 18 
            ? '¡Hola!' 
            : age < 100 
                ? '¡Felicidades!' 
                : '¡Qué edad tan inusual!';

console.log(message)

    La ejecución del ejemplo anterior es la siguiente.

    1. Se comprueba si la expresión age < 3 es verdadera. En caso de serlo, retornará el valor 'Hola, bebé', y no comprobará el resto de condiciones.
    2. Si es falsa, comprobará la siguiente condición age < 18. En este caso si es verdadera retornará '¡Hola!'.
    3. Si es falsa, comprobará la última condición, que de ser verdadera retornará ¡Felicidades!, por el contrario ¡Qué edad tan inusual!.

    Operador Nullish Coalescing¶

    El operador nullish coalescing (fusión de null) es un operador binario en el que retorna el primer operando si es distinto de null o undefined, o por el contario retorna el segundo:

    let number

console.log(number ?? 0) // 0

    En el ejemplo anterior, se mostrará 0, ya que la variable number es una variable undefined y por lo tanto la expresión number ?? 0 retorna el segundo operado (0). En el siguiente ejemplo, retornará el valor de la variable number ya que no es undefined:

    let number = 10

console.log(number ?? 0) // 10

    El operador nullish coalescing no es más que una sintaxis reducida del uso de un operador ternario con dicho propósito:

    let number

console.log(number !== undefined || number !== null ?  null : 0)

    También se puede usar una secuencia de dicho operador para seleccionar el primer valor que no sea null/undefined:

    let firstName = null;
let lastName = null;
let nickName = "MyNick";

console.log(firstName ?? lastName ?? nickName ?? "Anonymous"); // MyNick

    El ejemplo anterior, también podía haberse realizado con el operador OR (||). Históricamente, el operador OR estuvo primero y existe desde el origen de JavaScript, así que los desarrolladores lo estuvieron usando para tal propósito durante mucho tiempo.

    Por otro lado, el operador nullish coalescing es una adición reciente, y la razón es que la gente no estaba del todo satisfecha con ||.

    La gran diferencia es que OR devuelve el primer valor verdadero y nullish coalescing devuelve el primer valor definido. El || no distingue entre false, 0, un string vacío "", y null/undefined. Todos son lo mismo: valores falsos. Si cualquiera de ellos es el primer argumento de ||, obtendremos el segundo argumento como resultado. Pero cierto es, que en la práctica querremos usar dicho valor solo cuando no esté definido o sea null. En el ejemplo siguiente, me interesa seguir usando el valor 0, por eso no es conveniente usar el operador OR:

    const height = 0

console.log('La altura es', height ?? 'No hay altura')

    Si hubiésemos usado el operador OR, hubiese devuelto la cadena, pero eso no sería coherente ya que el valor de una altura si puede ser 0.

    Tips

    Por este motivo, se recomienda usar siempre el operador nullish coalescing en lugar del operador OR, y dejar dicho operador solo en situaciones concretas.

    Warning

    Por motivos de seguridad, JavaScript prohíbe el uso de ?? junto con los operadores && y ||, salvo que la precedencia sea explícitamente especificada con paréntesis.

    let x = 1 && 2 ?? 3; // ❌ Syntax error
let y = (1 && 2) ?? 3; // ✅ 2

    Precedencias¶

    En Javascript, como en cualquier otro lenguaje, existen demasiados operadores y a veces al usar varios de ellos a la vez es necesario tener en cuenta un precedencia a la hora de ejecutarse. Por ejemplo, en la expresión 3 + 5 * 2, aunque primero vaya la suma, según la precedencia de operaciones, la multiplicación tiene mayor precedencia que la suma por lo que primero se ejecutará 5 * 2 y luego el resultado se sumará a 3.     La asociatividad hace referencia a la situación en la cual dos operandos tenga la misma precedencia. En ese caso, se ejecutará primero según la asociatividad. Por ejemplo, en el caso de 3 * 3 / 3, la multiplicación y la división tiene la misma precedencia, y como su asociatividad es de izquierda, primero se ejecuta la operación que esté más a la izquierda, en este caso la multiplicación.

    A veces, en ocasiones queremos realizar una operación de baja precedencia antes que una de mayor procedencia, para ello se puede usar el operador con mayor precedencia los paréntesis (). Por ejemplo, si en la operación 2 + 3 * 5, donde la multiplicación tiene mayor precedencia que la suma, pero se desea hacer la suma antes, se debe colocar un paréntesis en la expresión que se desea hacer primero: (2 + 3) * 5.
