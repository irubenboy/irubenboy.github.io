    5 Grupos¶

    Introducción¶

    Un grupo de captura es una parte de una expresión regular que se desea obtener. Para indicar dentro de un expresión que un parte pertenece a un grupo de captura, se envuelve entre paréntesis (...)

    Podemos usar, después del paréntesis de captura, un cuantificador para cuantificar el número de veces que debe aparecer el grupo:

    console.log('Gogogo now!'.match(/(go)+/ig)) //[go, go go]

    En cambio, si se usa como grupo de captura podemos obtener una matriz independiente con los valores capturados. Supongamos que tenemos una cadena y queremos comprobar si es o no un email, y en caso de serlo obtener el nombre de dominio. Un email se compone de un nombre y un dominio unido por el carácter @ teniendo una estructura así name@domain. Como se desea obtener el dominio, esa parte de la expresión regular debe de estar envuelta entre paréntesis.

    Grupos con match¶

    Para obtener los grupos de las coincidencias encontradas haciendo uso de la función match(regex) de las cadenas, si no tiene el indicador g, se accederá a la posición del grupo, que están enumeradas de izquierda a derecha. De esta manera, para el primer grupo se accederá al índice 1, al grupo 2 el índice 2, etc. Al acceder al índice 0, se obtiene la coincidencia completa.

    Veamos un ejemplo, donde se obtiene la primera coincidencia de las etiquetas HTML:

    const str = "<h1>Hello world!</h1>"
const tags = str.match(/<(.*?)>)

console.log(tags[0]) // <h1>
console.log(tags[1]) // h1

    También existe la posibilidad de anidar grupos. De esta manera, el acceso varía un poco, siguiendo la numeración de izquierda a derecha, pero si alguno tiene algún subgrupo se cuenta primero los de adentro.

    Supongamos, ahora, que queremos obtener la etiqueta, el atributo con su valor. Para ello, usamos la siguiente expresión <(([a-z][a-z\d]*)\s*(([a-z]*)=\s*("[\w\s]*")))>. Al aplicárselo a la etiqueta <span class="my">:

    • el primer grupo corresponde a la etiqueta completa: span class="my"
    • el segundo grupo corresponde al nombre de la etiqueta: span
    • el tercer grupo corresponde al atributo y su valor: class="my"
    • el cuarto grupo corresponde al nombre del atributo: class
    • el quinto grupo corresponde al valor del atributo: "my"
    const regex = /<(([a-z][a-z\d]*)\s*(([a-z]*)=\s*("[\w\s]*")))>/
const str = `<span class="my">This is a span</span>`
const result = str.match(regex)

console.log(result[0]) //<span class="my">
console.log(result[1]) // span class="my"
console.log(result[2]) // span
console.log(result[3]) // class="my"
console.log(result[4]) // class
console.log(result[5]) // "my"

    Grupos opcionales¶

    A veces, usamos un grupo que puede aparecer una vez o ninguna, en ese caso, estamos tratando con un grupo opcional. Si el grupo no se encuentra, pero se ha encontrado coincidencia, el array contendrá en la posición del grupo específico un valor undefined:

    let match = 'a'.match(/a(z)?(c)?/);

console.log(match.length) // 3
console.log(match[0]) // a (coincidencia completa)
console.log(match[1]) // undefined
console.log(match[2]) // undefined

    La longitud sigue siendo tres, pero el array se compone de dos posiciones con valor undefined.

    Grupos en todas las coincidencias¶

    Al usar el indicador g el array devuelto, no nos devuelve los valores de los grupos que se desea obtener.

    const str = `<h1>Title</h1> <h2>Subtitle</h2>`

const tags = str.match(/<(.*?)>/g)

console.log(tags) // [<h1>, <h2>]

    Para poder obtener el valor de los grupos, se puede hacer uso del método matchAll(regex). Al igual que match(regex) busca coincidencias pero tiene algunas diferencias:

    • No devuelve un array, si no un objeto iterable
    • Cuando está presente el indicador g, devuelve todas coincidencias como un array de grupos.
    • Si no hay coincidencias, NO devuelve null, sino un objecto iterable vacío.
    const results = '<h1> <h2>'.matchAll(/<(.*?)>/gi)

// results - no es un array, sino un objeto iterable
console.log(results) // [object RegExp String Iterator]


results = Array.from(results) // lo convirtamos en array

console.log(results[0]) // <h1>,h1 (1er etiqueta)
console.log(results[1]) // <h2>,h2 (2da etiqueta)

    Cada coincidencia del matchAll(regex) tiene la misma estructura devuelta por un match(regex) sin el flag g:

    const results = '<h1> <h2>'.matchAll(/<(.*?)>/gi)

const [tag1, tag2] = results

console.log(tag1[0]) // <h1>
console.log(tag1[1]) // h1
console.log(tag1.index) // 0
console.log(tag1.input) // <h1> <h2>

    Grupos con nombre¶

    Cuando un patrón es simple, es factible enumerar los grupos. Sin embargo, cuando tenemos una expresión compleja y se desea recordar el número del grupo al que acceder, es difícil de recordar. Para ello, podemos nombrar a los grupos y acceder a ellos con mayor facilidad.

    Para nombrar un grupo, se debe indicar ?<name> después del paréntesis de apertura. Para acceder a un grupo por su nombre, debemos acceder al objeto groups del resultado:

    const dateRegexp = /(?<year>[0-9]{4})-(?<month>[0-9]{2})-(?<day>[0-9]{2})/
const str = "2019-04-30"

const groups = str.match(dateRegexp).groups

console.log(groups.year) // 2019
console.log(groups.month) // 04
console.log(groups.day) // 30

    También, se pueden usar en matchAll(regex):

    const dateRegexp = /(?<year>[0-9]{4})-(?<month>[0-9]{2})-(?<day>[0-9]{2})/g

const str = "2019-10-30 2020-01-01"

const results = str.matchAll(dateRegexp)

for(let result of results) {
  let {year, month, day} = result.groups

  console.log(`${day}.${month}.${year}`)
}

    Grupos en reemplazo¶

    Como hemos visto anteriormente, el método replace(regex, replacement) reemplaza todas las coincidencias por el valor indicado. Pero, supongamos que tenemos una cadena y lo que deseamos es cambiar el orden, gracias a $n podemos acceder al número de grupo y reescribir la cadena en otro orden:

    const str = "John Bull";
const regexp = /(\w+) (\w+)/;

console.log(str.replace(regexp, '$2, $1')) // Bull, John

    También se puede hacer con parámetros nombrados, haciendo uso de $<name>:

    const regexp = /(?<year>[0-9]{4})-(?<month>[0-9]{2})-(?<day>[0-9]{2})/g;

const str = "2019-10-30, 2020-01-01";

console.log(str.replace(regexp, '$<day>.$<month>.$<year>'));
// 30.10.2019, 01.01.2020

    Grupos no capturados¶

    Podemos no capturar un grupo, cuando queremos cuantificar un grupo pero no queremos obtener un resultado. Para excluir un grupo, se debe usar ?: después del paréntesis de apertura:

    const str = "Gogogo John!"

// ?: excluye 'go' de la captura
const regexp = /(?:go)+ (\w+)/i

const result = str.match(regexp)

console.log(result[0]) // Gogogo John (coincidencia completa)
console.log(result[1]) // John
console.log(result.length) // 2 (no hay más ítems en el array)

    Referencias inversas¶

    Supongamos que se desea encontrar una cadena que esté envuelta entre comillas, ya sea simple o doble, con la única condición de que la de comienzo y la de final deben ser la misma. El patrón sería algo tal que así ['"](.*?)['"]. Sin embargo, si tenemos usamos un tipo de comillas dentro de otras, esto puede dar coincidencias incorrectas:

    const str = `He said: "She's the one!".`
const regex = /['"](.*?)['"]/g

console.log(str.match(regex)) // "She'

    Como se puede observar, devuelve "She', que para nada es lo que deseamos, ya que no se cumple la condición de que la cadena obtenida debe comenzar con el mismo tipo de comillas.

    Para solucionar dicho problema, debemos agrupar ['"] y hacer referencia a él al final de la expresión. De esta forma, se entenderá la referencia como el mismo valor obtenido en el grupo referenciado. Para referenciar un grupo usamos la barra invertida seguida del número de grupo a referenciar.

    const str = `He said: "She's the one!".`
const regex = /(['"])(.*?)\1/g

console.log(str.match(regex)) // "She's the one"

    Ahora, el problema esta resuelto y obtenemos el valor que realmente deseamos. También se puede hacer referencia a los grupos nombrados, \k<name>.

    const str = `He said: "She's the one!".`
const regexp = /(?<quote>['"])(.*?)\k<quote>/g

console.log(str.match(regexp)) // "She's the one!"

    Warning

    Si usamos un grupo opcional (?:) no podemos hacer referencia a él dentro de una expresión regular, ya que no son memorizados por el motor.
