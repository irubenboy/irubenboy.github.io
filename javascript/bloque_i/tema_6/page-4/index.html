    4 Mapas y colecciones¶

    Introducción¶

    Hasta ahora hemos visto diferentes tipos de colecciones, como los objetos, los arrays o los iterados. Además de estas colecciones existen otro tipos de colecciones, como los mapas y los conjuntos.

    Además de estas colecciones, existen métodos para tratar a los objetos como iteradores. La clase de utilidad Object dispone de varios métodos estáticos para ellos:

    • Object.keys(obj). Devuelve un array con las claves del objeto.
    • Object.values(obj). Devuelve un array con los valores del objeto.
    • Object.entries(obj). Devuelve una matriz, es decir, un array de arrays, donde cada sub-array, es un elemento [clave, valor].
    const obj = {
    name: "John",
    age: 25
}


for(const key of Object.keys(obj)){
    console.log(key)
}

for(const value of Object.values(obj)){
    console.log(value)
}

for(const entry of Object.entries(obj)){
    console.log(obj[0], obj[1])
}

    Warning

    Al igual que un bucle for..in, estos métodos ignoran propiedades que utilizan Symbol(...) como nombre de propiedades.

    Normalmente, esto es conveniente. Pero si también queremos propiedades simbólicas, entonces hay un método aparte Object.getOwnPropertySymbols que devuelve un array de únicamente propiedades simbólicas. También existe un método Reflect.ownKeys(obj) que devuelve todas las propiedades.

    Al igual que tenemos el Object.entries(obj) tenemos el método Object.fromEntries(array) que retorna un objeto con el primer valor de la entry como clave, y el segundo como valor.

    Los objetos no puede utilizar por defecto, los métodos map, o filter, cosa que a veces complica un poco la lógica de programación. Sin embargo, gracias a los métodos vistos, podemos utilizarlo sin problemas. Para ello, en primer lugar, debemos transforma el objeto en un array de clave-valor con el método Object.entries(obj). Posteriormente, usar sobre dicho array, los métodos map, filter o el que se necesite. Por último, volver a tener un objeto con el resultado de dichos métodos con el objeto Object.fromEntries(entries). Este último paso solo es necesario si lo que se desea obtener es un objeto, y se han usado métodos que devuelvan un array. Veamos un ejemplo:

    const prices = {
    banana: 1,
    orange: 2,
    meat: 4
}

const entries = Object.entries(prices) // [["banana", 1], ["orange", 2], ["meat", 4]]
                    .map(entry => [entry[0], entry[1] * 2]) // [["banana", 2], ["orange", 4], ["meat", 8]]

const newObject = Object.fromEntries(entries)

console.log(newObject) // {banana: 2, orange: 4, meat: 8}

    Mapas¶

    Un mapa es un tipo de colección que almacena datos con la estructura clave:valor, al igual que los objetos. Sin embargo, la principal diferencia emana en que los mapas permiten que la clave sea cualquier tipo de dato, a diferencia de los objetos que siempre eran cadenas.

    Para crear un mapa se puede usar el constructor Map(). Al constructor se le puede pasar un array o iterador para inicializar el mapa.

    const empty = new Map()

const map = new Map([
    [1, "one"],
    [2, "two"],
    [3, "three"]
])

    Los mapas tienen métodos que nos permiten trabajar con ellos:

    • set(key, value). Añade la clave y el valor al mapa. Si la clave ya existe en el mapa, sobrescribe su valor.
    • get(key). Obtiene el valor dada una clave.
    • has(key). Comprueba si el mapa contiene la clave dada.
    • delete(key). Elimina tanto el valor como la clave dada.
    • clear(). Vacía el mapa.
    • size. Obtiene la cantidad de elementos (clave-valor) que posee el mapa
    const map = new Map()

map.set('1', 'str1')
map.set(1, 'num1')
map.set(true, 'bool1')

console.log(map.get(1))
console.log(map.get('1'))
console.log(map.size)

    Como se puede observar en el ejemplo, cualquier tipo de dato puede funcionar como clave de un mapa. En los objetos, las claves siempre eran cadenas, si se indicaban otro tipo de datos, estas se pasaban a cadena. En los mapas, incluso, se puede indicar un objeto como clave:

    const obj = {name: 'John'}

const map = new Map()

map.set(obj, 123)

console.log(map.get(obj)) // 123

    Si se intentase esto con un objeto, el objeto usado como clave pasaría a ser una cadena, lo que daría problemas si tenemos más de un objeto:

    const john = { name: "John" }
const ben = { name: "Ben" }

const visitsCountObj = {}

visitsCountObj[ben] = 234
visitsCountObj[john] = 123

console.log(visitsCountObj["[object Object]"]) // 123

    Al hacer uso del método set(key, value) para indicar propiedades, se pueden concatenar varios set(key, value), ya que este método retorna el mismo mapa:

    const map = new Map()

map.set('key1', 'value1')
    .set('key2', 'value2')
    .set('key3', 'value3')

    Podemos recorrer un mapa, a través de los siguientes métodos:

    • keys(). Retorna un iterador con las claves del mapa.
    • values(). Retorna un iterador con los valores del mapa.
    • entries(). Retorna un iterador con la clave y el valor como un array, la primera posición,0, representa la clave, y la segunda posición, 1, representa el valor.

    Son métodos parecidos a los que se usan con los objetos, siendo la principal diferencia que los métodos de mapas devuelven iteradores, mientras que los de objectos devuelve un array.

    Veamos un ejemplo:

    const recipeMap = new Map([
  ['pepino', 500],
  ['tomates', 350],
  ['cebollas',    50]
])

for (const vegetable of recipeMap.keys()) {
  console.log(vegetable) // pepino, tomates, cebollas
}

for (const amount of recipeMap.values()) {
  console.log(amount) // 500, 350, 50
}

for (const entry of recipeMap) { // lo mismo que recipeMap.entries()
  console.log(entry[0], entry[1]) // pepino,500 (etc)
}

    A diferente de los objetos, a la hora de iterar un Mapa, éste conserva el orden de interacción.

    Otra forma de iterar un mapa, es a través del método forEach(callback) que recibe una callback que se le aplicará a cada uno de los elementos (clave-valor) del mapa. Esta callback recibe tres argumento, el valor del elemento en cuestión, su clave, y el mapa en sí. Veamos un ejemplo:

    const recipeMap = new Map([
  ['pepino', 500],
  ['tomates', 350],
  ['cebollas',    50]
])

recipeMap.forEach((value, key, map) => {
    console.log(`${key}: ${value}`)
})

    Podemos crear un mapa a través de un objecto, usando el método Object.entries(entries) en el constructor del mapa. Además, de la misma forma, haciendo uso del método Object.fromEntries(entries) podemos crear un objeto a partir de un mapa, usando como entries el método entries() de los mapas.

    const obj = {
    name: "John",
    age: 30
}

const map = new Map(Object.entries(obj))

console.log(map.get('name')) // John

const newObj = Object.fromEntries(map.entries())

console.log(newObj) // {name: "John", age: 30}

    Es importante tener en cuenta que todas las claves del mapa, sean del tipo que sean, se pasarán a cadena para que funcionen como clave del objeto.

    Mapas débiles¶

    Como vimos ya vimos, el motor de JavaScript mantiene un valor en la memoria mientras sea accesible y pueda ser potencialmente usado. Normalmente, los elementos de una estructura, como las propiedades de un objeto o los elementos de un array, permanecen en memoria mientras dicha estructura esté en uso.

    Por ejemplo, si agregamos un objeto a un array, mientras el array permanezca en memoria, el objeto también estará en memoria, incluso si no hay otras referencias a él.

    Del mismo modo, si usamos un objeto como la clave en un Map regular, entonces mientras exista el Map, ese objeto también existe. Este objeto ocupa memoria y no puede ser reclamado por el recolector de basura.

    WeakMap es una estructura de datos similar a Map, pero no impide la recolección de basura de objetos usados como claves.

    La primera diferencia con Map es que en WeakMap las claves deben ser objetos, no valores primitivos:

    const weakMap = new WeakMap()

const obj = {}

weakMap.set(obj, "ok") // (1)!

weakMap.set("test", "Whoops") // (2)!
    1. ✅ La clave es un objeto, por lo que no hay problema.
    2. ❌ La clave es una cadena, aquí da error.

    Ahora, si usamos un objeto como clave y no hay otras referencias a ese objeto, se eliminará de la memoria (y del map) automáticamente.

    const john = { name: "John" }

const weakMap = new WeakMap()
weakMap.set(john, "...")

john = null

// ¡John se eliminó de la memoria!

    Ahora, si john solo existe como la clave de WeakMap, se eliminará automáticamente del map (y de la memoria).

    WeakMap no admite la iteración ni los métodos keys(), values(), entries(), así que no hay forma de obtener todas las claves o valores de él.

    WeakMap tiene solo los siguientes métodos: set(key, value), get(key), delete(key), has(key).

    Dicha limitación es debida por razones técnicas. Si un objeto ha perdido todas las demás referencias (como john en el código anterior), entonces se debe recolectar automáticamente como basura. Pero técnicamente no se especifica exactamente cuándo se realiza la limpieza.

    El motor de JavaScript decide eso. Puede optar por realizar la limpieza de la memoria inmediatamente o esperar y realizar la limpieza más tarde cuando ocurran más eliminaciones. Por lo tanto, técnicamente no se conoce el recuento actual de elementos de un WeakMap. El motor puede haberlo limpiado o no, o lo hizo parcialmente. Por esa razón, los métodos que acceden a todas las claves/valores no son soportados.

    El área principal de aplicación de WeakMap es como almacenamiento de datos adicional.

    Si estamos trabajando con un objeto que pertenece a otro código (tal vez incluso una biblioteca de terceros), y queremos almacenar algunos datos asociados a él que solo deberían existir mientras el objeto esté vivo, entonces WeakMap es exactamente lo que se necesita.

    Ponemos los datos en un WeakMap utilizando el objeto como clave, y cuando el objeto sea recolectado por el recolector de basura, esos datos también desaparecerán automáticamente: weakMap.set(john, "secrets documents"), de esta forma si john desaparece, secrets documents también lo hará.

    Por ejemplo, tenemos un código que mantiene un recuento de visitas para los usuarios. La información se almacena en un map: un objeto de usuario es la clave y el recuento de visitas es el valor. Cuando un usuario se va (su objeto será recolectado por el recolector de basura), ya no queremos almacenar su recuento de visitas.

    const visitsCountMap = new Map() // map: user => visits count

function countUser(user) {
  const count = visitsCountMap.get(user) || 0;
  visitsCountMap.set(user, count + 1)
}

const john = { name: "John" }

countUser(john)

john = null

    Ahora el objeto john debería ser recolectado como basura, pero permanece en la memoria, ya que es una propiedad en visitCountMap.

    Necesitamos limpiar visitCountMap cuando eliminamos usuarios, de lo contrario, crecerá en la memoria indefinidamente. Tal limpieza puede convertirse en una tarea tediosa en arquitecturas complejas.

    Lo podemos evitar cambiando a WeakMap en su lugar:

    const visitsCountMap = new WeakMap() //  user => visits count

function countUser(user) {
  const count = visitsCountMap.get(user) || 0
  visitsCountMap.set(user, count + 1)
}

const john = { name: "John" }

countUser(john)

john = null

    Ahora no tenemos que limpiar visitasCountMap. Después de que el objeto john se vuelve inalcanzable por todos los medios excepto como una propiedad de WeakMap, se elimina de la memoria junto con la información asociada a esa clave de WeakMap.

    Otro ejemplo común es el almacenamiento en caché cuando se debe recordar el resultado de una función (en caché), para que las llamadas futuras en el mismo objeto lo reutilicen. Podemos usar Map para almacenar resultados:

    const cache = new Map()

function process(obj) {
  if (!cache.has(obj)) {
    let result = obj

    cache.set(obj, result)
    return result
  }

  return cache.get(obj)
}


let obj = {/* digamos que tenemos un objeto */};

let result1 = process(obj); // calculado

// ...después, en otro lugar del código...
let result2 = process(obj); // resultado recordado tomado de la memoria caché

// ...después, cuando el objeto no se necesita más:
obj = null;

console.log(cache.size); // 1

    Para múltiples llamadas de proceso (obj) con el mismo objeto, solo calcula el resultado la primera vez, y luego lo toma de caché. La desventaja es que necesitamos limpiar el caché cuando el objeto ya no es necesario.

    Si reemplazamos Map por WeakMap, este problema desaparece: el resultado en caché se eliminará de la memoria automáticamente después de que el objeto se recolecte.

    const cache = new WeakMap()

function process(obj) {
  if (!cache.has(obj)) {
    const result = obj

    cache.set(obj, result)
    return result
  }

  return cache.get(obj)
}

const obj = {/* algún objeto */}

const result1 = process(obj)
const result2 = process(obj)

// ...después, cuando el objeto no se necesitará más:
obj = null

// No se puede obtener cache.size, ya que es un WeakMap,
// pero es 0 o pronto será 0
// Cuando obj se recolecte como basura, los datos en caché también se eliminarán

    Conjuntos¶

    Un conjunto es una colección especial sin claves, donde cada valor es único dentro de la colección. Se puede considerar como una especia de array pero sin elementos repetidos, solamente con elementos únicos. Para crear un conjunto se puede utilizar su constructor Set([iterable]) que puede recibir un iterable para inicializarlo. Si recibe un iterable con valores repetidos, estos son omitidos por el conjunto:

    const set = new Set([1,2,3,1,2,3])

console.log(set) // [1,2,3]

    Sus métodos más destacados son:

    • add(value). Añade el valor (si no existe en el conjunto), y devuelve el conjunto en sí.
    • delete(value). Elimina el valor, y devuelve true si el valor existía al momento de la llamada; si no, devuelve false.
    • has(value). Devuelve true si el valor existe en el conjunto, si no, devuelve false.
    • clear(). Elimina todo el contenido del conjunto
    • size. Devuelve la cantidad de elementos del conjunto.

    La característica principal es que llamadas repetidas de set.add(value) con el mismo valor no hacen nada. Esa es la razón por la cual cada valor aparece en Set solo una vez.

    const set = new Set();

const john = { name: "John" }
const pete = { name: "Pete" }
const mary = { name: "Mary" }

set.add(john)
set.add(pete)
set.add(mary)
set.add(john)
set.add(mary)

console.log(set.size) // 3

    La alternativa a Set podría ser un array de usuarios, y código para verificar si hay duplicados en cada inserción usando el método find(callback). Pero el rendimiento sería mucho peor, porque este método recorre el array completo comprobando cada elemento. Set está optimizado internamente para verificar unicidad.

    Podemos recorrer un conjunto con el bucle for..of como cualquier iterable, o haciendo uso del método forEach(callback), donde recorrerá cada elemento del conjunto aplicándole el callback:

    const set = new Set(["oranges", "apples", "bananas"])

for (let value of set) console.log(value)

set.forEach((value, valueAgain, set) => {
  console.log(value)
})

    Tenga en cuenta que la función callback pasada en forEach tiene 3 argumentos: un valor, luego el mismo valor valueAgain y luego el objeto de destino que es set. El mismo valor aparece en los argumentos dos veces. Eso es por compatibilidad con Map donde la función callback tiene tres argumentos. Aunque parezca extraño, en ciertos casos puede ayudar a reemplazar Map con Set y viceversa con facilidad.

    Conjuntos débiles¶

    WeakSet es una clase análoga a Set, pero en esta estructura solo podemos agregar objetos y no tipos primitivos. El objeto, al igual que en WeakMap, solo existirá en la colección mientras sea accesible desde otro lugar. Al igual que Set, admite add(value), has(value) y delete(value), pero no size ni iteraciones.

    Al ser débil, también sirve como almacenamiento adicional. Pero no para datos arbitrarios, sino para hechos sí/no. Una membresía en WeakSet puede significar algo sobre el objeto. Por ejemplo, podemos agregar usuarios a WeakSet para realizar un seguimiento de los que visitaron nuestro sitio:

    const visitedSet = new WeakSet()

const john = { name: "John" }
const pete = { name: "Pete" }
const mary = { name: "Mary" }

visitedSet.add(john) // John nos visita
visitedSet.add(pete) // luego Pete
visitedSet.add(john) // John otra vez

// visitedSet tiene 2 usuarios ahora

// comprobar si John nos visitó?
console.log(visitedSet.has(john)) // true

// comprobar si Mary nos visitó?
console.log(visitedSet.has(mary)) // false

john = null;

// visitedSet se limpiará automáticamente

    La limitación más notable de WeakMap y WeakSet es la ausencia de iteraciones y la imposibilidad de obtener todo el contenido actual. Esto puede parecer inconveniente, pero no impide que WeakMap / WeakSet haga su trabajo principal: ser un almacenamiento adicional de datos para objetos que se almacenan/administran en otro lugar.
