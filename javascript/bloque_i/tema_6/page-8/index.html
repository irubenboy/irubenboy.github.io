    8 Fechas¶

    Introducción¶

    El objeto Date es un objeto de Javascript que representa una unidad de tiempo. Este objeto almacena la fecha, la hora, y brinda métodos para administrarlas.

    Creación¶

    Para crear un nuevo objeto Date se lo instancia con su constructor new Date(). Si no se le indica ningún argumento, se obtendrá la fecha y hora actual:

    console.log(new Date()) // Fecha y hora actual

    Si recibe un único argumento numérico, este argumento se entenderá un timestamp. Un timestamp es un número entero que representa la cantidad de milisegundos transcurridos desde el inicio de 1970. Este timestamp es una representación numérica liviana de una fecha. Es posible crear una fecha a partir de un timestamp usando new Date(timestamp), y convertir el objeto Date actual a un timestamp utilizando el método date.getTime(). Las fechas anteriores a 01.01.1970 tienen timestamps negativos,

    const Jan02_1970 = new Date(24 * 3600 * 1000)
console.log(Jan02_1970)

    Si por el contrario, se pasa un único argumento de cadena, entonces la cadena será analizada y se convierte en fecha automáticamente.

    const date = new Date("2017-01-26")

    También se puede crear una fecha con los siguientes argumentos new Date(año, mes, día, horas, minutos, segundos, ms) siendo solamente obligatorio los dos primeros.:

    • El año debería tener 4 dígitos. Por compatibilidad, aquí 2 dígitos serán considerados 19xx, pero 4 dígitos es lo firmemente sugerido.
    • La cuenta del mes comienza desde el 0 (enero), y termina en el 11 (diciembre).
    • El parámetro fecha efectivamente es el día del mes, si está ausente se asume su valor en 1.
    • Si los parámetros horas/minutos/segundos/ms están ausentes, se asumen sus valores iguales a 0
    new Date(2011, 0, 1, 0, 0, 0, 0) // 1 Jan 2011, 00:00:00
new Date(2011, 0, 1)

    Métodos para trabajar con los componentes¶

    Una vez creado el objeto Date, dicho objeto dispone de varios métodos que se encargan de obtener las diferentes partes de una fecha. Entre ellos, encontramos:

    • getFullYear(). Devuelve el año con 4 dígitos.
    • getMonth(). Devuelve el mes de 0 (Enero) a 11 (Diciembre).
    • getDate(). Devuelve el día del mes, un número comprendido entre 0 y 31.
    • getHours(). Obtiene la hora.
    • getMinutes(). Obtiene los minutos.
    • getSeconds(). Obtiene los segundos.
    • getMilliseconds(). Devuelve los componentes horario correspondientes.
    • getDay(). Devuelve el día de la semana, siendo 0 el Domingo hasta el 6 siendo Sábado.
    • getTime(). Devuelve el timestamp de una fecha.
    • getTimezoneOffset(). Devuelve la diferencia entre UTC y el huso horario actual, en minutos.

    Todos estos métodos devuelven los componentes correspondientes a la zona local.Además, existes los métodos UTC, que devuelven los componentes para la zona horaria UTC+0, a excepción de getTime() y getTimezoneOffset(). Esos métodos tienen una nomenclatura similar a los anteriores, solo hay que poner la palabra UTC entre get y el nombre del método, por ejemplo: getUTCFullYear(), getUTCDay(), etc.

    Al igual que tenemos los métodos para obtener los componentes, existen diferentes métodos para modificarlos. Su nomenclatura es bastante similar, sustituyendo el get por el set.

    • setFullYear(year, [month], [date]). Específica el año de una fecha, así como de forma opcional, el mes y el día.
    • setMonth(month, [date]). Específica el mes de una fecha, así como de forma opcional, el día.
    • setDate(date). Específica el día de una fecha.
    • setHours(hour, [min], [sec], [ms]). Específica la hora de una fecha, así como de forma opcional, los minutos, los segundos y los milisegundos.
    • setMinutes(min, [sec], [ms]). Específica los minutos de una fecha, así como, opcionalmente, los segundos y milisegundos.
    • setSeconds(sec, [ms]). Específica los segundos de una fecha, y opcionalmente, los milisegundos.
    • setMilliseconds(ms). Específica los milisegundos de una fecha.
    • setTime(ms). Establece la cantidad de segundos transcurridos desde 01.01.1970 GMT+0.

    A excepción de setTime(), todos los demás métodos poseen una variante UTC, por ejemplo: setUTCHours().

    const date = new Date()

console.log('Year:', date.getFullYear())

date.setFullYear(2020)

console.log('Year:', date.getFullYear()) // 2020

    Autocorrección¶

    La autocorrección es una característica muy útil de los objetos Date. Podemos fijar valores fuera de rango, y se ajustarán automáticamente. Veamos un ejemplo:

    const date = new Date(2013, 0, 32)
console.log(date) // 1-02-2013

    Los componentes de la fecha que están fuera de rango se distribuyen automáticamente. Por ejemplo, supongamos que necesitamos incrementar la fecha 28 Feb 2016 en 2 días. El resultado puede ser 2 Mar o 1 Mar dependiendo de si es año bisiesto. Afortunadamente, no tenemos de qué preocuparnos. Sólo debemos agregarle los 2 días y el objeto Date se encargará del resto:

    const date = new Date(2016, 1, 28)
date.setDate(date.getDate() + 2)

console.log(date) // 1 Mar 2016

    Esta característica se usa frecuentemente para obtener la fecha, a partir de un período de tiempo específico. Por ejemplo, supongamos que queremos obtener la fecha de hoy pero transcurridos 70 segundos a partir de este preciso instante.

    const date = new Date()
date.setSeconds(date.getSeconds() + 70)

console.log(date) // Se muestra la fecha correcta.

    También podemos fijar valores en 0 o incluso valores negativos. Por ejemplo:

    const date = new Date(2016, 0, 2) // 2 Jan 2016

date.setDate(1)
console.log(date) // 1 Jan 2016

date.setDate(0)
console.log(date) // 31 Dec 2015

    Conversión de fechas a números¶

    Cuando convertimos un objeto Date a número toma el valor del timestamp actual, al igual que el método date.getTime():

    const date = new Date()

console.log(+date) // similar a date.getTime()

    Las fechas pueden ser restadas, el resultado es su diferencia en milisegundos. Suele ser usado para la medición de tiempo:

    const start = new Date() 

// simula una operación de 100.000 milisegundos
for (const i = 0; i < 100000; i++) {
  const doSomething = i * i * i
}

const end = new Date()

console.log(`El tiempo transcurrido es de ${end - start} ms`)

    Fecha actual¶

    Para obtener los milisegundos actuales, hasta hora creábamos un objeto con new Date() y llamabamos al método getTime(). Sin embargo, existe un método estático Date.now() que nos devuelve el timestamp actual. Esta forma es mucho más rápida, y por lo consiguiente, no afecta al recolector de basura.

    Mayormente se utiliza por conveniencia o cuando la performance del código es fundamental, como por ejemplo en juegos de JavaScript u otras aplicaciones específicas.

    Veamos el ejemplo anterior con el nuevo método:

    const start = Date.now() 

// simula una operación de 100.000 milisegundos
for (const i = 0; i < 100000; i++) {
  const doSomething = i * i * i
}

const end = Date.now()

console.log(`El tiempo transcurrido es de ${end - start} ms`)

    Formato de fecha¶

    En JavaScript, el constructor new Date() puede aceptar una cadena que representa una fecha. Hay varios formatos de cadena que son compatibles con new Date(), como el formato UTC, el formato ISO, o formatos personalizados. Si una fecha es creada a partir de una cadena y ésta no es correcta, se retornará un Invalid Date.

    Existen diferentes patrones para especifica una fecha, por ejemplo YY representa la fecha con dos dígitos, MM, el mes con dos dígitos, etc. Al hacer uso de estas expresiones puede dar lugar a ambigüedades dependiendo del la zona horaria en la que te encuentres, por lo que no se recomienda su uso.

    Sin embargo, Javascript, dispone de diferentes métodos que realizan dichas operaciones de forma segura.

    El primero de ellos toString() devuelve la fecha con un formato UTC con la timezone correspondiente a su llamada, existe el método equivalente UTC toUTCString():

    const event = new Date('August 19, 1975 23:15:30')

console.log(event.toString()) // Tue Aug 19 1975 23:15:30 GMT+0200 (CEST)

    También existe el método toDateString() que devuelve una cadena con solo la fecha, en formato UTC (y un similar toTimeString() para únicamente el tiempo).

    const event = new Date('August 19, 1975 23:15:30')

console.log(event.toDateString()) // Tue Aug 19 1975

    Si se desea obtener la fecha en formato ISO 8601, existe el método toISOString(). El formato ISO es similar a: YYYY-MM-ddTHH:mm:ss.ms

    const date = new Date()
const isoString = date.toISOString()
console.log(isoString) // 2024-05-28T12:35:20.000Z

    El método toLocaleString([locale], [options]) devuelve una cadena con el formato local indicado:

    const date = new Date();
const formattedDate = date.toLocaleString('en-US') // Formato para EE.UU.
console.log(formattedDate) // 5/28/2024, 18:00:00

const formattedDateEs = date.toLocaleString('es-ES') // Formato para España
console.log(formattedDateEs) // 28/5/2024, 18:00:00

    Pueden recibir un objeto con las opciones a modificar de la cadena a devolver. Entre sus propiedades encontramos:

    • weekday. Indica el nombre de la semana. Puede tener tres valores diferentes: long para indicar el nombre completo (ej: Tuesday), short para indicar el nombre con las tres primeras letras (ej: Tue), o narrow para indicar el nombre con la inicial (ej: T).
    • year. Representa el año. Sus valores pueden ser numeric para mostrar el año con los 4 dígitos o 2-digit para representarlo con 2.
    • month. Representa el mes. Sus valores pueden ser, para representar el nombre: long, short, narrow; y para representarlo en dígito: numeric, que representa el mes con el número del mes (ej: 5) o 2-digit (ej: 05).
    • day: Representa el día del mes. Sus valores pueden ser numeric o 2-digit.
    • timezone: Específica la zona a utilizar.
    • timezoneName: Representa la forma en la que se representa.
    • hour, minute, second: Representa horas, minutos y segundos respectivamente, pudiendo ser sus valores numeric o 2-digit.
    • hour12: Indica si la hora se representa con formato 12 (true) o 24 (false) horas. Si el valor es true, además se añadirá AM o PM según el caso.
    const options = {
  year: 'numeric',
  month: 'long',
  day: 'numeric',
  hour: 'numeric',
  minute: 'numeric',
  second: 'numeric',
  hour12: true
}

console.log(new Date().toLocaleString('es-ES', options)) // 30 de mayo de 2024, 11:37:57 a. m.

    Además,existen los métodos toLocaleDateString() y toLocaleTimeString() para formatear solo la fecha o la hora, respectivamente.

    El método Date.parse(str) permite convertir una cadena en una fecha, devolviendo un timestamp. Si el formato no es válido retorna un NaN:

    const date = new Date(Date.parse("2012-01-26T13:51:50.417-07:00"))

console.log(date)
