    2 Clases de caracteres¶

    Introducción¶

    Una clase de caracteres es una notación especial que coincide con cualquier símbolo de un determinado conjunto

    Entre las diferentes clases de caracteres, la más usadas son:

    • \d. Representa un dígito, es decir, un carácter de 0 a 9.
    • \w. Representa un carácter de palabra, es decir, letra de alfabeto latino, un dígito, o un guión bajo (_). Las letras no latina no pertenecen a este conjunto.
    • \s. Representa un espacio en blanco, incluyendo tabulaciones, líneas nuevas, etc.
    const str = '+7(903)-123-45-67'

const regEx = /\d/g

console.log(str.match(regEx)) // [7, 9, 0, 3, 1, 2, 3, 4, 5, 6, 7]

    Los conjuntos pueden ser usados individualmente o seguidos. Por ejemplo, la expresión /\d\s\w/i indica que buscamos un número seguido de un espacio seguido de un carácter de palabra.

    Inversas¶

    Para cada clase de caracteres que hemos visto anteriormente, existe una clase inversa, indicada con la misma letra pero en mayúsculas y hace que coincida con el resto de caracteres que no cumple con el conjunto original:

    • \D. Representa cualquier carácter que no sea un dígito.
    • \W. Representa cualquier carácter que no sea un carácter de palabra, como un carácter no latino, o un espacio.
    • \S. Representa cualquier carácter que no sea un espacio.
    const str = '+7(903)-123-45-67'

const regEx = /\D/g

console.log(str.replace(regEx, str)) // 79031234567

    Punto¶

    El patrón punto (.) es una clase de caracteres especial que representa a cualquier carácter excepto a una nueva línea.

    console.log(/./.test('Z')) // true

    El punto, por defecto, no contempla el salto de línea (\n), por lo que la siguiente expresión va a dar null:

    console.log('A\nB'.match(/A.B/)) // null

    Sin embargo, haciendo uso de la bandera \s, hace que el punto contemple el salto de línea como cualquier carácter. De esta forma, el ejemplo anterior, ya tiene una coincidencia:

    console.log('A\nB'.match(/A.B/s)) // [A\nB]

    Warning

    Por lo general, se presta poco atención a los espacios, se da por hecho que las cadenas 1-5 y 1 - 5 son idénticas, cuando en realidad no lo son. A la hora de comprobar un patrón sobre una cadena hay que tener en cuenta los espacios, ya que si no se les tiene en cuenta la comprobación podría fallar:

    console.log('1 - 5'.match(/\d-\d/)) // null

    Unicode¶

    Cada carácter en Unicode tiene varias propiedades, es decir, describen a que categoría pertenece el carácter y contienen información diversa al respecto. Por ejemplo, si un carácter tiene la propiedad Letter, significa que pertenece a una alfabeto de cualquier idioma, o Number que significa que es un dígito.

    Se puede buscar caracteres teniendo en cuenta a la propiedad Unicode a la que pertenece, haciendo uso de \p{...}. Para usar dicha expresión, es necesario usar la flag u. Por ejemplo, p{Letter} representa una letra en cualquier idioma. También se puede usar p{L}:

    const str = "A ბ ㄱ"

console.log(str.match(/\p{L}/gu)) // [A, ბ, ㄱ]
console.log(str.math(/\p{L}/g)) // null

    Haciendo uso de \P{...} estaremos excluyendo los caracteres de la propiedad Unicode indicada. Por ejemplo, con \P{L}, excluimos todas las letras de cualquier idioma.

    Las categorías de letras (Letter o L), son:

    • lowercase (Ll)
    • modifier (Lm)
    • titlecase (Lt)
    • uppercase (Lu)
    • other (Lo)

    Las categorías de números (Number o N), son:

    • Dígito decimal Nd
    • Letter number Nl
    • other No

    Otras categorías y subcategorías importantes:

    • Punctuation (puntuación) P:
      • connector (conector) Pc,
      • dash (guión) Pd,
      • initial quote (comilla inicial) Pi,
      • final quote (comilla final) Pf,
      • open (abre) Ps,
      • close (cierra) Pe,
      • other (otro) Po.
    • Mark (marca) M (acentos etc):
      • spacing combining (combinación de espacios) Mc,
      • enclosing (encerrado) Me,
      • non-spacing (sin espaciado) Mn.
    • Symbol (símbolo) S:
      • currency (moneda) Sc,
      • modifier (modificador) Sk,
      • math (matemática) Sm,
      • other (otro) So.
    • Separator (separador) Z:
      • line (línea) Zl,
      • paragraph (párrafo) Zp,
      • space (espacio) Zs.
    • Other (otros) C:
      • control Cc,
      • format (formato) Cf,
      • not assigned (sin asignación) Cn,
      • private use (uso privado) Co,
      • surrogate (sustituto) Cs.

    También existen categorías derivadas, como Alphabetic o Hex_digit. Veamos un ejemplo, donde busquemos números escritos en Hexadecimal:

    const regEx = /x\p{Hex_Digit}/u

console.log('Número: xAF'.match(regEx)) // xAF

    Existe una propiedad Unicode llamada Script, que permite buscar caracteres de un sistema de escritura dado. Para ello, debemos usar Script=<value> siendo value el valor del sistema de escritura:

    const regexp = /\p{sc=Han}/gu // devuelve sinogramas chinos

const str = `Hello Привет 你好 123_456`

console.log(str.match(regexp)) // 你,好

    Anclas¶

    Los caracteres caret ^ y dólar $ tienen un significado especial en una expresión regular, tienen una función de anclas.

    El carácter ^ representa el principio de una cadena, mientras que $ representa el final:

    const str = 'Javascript is a programming language'

console.log(/^Javascript/.test(str)) // true

    La expresión ^Javascript indica que la cadena comienza con la palabra Javascript. De igual manera, funciona el carácter $:

    const str = 'Javascript is a programming language'

console.log(/programming$/.test(str)) // true

    Al usar ambos anclajes a la vez, nos permite probar si una cadena coincide al completo con el patrón deseado. Por ejemplo, para comprobar si la entrada de un usuario es correcta, o si la contraseña indicada por el usuario cumple los requisitos:

    const good = '12:34'
const bad = '12:234'

const regEx = /^\d\d:\d\d$/

console.log(regEx.test(good)) // true
console.log(regEx.test(bad)) // false

    Cuando el modo multilinea está activado (m) el comportamiento de las anclas se ve afectado. De esta forma, no solo ha de coincidir el principio y el final de la cadena con el patrón, sino que cada una de las líneas de la cadena:

    const str = `1er lugar: Winnie
2do lugar: Piglet
3er lugar: Eeyore`

console.log(str.match(/^\d/gm)) // [1, 2, 3]

    En el ejemplo anterior, vemos como la cadena debe empezar por un dígito, y el resultado devuelve un array con los 3 valores, uno por cada línea. Sin hacer uso de la flag m, solo devolvería 1, que es el inicio de la cadena. De la misma forma funciona el $.

    Límite de palabra¶

    Como hemos visto antes, existen dos anclas para indicar el comienzo y el final de una cadena. Pero, en ocasiones, nos interesa limitar una palabra dentro del patrón. Para ello, se usa el carácter \b. Cuando el motor regex se encuentra con dicho carácter, comprueba que la posición en la cadena es un límite de palabra. Existen tres posiciones que se califican como limites de palabra:

    • Al comienzo de la cadena, si el primer carácter de la cadena es una carácter de palabra (\w).
    • Entro dos caracteres de la cadena.
    • Al final de la cadena, si el último carácter es un carácter de palabra.
    console.log("Hello, Java!".match(/\bJava\b/)) // Java
console.log("Hello, JavaScript!".match(/\bJava\b/)) // null

    También se puede usar el límite de palabra con dígitos.
