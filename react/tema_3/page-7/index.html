    5 React Router¶

    Introducción¶

    Existen varios paquetes que manejan todo el tema de las rutas, de forma muy parecido a la forma en la que la hemos realizado. Uno de los más conocidos es react-router y su versión más reciente es la v6. Otra de las librerías es wouter. La ventaja de esta con la de rect-router-dom es que es más liviana. Ambas librerías te permiten crear enrutados de forma declarativa, sencilla y de forma eficiente.

    El paquete React Router es una librería para realizar navegaciones, pero es muy limitada a en cuanto el navegador, por lo que existe una versión más específica y con el acceso a la API del navegador React Router Dom que debe ser instalada:

    npm install react-router-dom

    BrowserRouter¶

    BrowserRouter es un componente de React Router DOM que se utiliza para conectar la aplicación React a la URL del navegador. Esto significa que, cuando un usuario cambia la URL en el navegador, el componente BrowserRouter actualizará la interfaz de usuario de la aplicación para que coincida con la nueva ruta.

    En concreto, BrowserRouter utiliza la API History de HTML5 para mantener la sincronización entre la URL del navegador y la interfaz de usuario de la aplicación. Cuando un usuario cambia la URL, la API History llama a un método en el componente BrowserRouter para que actualice la interfaz de usuario.

    BrowserRouter es el componente de enrutador más utilizado en React Router DOM. Se recomienda utilizarlo en la mayoría de las aplicaciones React, ya que proporciona una forma sencilla y eficaz de conectar la aplicación a la URL del navegador.

    ReactDOM.createRoot(document.getElementById('root')).render(
    <BrowserRouter>
        <App />
    </BrowserRouter>
)

    Aquí hay algunos ejemplos de cómo se puede utilizar BrowserRouter:

    • Para crear una aplicación web que se comporte como una aplicación web progresiva (PWA).
    • Para crear una aplicación web que se pueda utilizar en dispositivos móviles y de escritorio.
    • Para crear una aplicación web que utilice la navegación por historial para permitir a los usuarios volver a páginas anteriores.

    Tiene un comportamiento muy parecido a nuestro componente Router del ejemplo anterior.

    Routes¶

    El componente Routes es un componente de React Router DOM que se utiliza para definir las rutas de una aplicación React. Cada ruta tiene un nombre y un componente asociado que se indica a través del componente Route. Cuando un usuario navega a una ruta, el componente asociado se renderiza en la interfaz de usuario de la aplicación.

    El componente Route tiene dos propiedades principales:

    • path: La ruta que debe coincidir con la URL del navegador.
    • element: El componente que se debe renderizar cuando la ruta coincide.

    El componente Routes hace otra de las funcionalidades de nuestro componente Router, y el componente Route es muy parecido al de nuestro ejemplo anterior, pero el componente Route de esta librería recibe el elemento (componente ejecutado) mientras que en nuestro ejemplo le pasábamos el componente para luego ejecutarlo internamente.

    function App() {
    return (
        <div className='App'>
            <Routes>
                <Route path='/' element={<HomePage />} />
                <Route path='/search' element={<SearchPage />} />
            </Routes>
        </div>
    )
}

    Fuera del componente Routes se puede tener otros componentes o elementos. Esto puede ser útil para elementos que se repita en todas las rutas, como por ejemplo un menú de navegación:

    function App() {
    return (
        <div className='App'>
            <header>
                <h1>Chollos</h1>
                <nav>
                    <ul>
                        <li><a href='/search'>Search Page</a></li>
                        <li><a href='/'>Home</a></li>
                    </ul>
                </nav>
            </header>
            <Routes>
                <Route path='/' element={<HomePage />} />
                <Route path='/search' element={<SearchPage />} />
            </Routes>
        </div>
    )
}

    SPA con React Router Dom¶

    Al ejecutar ele ejemplo anterior, comprobamos que no se cumple el propósito de una SPA, si no que se sigue recargando cada vez que se hace clic en un anchor. Por lo que debemos cambiar la url, sin necesidad de hacer la navegación. Para ello, podemos usar el componente Link de la librería. De esta forma conseguimos nuestro objetivo:

    function App() {
    return (
        <div className='App'>
            <header>
                <h1>Chollos</h1>
                <nav>
                    <ul>
                        <li><Link to='/search'>Search Page</Link></li>
                        <li><Link to='/'>Home</Link></li>
                    </ul>
                </nav>
            </header>
            <Routes>
                <Route path='/' element={<HomePage />} />
                <Route path='/search' element={<SearchPage />} />
            </Routes>
        </div>
    )
}

    El componente Link funciona de forma muy parecida a nuestro componente del ejemplo anterior.

    Rutas con parámetros¶

    React router dom usa path-to-regex para las rutas con parámetros, por lo que si se desea indicar una ruta que puede recibir parámetros se indicaría :name-of-parameter.

    function App() {
    return (
        <div className='App'>
           {/* ... */}
            <Routes>
                <Route path='/' element={<HomePage />} />
                <Route path='/search' element={<SearchPage />} />
                <Route path='/product/:name' element={<ProductPage />} />
            </Routes>
        </div>
    )
}

    Para recuperar dicho valor en el componente, React router dom nos pone a nuestra disposición el hook useParams, que nos devuelve un objeto con los nombre de los parámetros que le hemos indicado:

    function ProductPage() {
    const { name } = useParams()
    return (
        <div>
            <h1>Product {name}</h1>
        </div>
    )
}

    Nested Routes¶

    Una nested routes (rutas anidadas) es una ruta que en su interior quiere otro tipos de rutas:

    function ProductDetails() {
    const { name } = useParams()
    return (
        <h1>Details from {name}</h1>
    )
}

function App() {
    return (
        <div className='App'>
            {/* ... */}
            <Routes>
                <Route path='/' element={<HomePage />} />
                <Route path='/search' element={<SearchPage />} />
                <Route path='/product/:name' element={<ProductPage />}>
                    <Route path='details' element={<ProductDetails />} />
                </Route>
            </Routes>
        </div>
    )
}

    Hay que tener en cuenta, que ñas rutas indicadas debe ser de forma relativas, para que no haya incompatibilidad con el resto de rutas y me parta desde la ruta absoluta indicada. Para que el componente anidado se renderice debo indicarle donde se debe realizar, para ellos usamos el componente <Outlet/>. Indicamos dicho componente donde queremos que se renderice nuestro componente:

    function ProductPage() {
    const { name } = useParams()
    return (
        <div>
            <h1>Product {name}</h1>
            <Link to='details'>Go to the details</Link>
            <Outlet />
        </div>
    )
}

    Cuando se usa rutas anidadas se puede indicar un índice como elemento principal de la ruta, renderizando de esa forma el componente de la ruta padre y el componente índice:

    <Routes>
    <Route path='/product' element={<h1>Product</h1>}>
        <Route index element={<h2>Esto es un índice</h2>} />
    </Route>
</Routes>

    Ruta por defecto¶

    React router dom no admite las rutas por defecto como en nuestro ejemplo anterior, pero podemos usar el path-to-regex para poder trabajar con ellas. La ruta por defecto, sería una ruta que renderizará un componente para cualquiera de los casos no contemplado, en regex sería la ruta /*:

    function App() {
    return (
        <div className='App'>
            {/* ... */}
            <Routes>
                <Route path='/' element={<HomePage />} />
                <Route path='/search' element={<SearchPage />} />
                <Route path='/product/:name' element={<ProductPage />}>
                    <Route path='details' element={<ProductDetails />} />
                </Route>
                <Route path='*' element={<h1>Not found</h1>} />
            </Routes>
        </div>
    )
}

    Hay que tener en cuenta que en versiones anterior de react router dom, las rutas eran comprobadas de arriba a abajo, pero desde la versión 6 existe un algoritmo que intenta averiguar que ruta es más importante. Al entrar solo en uno, si yo tengo dos rutas parecidas, pero con componentes diferentes entraría en el primero que encuentre mayor coincidencia. Por ejemplo:

    <Routes>
    <Route path='/products/pc' element={/*...*/>} />
    <Route path='/products/:name' element={/*...*/>} />
</Routes>

    En este caso, cuando hagamos una navegación a /products/pc entraría en la primera ruta ya que es la que más coincide con la ruta, por lo que renderizará ese componente. Es importante que hay que tener en cuenta que son dos rutas muy parecidas pero con elementos diferentes, y es importante saber que querer renderizar en cada caso. Si se desea entrar en /products/other, en este caso, si entraría en el segundo.

    NavLink¶

    El componente NavLink es un componente de React Router DOM que se utiliza para crear enlaces de navegación. Los enlaces de navegación son elementos de la interfaz de usuario que permiten a los usuarios navegar entre diferentes rutas de una aplicación React. Se diferencias de Link es que NavLink tiene la capacidad de determinar si está activo, por lo que estos se suelen usar en el menú de navegación.

    En versiones anteriores de la librería, el componente Link recibía una prop para indicar el estilo de un enlace cuando nos encontrábamos en dicha ruta (activeClassName), pero esto fue eliminado en la versión 6. Ahora el prop className puede recibir, o el nombre de la clase que le quieres indicar, o una función. Dicha función recibe información sobre el enlace, entre ellos si esta activo (isActive) y devuelve el nombre de la clase dependiendo de la lógica que le deseas aplicar:

    function App() {
    return (
        <div className='App'>
            <header>
                <h1>Chollos</h1>
                <nav>
                    <ul>
                        <li>
                            <NavLink
                                to='/search'
                                className={({ isActive }) => {
                                    return isActive ? 'isActive' : ''
                                }}
                            >
                                Search Page
                            </NavLink>
                        </li>
                        <li><NavLink to='/'>Home</NavLink></li>
                    </ul>
                </nav>
            </header>
            {/* ... */}
        </div>
    )
}

    En caso de que no sé indique, se le añade por defecto la clase active cuando este activo. También podemos evitar pasarle una función, creando un componente que trabaje con ello:

    function NavLink({ to, children, ...props }) {
    return (
        <NavLinkReactRouter
            {...props}
            className={({ isActive }) => isActive ? 'isActive' : undefined}
            to={to}
        >
            {children}
        </NavLinkReactRouter>
    )
}

    El componente NavLinkReactRouter es un alías al componente NavLink de React Router dom, para evitar la incompatibilidad. De esta forma, podemos usar nuestro componente sin tener que repetir el className para cuando esté activo. Esto se puede modificar de muchas formas, por ejemplo, puede recibir un nombre para la clase cuando esté activo y hacerlo más dinámico.

    useRoutes¶

    useRoutes es un hook de react router dom que es equivalente al componente Routes, es decir, sirve para indicar las rutas de nuestra aplicación. Sin embargo, usa un array de objetos en lugar del componente Route. Tanto el objeto como el componente Route tienen las mismas propiedad: path para indicar la ruta y element para indicar el elemento a renderizar. Este hook retorna el elemento que debe ser renderizado según nuestra ruta:

    const routes = [
    {
        path:'/',
        element: <HomePage/>
    },
    {
        path:'/search',
        element: <SearchPage/>   
    },
    {
        path:'/product/:name',
        element: <ProductPage/>,
        children: {
            path: 'details',
            element: <ProductDetails />
        }
    },
    {
        path: '*',
        element: <h1>Not found</h1>
    }
]
function App() {

    const element = useRoutes(routes)

    return (
        <div className='App'>
            <header>
                <h1>Chollos</h1>
                <nav>
                    <ul>
                        <li>
                            <NavLink to='/search'>Search Page</NavLink>
                        </li>
                        <li><NavLink to='/'>Home</NavLink>
                        </li>
                        <li>
                            <NavLink to='/adios'>Not found</NavLink>
                        </li>
                    </ul>
                </nav>
            </header>
            {element}
        </div>
    )
}

    Rutas protegidas¶

    Una ruta protegida es aquella ruta que solo podemos acceder en caso de realizar algún tipo de acceso (login). Para ello debemos renderizar nuestros elementos dependiendo de si estamos o no autenticados. Para ello vamos a usar un contexto que lo comprueba:

    const AuthContext = createContext()

export const useAuth = () => {
    const auth = useContext(AuthContext)
    return auth
}

export const AuthProvider = ({ children }) => {
    const [isAuthenticated, setIsAuthenticated] = useState(false)

    const login = () => {
        setIsAuthenticated(true)
    }

    const logout = () => {
        setIsAuthenticated(false)
    }

    return (
        <AuthContext.Provider value={{
            isAuthenticated,
            login,
            logout
        }}
        >
            {children}
        </AuthContext.Provider>
    )
}

    El hook useNavigate es un hook que permite cambiar la localización cuando se necesario, por ejemplo, cuando se hace clic en un botón que no sea un anchor.

    El componente <Navigate> permite cambiar de localización cuando es renderizado. Internamente, dicho componente envuelve el hook useNavigate. Tanto el componente como el hook recibe la ruta a la cual quiere ser enviada, además puede recibir información extra en una propiedad de objeto de llamado state.

    Vamos a envolver el componente que queremos renderizar cuando estemos en una ruta privada, en un componente que comprueba si estamos logueado, en caso de estarlo, devuelve el componente que deseemos renderizar, y si no el componente que se encarga del login.

    function Login() {
    const { login } = useAuth()
    const navigate = useNavigate()

    const handleClick = () => {
        login()
        navigate('/search')
    }
    return (
        <button onClick={handleClick}>Login</button>
    )
}

const ProtectedRoute = ({ children }) => {
    const { isAuthenticated } = useAuth()

    if (!isAuthenticated) return <Navigate to='/login' />

    return children
}

function App() {
    return (
        <AuthProvider>
            <div className='App'>
               {/*  */}
                <Routes>
                    {/* ... */}
                    <Route path='/search' element={<ProtectedRoute><SearchPage /></ProtectedRoute>} />
                   {/* ... */}
                </Routes>
            </div>
        </AuthProvider>
    )
}

    A la hora hacer el login, le podemos indicar la ruta desde la que viene, para volver a navegar a ella cuando realicemos el login. Para ello, usaremos el hook useLocation, que se encarga de la información sobre la localización y se lo mandaremos desde el objeto state, que es el objeto donde se le puede mandar información entre navegaciones.

    const ProtectedRoute = ({ children }) => {
    const { isAuthenticated } = useAuth()
    const location = useLocation()

    if (!isAuthenticated) return <Navigate to='/login' state={{ location }} />

    return children
}

function Login() {
    const { login } = useAuth()
    const navigate = useNavigate()
    const { state } = useLocation()

    const handleClick = () => {
        login()
        navigate(state?.location?.pathname ?? '/')
    }
    return (
        <button onClick={handleClick}>Login</button>
    )
}
