    4 Navegación¶

    Introducción¶

    En React para poder realizar navegaciones se puede realizar de la misma forma que Javascript Vanilla. A través de un enlace podemos indicar la ruta a la que queremos acceder, teniendo en cuenta que en nuestro componente principal debemos tratar dicha ruta para saber que componente poder renderizar:

    function HomePage() {
    return (
        <>
            <h1>Home</h1>
            <p>Esta es una página de ejemplo para crear un React Router desde cero</p>
            <a href='/about'>Ir a sobre nosotros</a>
        </>
    )
}

function AboutPage() {
    return (
        <>
            <h1>About</h1>
            <div>
                <img src='https://pbs.twimg.com/profile_images/1737906083207720961/53U-hMUE_400x400.jpg' alt='Foto' />
                <p>¡Hola! Me llamo Miguel Ángel y estoy creando un clon de React Router</p>
            </div>
            <a href='/'>Ir a la hombre</a>
        </>
    )
}

function App() {
    const [currentPath, setCurrentPath] = useState(window.location.pathname)
    return (
        <main>
            {currentPath === '/' && <HomePage />}
            {currentPath === '/about' && <AboutPage />}
        </main>
    )
}

    En el ejemplo anterior tenemos un estado currentPath cuya inicialización es la ruta actual, dependiendo de si la ruta es '/' o '/about' renderizará un componente u otro. En cada uno de los componentes hay un enlace para redirigirse al otro componente.

    Multiple Page Application vs Single Page Application¶

    Un Multi Page Application (MPA) es un tipo de aplicación basado en varias páginas, cada una de las cuáles se carga por separado. Esta es la forma que usa por defecto en cualquier proyecto nativo de HTML o Javascript. En React, este tipo de aplicación puede dar problemas ya que cada vez que realicemos una navegación se realizará una carga de todos los componentes y utilidades de nuestro proyecto, lo que no sería eficiente.

    Para poder evitar eso, podemos hacer que nuestra aplicación sea de tipo Single Page Application (SPA). Una SPA es un tipo de aplicación donde sólo existe una única página y se va cambiando el contenido de la misma según la navegación.

    Para poder conseguir esto en React, creamos una función navigate(href) que reciba la ruta, y realizamos un pushstate al historial de navegación. Podemos acceder al historial de navegación con la variable global window.history. pushState es un método de window.history que añade al historia una nueva ruta sin necesidad de tener que hacer una nueva carga. El método recibe tres parámetros:

    • Objeto del estado: información que se desea mantener entre navegaciones.
    • Título: título de la página (normalmente se suele indicar vacío).
    • Path: ruta que añadir al historial.
    function navigate(href){
    window.history.pushState({}, '', href)

    const navigationEvent = new Event('pushstate')
    windows.dispatchEvent(navigationEvent)
}

    Además, dicho método despacha un nuevo evento que será escuchado por nuestro componente cada vez que se haga una nueva navegación:

    //...
function App(){
    const [currentPath, setCurrentPath] = useState(window.location.pathname)

    useEffect(() => {
        const onLocationChange = () => {
            setCurrentPath(window.location.pathname)
        }

        window.addEventListener('pushstate', onLocationChange)

        return () => {
            window.removeEventListener('pushstate', onLocationChange)
        }
    }, [])
}

    Hay que tener en cuenta que en los componente HomePage y AboutPage hay que quitarle el valor por defecto del anchor y poder realizar la navegación con nuestra función navigate:

    function HomePage() {
    const handleClick = event => {
        event.preventDefault()

        const href = event.target.getAttribute('href')
        navigate(href)
    }
    return (
        <>
            <h1>Home</h1>
            <p>Esta es una página de ejemplo para crear un React Router desde cero</p>
            <a href='/about' onClick={handleClick}>Ir a sobre nosotros</a>
        </>
    )
}

function AboutPage() {
    const handleClick = event => {
        event.preventDefault()

        const href = event.target.getAttribute('href')
        navigate(href)
    }

    return (
        <>
            <h1>About</h1>
            <div>
                <img src='https://pbs.twimg.com/profile_images/1737906083207720961/53U-hMUE_400x400.jpg' alt='Foto' />
                <p>¡Hola! Me llamo Rubén y estoy creando un clon de React Router</p>
            </div>
            <a href='/' onClick={handleClick}>Ir a la hombre</a>
        </>
    )
}

    Back Navigation¶

    Otro de los problemas que nos podemos encontrar en nuestra navegación SPA es que al hacer clic en el botón de atrás del navegador, no ocurre nada. Por lo que, en el efecto de nuestro componente, debemos escuchar el evento popstate y cambiar el estado de nuestra variable currentPath:

    //...
function App(){
    const [currentPath, setCurrentPath] = useState(window.location.pathname)

    useEffect(() => {
        const onLocationChange = () => {
            setCurrentPath(window.location.pathname)
        }

        window.addEventListener('pushstate', onLocationChange)
        window.addEventListener('popstate', onLocationChange)

        return () => {
            window.removeEventListener('pushstate', onLocationChange)
            window.removeEventListener('popstate', onLocationChange)
        }
    }, [])
}

    Otras formas de navegación¶

    Existen otras formas de realizar una navegación, a parte de hacer clic con el botón derecho del ratón. Por ejemplo, con Ctrl y clic derecho del ratón se puede abrir el enlace en otra pestaña, con Alt y clic derecho del ratón se puede abrir el enlace en otra ventana, con el clic izquierdo del ratón se abre en otra pestaña, etc.

    Para poder manejar esto, vamos a crear un componente Link que renderice un enlace, con la ruta a seguir y el target como props.

    function Link({target, to, ...props}) {
    const handleClick = () => { //(1)!
        event.preventDefault()
        navigate(to)
    }

    return <a onClick={handleClick} href={to} target={target} {...props} /> 
}
    1. Con ...props indicamos que puede recibir más props, de los cuáles no tenemos porque conocerlos. Esto sirve para poder recibir atributos HTML sin tener que declararlos, como el className o los hijos envueltos por el elemento; luego, se inyecta en el elemento deseado

    Luego, podemos usar ese componente en nuestros componentes Home y About en lugar del enlace:

    function HomePage() {
    return (
        <>
            {/* ... */}
            <Link to='/about'>Ir a sobre nosotros</Link>
        </>
    )
}

function AboutPage() {
    return (
        <>
            {/* ... */}
            <Link to='/'>Ir a la hombre</Link>
        </>
    )
}

    De esta forma, el resto de tipos de navegación no se puede realizar, ya que ha sido anulado por el preventDefault. A través del evento podemos comprobar si se ha usado alguna tecla o que botón del ratón se ha clicado. Por lo que, el método handleClick del component Link quedaría:

    function Link({target, to, ...props}){
    const handleClick = event => {
        const isMainEvent = event.button === 0 //(1)!
        const isModifiedEvent = event.metaKey || event.altKey || event.ctrlKey || event.shiftKey // (2)!
        const isManageableEvent = target === undefined || target = '_self' // (3)!

        if (isMainEvent && isManageableEvent && !isModifiedEvent) {
            event.preventDefault()
            navigate(to)        
        }
    }
    //...
}
    1. Se comprueba que se ha clicado el botón principal del ratón
    2. Comprueba si se ha clicado un tecla especial
    3. Se comprueba si el target no esta definido o por el contrario es _self

    Como se puede observar, se realiza las comprobaciones pertinente y luego se navega con nuestro método navigate haciendo el preventDefault si se ha clicado con el botón principal del ratón y si se debe abrir el enlace en la misma página y no se ha clicado ningún botón especial. Para cualquier otro caso, se deja la navegación por defecto.

    Componente Router¶

    Podemos crear un componente Router que pueda manejar de forma más sencilla todo lo que hemos creado hasta ahora. Este componente recibirá además, un objeto con las rutas y los componentes a usar.

    function Router({ routes = [], defaultComponent: DefaultComponent = () => <h1>404</h1> }) {
    const [currentPath, setCurrentPath] = useState(window.location.pathname)

    useEffect(() => {
        const onLocationChange = () => {
            setCurrentPath(window.location.pathname)
        }

        window.addEventListener(EVENTS.PUSH_STATE, onLocationChange)
        window.addEventListener(EVENTS.POP_STATE, onLocationChange)

        return () => {
            window.removeEventListener(EVENTS.PUSH_STATE, onLocationChange)
            window.removeEventListener(EVENTS.POP_STATE, onLocationChange)
        }
    }, [])

    const Page = routes.find(({ path }) => path === currentPath)?.Component

    return Page ? <Page /> : <DefaultComponent />
}

    En dicho componente, recibe un lista con las rutas a renderizar y un componente por defecto (para cuando se indica una ruta no incluida en nuestras rutas). Además, manejamos el estado de la variable currentPath y a través del useEffect manejamos las escuchas a los eventos. Nuestro componente principal quedará así:

    function App() {
    const routes = const routes = [
        {
            path: '/',
            Component: HomePage
        },
        {
            path: '/about',
            Component: AboutPage
        }
    ]

    return (
        <main>
            <Router routes={routes} />
        </main>
    )
}

    Rutas con parámetros¶

    En ciertas ocasiones las rutas pueden contener cierta información que queremos renderizar, como por ejemplo, el id de un elemento para hacer un fetching de datos, el valor para realizar una búsqueda, realizar la internacionalización de nuestra App, etc. Para poder tratarlo hacemos uso del paquete path-to-regex que haciendo uso de parámetros con el formato :value podemos acceder a dicho valor indicado.

    function Router({ routes = [], defaultComponent: DefaultComponent = () => <h1>404</h1> }) {

    let routeParams

    const isValidPath = (path) => {
        if (path === currentPath) return true

        const matcherURL = match(path, { decode: decodeURIComponent })
        const matched = matcherURL(currentPath)

        if (!matched) return false

        routeParams = matched.params
        return true
    }

    // ...

    const Page = routes.find(({ path }) => isValid(path))?.Component

    return Page 
            ? <Page routeParams={routeParams} /> 
            : <DefaultComponent routeParams={routeParams} />
}

    En el método isValidPath lo primero que se comprueba es si la ruta coincide exactamente con la ruta actual, de esta forma nos saltamos el resto de navegaciones. Luego se decodifica la ruta, de tal forma que podamos relacionarlo con la ruta de tipo /ruta/:param ya que el valor de :param puede ser cualquiera, comprobamos si coincide de forma parcial y además valida que hay un parámetro. El método matcherURL me devuelve un objeto con la información de la ruta, entre ellas los parámetros, que se le envía a los componentes para que puedan utilizarlo.

    Enviar los parámetros como props del componente quizás no sea la mejor práctica, pero se puede evitar haciendo uso del Context o de custom hooks.

    Componente Route¶

    En nuestra aplicación controlamos la navegación a través del componente Router y le inyectamos las rutas a través de un prop. Podemos crear un componente para que defina una ruta y reciba como prop la ruta y el componente a renderizar, este componente NO va a renderizar nada, simplemente me va a servir para manejar las rutas de una forma diferente:

    function Route({ path, Component }) {
    return null
}

    En nuestro componente Router podemos trabajar con los children, de tal forma que podemos extraer información de estas a través de la utilidad Children que nos ofrece React para poder iterar con los hijos de un componente:

    function Router({ routes = [], defaultComponent: DefaultComponent = () => <h1>404</h1> }) {

    let routeParams

    // ...

    const routesFromChildren = Children.map(children, ({ props, type }) => {
        const { name } = type
        const isRoute = name === 'Route'

        return isRoute ? props : null
    })

    const routeToUse = routes.concat(routesFromChildren)

    const Page = routeToUse.find(({ path }) => isValid(path))?.Component

    return Page 
            ? <Page routeParams={routeParams} /> 
            : <DefaultComponent routeParams={routeParams} />
}

    En el componente anterior, transformamos cada children de nuestro componente en un objeto donde obtenemos la ruta y el componente a renderizar, comprobando antes si el children indicado es del tipo Route. Posteriormente, dicho array de objetos se añade a nuestras rutas indicadas por parámetros (en caso de que se quiera mantener la opción con el array de rutas en el prop), y luego se validan funcionando como hasta ahora.

    Nuestro componente principal quedaría de la siguiente forma:

    //...
function App() {
    const routes = const routes = [
        {
            path: '/search/:query',
            Component: SearchPage
        }
    ]
    return (
        <main>
            <Router routes={routes} defaultComponent={Page404}>
                <Route path='/' Component={HomePage} />
                <Route path='/about' Component={AboutPage} />
            </Router>
        </main>
    )
}

    Carga perezosa de componente¶

    A la hora de realizar las rutas cargamos todos los componentes que forma parte de nuestra aplicación. Lo malo de esto es que cargamos componentes en momentos que no vamos a usar en este momento. Por ejemplo, si estamos en la ruta \ se carga el componente HomePage (el utilizado), pero además se carga el componente AboutPage (el no utilizado). Este último no es necesario hacer una carga en este caso.

    React nos ofrece el método lazy para poder realizar importaciones dinámicas (con el método import) para poder cargar los componentes necesarios en el momento que vayan a ser usados y no antes:

    const LazyHomePage = lazy(() => import('./pages/Home.jsx'))
const LazyAboutPage = lazy(() => import('./pages/About.jsx'))

    Para que nuestro componente principal pueda importarlo cuando sea necesario, se debe volver en un componente Suspend que puede recibir en una prop llamada fallback un componente para indicar que se está cargando:

    function App() {
    //...
    return (
        <main>
            <Suspense fallback={<div>Loading...</div>}>
                <Router routes={routes} defaultComponent={Page404}>
                    <Route path='/' Component={HomePage} />
                    <Route path='/about' Component={AboutPage} />
                </Router>
            </Suspense>
        </main>
    )
}
