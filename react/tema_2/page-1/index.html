    1 Introducción a los Hooks¶

    Introducción¶

    Un componente en React puede sufrir cambios. Podemos tener un botón con el mensaje de Seguir para comenzar a seguir una persona en una red social. Una vez clicado el botón comenzamos a seguir a dicha persona,por lo que no tiene sentido que siga apareciendo el mensaje de Seguir si no uno que indique que ya se está siguiente. Podemos realizar este cambio en la interfaz haciendo uso de una nueva renderización.

    Debemos tener en cuenta que cuando queremos que una renderización se haga es para poder reflejar los cambios en la interfaz de usuario. Supongamos que tenemos un componente que contiene otro componente que retorna el botón, el mensaje de texto lo recibe un prop, pero se puede cambiar y se realiza en el componente padre.

    const App = () =>{
    const changeText = () => {
        text = "Mensaje nuevo"
        // ...
    }

    return (
        <p onClick={changeText}>Si clicas aquí cambias el texto del botón</p>
        <Button text="Mensaje del botón"/>
    )
}

    Al querer que ese cambio se vea reflejado, podemos hacer que se vuelva a renderizar. Por el contrario, si el cambio de texto de un botón sucede dentro desde el mismo botón, podemos hacer uso de la propiedad target del evento asociado, de esta forma no será necesaria una renderización.

    const App = () => {

    const toggleFollow = e => {
        const target = e.target

        if(target.innerText === 'Seguir'){
            target.innerText = 'Siguiendo'
        } else {
            target.innerText = 'Seguir'
        }
    }

    return (
        <>
            <button onClick={toggleFollow}>Seguir</button>
        </>
    )
}

    Ciclo de vida de un componente¶

    En React, los componentes pasan por diferentes fases durante su ciclo de vida. Sin embargo, con la introducción de los hooks en React 16.8, el término ciclo de vida ha cambiado un poco. Antes, los componentes de clase tenían métodos de ciclo de vida, pero con los hooks, los componentes de función también pueden tener un ciclo de vida.

    1. Inicialización. Sucede cuando se crea una instancia del componente e inicializa su estado. En los componente de clase sería la llamada al constructor().
    2. Montaje. Adapta las propiedades del componente y lo renderiza. También se realiza las operaciones iniciales que sean requeridas por el DOM. Por ejemplo, hacer una llamada a una API. En los componente de clase se llamaría a los siguientes métodos en este orden: static getDerivedStateFromProps(), render(), componentDidMount().
    3. Actualización. Sucede cuando se produce un cambio en el estado del componente, es decir, en algunas de sus propiedades. Se va renderizando cada uno de los elementos que sea necesario, si no se produce ningún cambio se evita la actualización para así optimizar el rendimiento. Los métodos de los componentes de clase a llamar son: static getDerivedStateFromProps(), shouldComponentUpdate(), render(), getSnapshotBeforeUpdate(), componentDidUpdate().
    4. Desmontaje. Sucede justo antes de que el componente sea eliminado del árbol DOM. componentWillUnmount()

    Hooks¶

    Un hook es una función especial de React que te permite llevar a cabo ciertas tareas, como por ejemplo, manejar el estado de un componente, compartir el contexto, etc.

    Los hooks fueron añadidos en la versión 16.8 de React, para poder tratar con el tema de los estados de una forma más sencilla en los componentes de función. Los hooks no funcionan con los componentes de clases ya que estos manejan el estado de una forma más diferente y ya implementada.

    Algunos de los hooks más comunes son:

    • useRef(): Sirve para referenciar un elemento.
    • useState(): Sirve para manejar estados de un componente.
    • useEffect(): Permite realizar efectos secundarios en componentes de función, como solicitudes de red o manipulaciones DOM.
    • useContext(): Permite acceder al contexto de React dentro de un componente.
    • useReducer(): Es una alternativa a useState(), por lo que también, sirve para gestionar el estado de un componente.

    useRef¶

    useRef es un hook de React que te permite referenciar un valor que no es necesario renderizarlo. Se trata de una herramienta poderosa que ofrece una gran flexibilidad y unas capacidades inmensas, pero los desarrolladores a menudo lo malinterpretan y lo utilizan incorrectamente.

    El hook useRef tiene dos propósitos principales:

    • Almacenar valores mutables que no provocan una nueva renderización cuando se actualizan.
    • Almacenar referencias a elementos DOM.

    Cuando un componente se renderiza en React, normalmente se restablecen su estado y otras variables. Sin embargo, hay casos en los que necesitas conservar ciertos valores incluso cuando el componente se vuelva a renderizar a un valor que persistirá entre renderizaciones, asegurando que ell valor permanece intacto aunque cambien otras partes del componente. Además, es fundamental para trabajar con elementos DOM.

    En React, acceder a los elementos del DOM y modificarlos directamente puede resultar complicado, especialmente sin el Hook useRef. Con él, puedes obtener una referencia a un elemento DOM concreto y realizar operaciones sobre él. Esto elimina la necesidad de bibliotecas externas o de complicadas soluciones.

    Para usar el hook, será necesario primero importarlo de la librería de react, y luego crear una constante que almacene dicha referencia:

    import {useRef} from 'react'

const myRef = useRef()

    Una vez creado el objeto ref se puede utilizar para almacenar valores y acceder a ellos. Se le puede asignar a la prop ref del elemento que nos interese:

    <div ref={myRef}>This is an element</div>

    Para acceder al valor de la referencia, se usa la propiedad current del objeto ref:

    const value = myRef.current
console.log(value) // <div>This is an element</div>

    Manipular el DOM es una tarea habitual en el desarrollo web porque te permite cambiar y actualizar dinámicamente el contenido, la estructura y la apariencia de una página web. En el desarrollo tradicional de JavaScript, acceder y manipular elementos del DOM requería utilizar métodos como getElementById, querySelector, o getElementsByClassName para seleccionar elementos específicos del documento. Una vez seleccionados, puedes actualizar el contenido, modificar los estilos o adjuntar escuchadores de eventos.

    // HTML
<div>
  <input type="text" id="myInput" />
  <button id="focusButton">Focus Input</button>
</div>
// JavaScript
<script>
      const inputRef = document.getElementById('myInput');
      const focusButton = document.getElementById('focusButton');
      const handleFocus = function() {
        inputRef.focus();
      };
      focusButton.addEventListener('click', handleFocus);
</script>

    Sin embargo, cuando se trabaja con elementos del DOM en un componente React, el proceso no es el mismo debido al DOM virtual del componente y a la necesidad de gestionar las actualizaciones de forma eficiente. A menudo, los desarrolladores recurren a diversos enfoques, como el uso de refs o librerías externas como jQuery, para acceder a los elementos del DOM y manipularlos.

    Con la introducción del Hook useRef en React, el proceso de trabajar con elementos DOM dentro de los componentes se ha agilizado significativamente. El Hook useRef proporciona una forma directa de crear una referencia a un elemento DOM, haciéndolo fácilmente accesible y manipulable dentro del contexto del componente.

    import { useRef } from 'react';

const FocusComponent = () => {
  const inputRef = useRef(null);

  const handleFocus = () => {
    // accessing the input element
    let inputElement = inputRef.current;

   // modify the DOM element
   inputElement.focus();
  };

 return (
    <div>
      <input type="text" ref={inputRef} />
      <button onClick={handleFocus}>Focus Input</button>
    </div>
  );
}

    Proyecto TwitterFollowCard¶

    En este proyecto vamos a crear una tarjeta de usuario de twitter, en la cual, se va modificar el botón cada vez que se interactúe sobre él.

    En nuestro archivo principal, main.js, hacemos nuestra llamada a nuestro componente principal App:

    import ReactDOM from 'react-dom/client'
import { App } from './App'
import './index.css'

const root = ReactDOM.createRoot(document.getElementById('root'))

root.render(
  <App />
)

    Nuestro componente principal hace una llamada a nuestro componente TwitterFollowCard, 5 veces:

    import './App.css'
import { TwitterFollowCard } from './TwitterFollowCard'

export function App() {
    return (
        <section className='App'>
            <TwitterFollowCard isFollowing username="selenagomez" name="Selena Gomez" />
            <TwitterFollowCard username="taylorswift13" name="Taylor Swift" />
            <TwitterFollowCard isFollowing username="midudev" name="Miguel Ángel Duran" />
            <TwitterFollowCard isFollowing username="irubenboy" name="Rubén Moreno Narbona" />
            <TwitterFollowCard isFollowing name="Desconocido" />
        </section>
    )
}

    El componente TwitterFollowCard contiene los elementos principales para tratar con los elementos, incluido una referencia para trabajar con el botón:

    import { useRef } from 'react'

export function TwitterFollowCard({ username = 'unknown', name, isFollowing = false }) {

    const buttonRef = useRef()
    const followedClassName = 'tw-followCard-button-followed'

    const handlerButtonClick = () => {
        if (buttonRef.current.classList.contains(followedClassName)) {
            buttonRef.current.classList.remove(followedClassName)
            buttonRef.current.innerText = 'Seguir'
        } else {
            buttonRef.current.classList.add(followedClassName)
            buttonRef.current.innerHTML = ''
        }

    }
    return (
        <article className='tw-followCard'>
            <header className='tw-followCard-header'>
                <img className='tw-followCard-avatar' src={`https://unavatar.io/twitter/${username}`} alt="Avatar" />
                <div className='tw-followCard-info'>
                    <strong>{name}</strong>
                    <span className='tw-followCard-infoUsername'>@{username}</span>
                </div>
            </header>
            <aside>
                <button
                    ref={buttonRef}
                    className={`tw-followCard-button${isFollowing ? (' ' + followedClassName) : ''}`}
                    onClick={handlerButtonClick}
                >
                    {isFollowing ? "" : "Seguir"}
                </button>
            </aside>

        </article>
    )
}

    Más detalladamente, el elemento button es el que contiene la referencia:

    const buttonRef = useRef()

//...

return (
    //...
    <button ref={buttonRef}>//...</button>
)

    Manejaremos el contenido haciendo uso de la clase tw-followCard-button-followed si es seguido, por el contrario el elemento, no contendrá dicha clase. Este sería el CSS del botón:

    .tw-followCard-button {
    margin-left: 16px;
    border: 0;
    border-radius: 999px;
    padding: 6px 16px;
    font-weight: bold;
    cursor: pointer;
}

.tw-followCard-button:hover {
    background-color: #a3a3a3;
    transition: .3s ease background-color;
}

.tw-followCard-button-followed {
    background-color: transparent;
    color: white;
    border: 0.5px solid white;
}

.tw-followCard-button-followed::after {
    content: 'Siguiendo';
}

.tw-followCard-button-followed:hover {
    color: rgb(190, 47, 47);
    border-color: rgb(190, 47, 47);
    background-color: rgb(190, 47, 47, .25);
}

.tw-followCard-button-followed:hover::after {
    content: 'Dejar de seguir';
    transition: background-color 0.1s linear 0.1s;
}

    Y con el método handleButtonClick manejamos la acción de hacer clic al botón:

    const handlerButtonClick = () => {
    if (buttonRef.current.classList.contains(followedClassName)) {
        buttonRef.current.classList.remove(followedClassName)
        buttonRef.current.innerText = 'Seguir'
    } else {
        buttonRef.current.classList.add(followedClassName)
        buttonRef.current.innerHTML = ''
    }
}

    En el código anterior, gracias a la propiedad current de la referencia, podemos acceder a la propiedad classList que devuelve la lista de clases del elemento. De esta manera, comprobando si contiene o no la clase tw-followCard-button-followed se le eliminará o añadirá dicha clase, respectivamente, además de que se le indique un texto Seguir en caso de que la contenga, y nada en caso de que no la contenga (en ese caso, el texto es manejado con CSS que al añadir la clase trabajará con las propiedad hover y after)
