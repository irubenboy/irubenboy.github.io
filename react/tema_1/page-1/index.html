    1 Introducción a React¶

    ¿Qué es react?¶

    React es una biblioteca de Javascript utilizada para construir interfaces de usuario (UI) interactivas y componentes reutilizables en aplicaciones web. Fue desarrollada y es mantenida por Facebook, y es ampliamente utilizada en la industria para crear aplicaciones web de una sola página (Single Page Applications) y aplicaciones móviles utilizando React Native.

    Algunas de las características clave de React incluyen:

    • Componentes: React se basa en el concepto de componentes, que son piezas independientes de la interfaz de usuario que pueden ser reutilizadas en diferentes partes de una aplicación.
    • Virtual DOM: React utiliza un Virtual DOM (DOM virtual) para optimizar el rendimiento. En lugar de actualizar directamente el DOM del navegador cada vez que cambia el estado de una aplicación, React actualiza primero el Virtual DOM y luego compara las diferencias (diffing) para actualizar solo las partes del DOM que han cambiado. Esto mejora la eficiencia y la velocidad de la aplicación.
    • Unidireccionalidad de datos: React promueve el flujo de datos en una sola dirección, lo que significa que los datos fluyen desde un componente principal hacia sus componentes secundarios. Esto hace que sea más fácil de entender y depurar cómo los datos afectan a la interfaz de usuario.
    • JSX: React utiliza una extensión de JavaScript llamada JSX (JavaScript XML) que permite definir la estructura de la interfaz de usuario de manera similar al HTML, pero dentro de JavaScript. Esto facilita la creación de componentes de manera declarativa.
    • Amplia comunidad y ecosistema: React tiene una comunidad activa y un amplio ecosistema de bibliotecas y herramientas complementarias que lo hacen aún más poderoso y versátil.

    En resumen, React es una herramienta popular y poderosa para crear interfaces de usuario dinámicas y eficientes en aplicaciones web y móviles, y ha sido ampliamente adoptada en la industria del desarrollo de software.

    React es agnóstica de la plataforma, es decir, no está vinculado a una plataforma específica de desarrollo o de tecnología. React tiene la capacidad de trabajar de manera afectiva en diferentes entornos y plataformas.

    React es utilizado comúnmente para construir interfaces de usuario en aplicaciones web, pero su diseño modular y su arquitectura permiten que se utilice en una variedad de contextos. Por ejemplo, puede ser usado para construir aplicaciones web de una sola página (SPA), aplicaciones móviles, e incluso aplicaciones de escritorio.

    Uno de los principales motivos para usar React es que usa Javascript de forma declarativa, y no de forma imperativa, y además está basado en componentes lo que permite reutilizar y escalar código.

    ¿Por qué React?¶

    React fue lanzada por primera vez en 2013 y desde entonces ha experimentado un rápido crecimiento en popularidad debido a su enfoque en la creación de componentes reutilizables y su paradigma de programación declarativa. Meta, la empresa detrás de React, la utiliza en todos sus productos (Facebook, Instagram, Whatsapp, etc)

    Aunque al principio no se consideraba open source, Meta decidió liberar el código fuente, pero aún así tiene una licencia permisiva.

    React es una de las librerías más demandadas y con mayor salida laboral. Además, en su versión React Native permite crear aplicaciones para móviles, y también aplicaciones para escritorio Windows y macOS.

    React tiene un gran soporte y es un gran ecosistema, además te ayudará, en caso de que quieras migrar, a otras librerías o frameworks como Angular, Vue y Svelt. También, cuenta con una API estable (en varios idiomas) y tiene un gran soporte ya que cuenta con una gran comunidad.

    Javascript vs React¶

    El uso de Javascript nativo (vainilla Javascript) es diferente a como se usa en React. En el primer caso, se usa de forma imperativo, es decir, indica COMO lo tiene que hacer, lo que imposibilita la escalabilidad, mientras que en React, se usa de forma declarativa.

    React define un elemento raíz, el cual contendrá toda el contenido de nuestra aplicación. Dicha raíz estará formada por diferentes componentes, con los elementos que se desee mostrar.

    Importante

    Es importante resaltar que en React TODO se considera un componente, desde el elemento más primario hasta el elemento más compuesto.

    React recibe los diferentes elementos que desea renderizar. La renderización es el proceso de generar la representación visual de un elemento o conjunto de elementos para que pueda ser percibido por los usuarios.

    Veamos un ejemplo en el que se renderice un botón con javascript nativo. Para ello, necesitamos un elemento HTML:

    <!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title></title>
    <style>
        button {
            background-color: #09f;
            color: #fff;
            border: 0;
            padding: 4px 8px;
            font-size: 18px;
            cursor: pointer;
        }

        body {
            background-color: #222;
            color: #f0f0f0;
        }
    </style>
</head>

<body>
    <button data-id="123">Me gusta</button>

    <script>
    // Recuperamos el botón
    const button = document.querySelector('button')

    // al hacer click en el botón, tenemos que ejecutar una función
    button.addEventListener('click', function () {
        // recuperar el id del atributo del HTML
        const id = button.getAttribute('data-id')

        // llamar a un servicio para actualizar si me gusta
        // toggleLike(id) // no necesario

        if (button.classList.contains('liked')) {
            button.classList.remove('liked')
            button.innerText = 'Me gusta'
        } else {
            button.classList.add('liked')
            button.innerText = 'Quitar me gusta'
        }
    })
    </script>
</body>

</html>

    En el ejemplo anterior, podemos observamos como tenemos un elemento HTML que representa un botón. Al hacer clic en ese botón, recuperamos su id (data-id) y modificamos su contenido de me gusta a no me gusta o viceversa. Pero, ¿y si añadimos más botones al código HTML?:

    <!-- ... -->
    <button data-id="123">Me gusta</button>
    <button data-id="456">Me gusta</button>
    <button data-id="789">Me gusta</button>


    <script>
    // al hacer click en el botón, tenemos que ejecutar una función
    button.addEventListener('click', function () {
        // recuperar el id del atributo del HTML
        const id = button.getAttribute('data-id')

        // llamar a un servicio para actualizar si me gusta
        // toggleLike(id) // no necesario

        if (button.classList.contains('liked')) {
            button.classList.remove('liked')
            button.innerText = 'Me gusta'
        } else {
            button.classList.add('liked')
            button.innerText = 'Quitar me gusta'
        }
    })
    </script>

    En este caso, el código javascript no funcionará, por lo que habrá que modificar el código javascript para implementar este cambio:

    <!-- ... -->
    <button data-id="123">Me gusta</button>
    <button data-id="456">Me gusta</button>
    <button data-id="789">Me gusta</button>


    <script>
    // Recuperamos todos los botones
    const buttons = document.querySelectorAll('button')

    buttons.forEach(button => {
        // al hacer click en el botón, tenemos que ejecutar una función
        button.addEventListener('click', function () {
            // recuperar el id del atributo del HTML
            const id = button.getAttribute('data-id')

            // llamar a un servicio para actualizar si me gusta
            // toggleLike(id) // no necesario

            if (button.classList.contains('liked')) {
                button.classList.remove('liked')
                button.innerText = 'Me gusta'
            } else {
                button.classList.add('liked')
                button.innerText = 'Quitar me gusta'
            }
        })
    })
    </script>

    Esto a la larga puede convertirse un trabajo pesado y no es nada eficiente si queremos crear diferentes elemento y cada uno con una funcionalidad diferente. Para eso y más, React nos facilitará la tarea.

    En este enlace, https://es.react.dev/, se encuentra la API de React oficial y en español.

    Trabajando con React¶

    Para trabajar con React, será necesario usar la librería React, que contiene los métodos para crear elementos y componentes, y la librería ReactDOM para poder trabajar con el DOM de forma sencilla.

    En primer lugar, para poder trabajar con React necesito un fichero HTML que defina el elemento raíz de nuestra aplicación:

    <!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>React</title>
    <style>
        button {
            background-color: #09f;
            color: #fff;
            border: 0;
            padding: 4px 8px;
            font-size: 18px;
            cursor: pointer;
        }

        body {
            background-color: #222;
            color: #f0f0f0;
        }
    </style>
</head>

<body>
    <div id="app"></div>

    <script src="react.js"></script>
</body>

</html>

    En este ejemplo, tenemos un contenedor que hemos definido con la clase app. Desde nuestro archivo javascript, obtenemos ese elemento HTML que funciona como raíz y con el método createRoot(element), de la librería ReactDOM:

    import ReactDOM from "https://esm.sh/react-dom@18.2.0/client"
import React from "https://esm.sh/react@18.2.0"

const app = document.getElementById('app') // Obtenemos el elemento HTML que funcionará como raíz
const root = ReactDOM.createRoot(app) // Indicamos que elemento es la raíz

    Para poder renderizar el contenido usamos el método render(elemento) de la librería React:

    import ReactDOM from "https://esm.sh/react-dom@18.2.0/client"
import React from "https://esm.sh/react@18.2.0"

const app = document.getElementById('app') // Obtenemos el elemento HTML que funcionará como raíz
const root = ReactDOM.createRoot(app) // Indicamos que elemento es la raíz

root.render('Hola mundo')

    Renderizar elementos¶

    Si se intenta renderizar un elemento HTML sin comillas (""), eso no funcionará ya que dará un error y no se renderizará, y si se usa con las comillas los renderizará como textos. Los elementos HTML NO se puede renderizar de forma directa, habrá que crearlos primero como elementos en React y luego ya se podrán renderizar.

    Para poder crear elementos en React, existe la función createElement() que recibe tres parámetros:

    • El primero de ellos, hace referencia al elemento HTML que debe crear.
    • El segundo, es un objeto con los atributos del elemento, como por ejemplo, su clase o su id.
    • El tercero, es el contenido de dicho elemento. Puede ser un simple dato primitivo (por ejemplo una cadena), o uno o varios elementos de React.
    import ReactDOM from "https://esm.sh/react-dom@18.2.0/client"
import React from "https://esm.sh/react@18.2.0"

const app = document.getElementById('app') // Obtenemos el elemento HTML que funcionará como raíz
const root = ReactDOM.createRoot(app) // Indicamos que elemento es la raíz
const button = React.createElement('button', {
    'data-id': 123
}, 'Button')


root.render(button)

    La función render solo puede renderizar un único elemento a la vez. Si quisiéramos renderizar por ejemplo varios botones, sería necesario envolverlos en un elemento superior y renderizar dicho elemento:

    import ReactDOM from "https://esm.sh/react-dom@18.2.0/client"
import React from "https://esm.sh/react@18.2.0"

const app = document.getElementById('app') // Obtenemos el elemento HTML que funcionará como raíz
const root = ReactDOM.createRoot(app) // Indicamos que elemento es la raíz
const button1 = React.createElement('button', {
    'data-id': 123
}, 'Button')
const button2 = React.createElement('button', {
    'data-id': 123
}, 'Button')
const button3 = React.createElement('button', {
    'data-id': 123
}, 'Button')
const wrapped = React.createElement('div', null, [button1, button2, button3]);


root.render(wrapped)

    React Fragment¶

    React a la hora de renderizar los elementos, los transforma a elementos de HTML, por lo que los añade al árbol DOM de la página. De esa forma, el ejemplo anterior pasaría a ser de la siguiente forma:

    html
    head
        title
        metas
    body
        div.root
            div
                button
                button
                button

    De forma innecesario ha creado un div para envolver a los tres botones. Esta pequeña molestia puede provocar errores a cuánto los estilos de CSS o incluso en la interactividad con el usuario, sino se tiene en cuenta. Pero, si tenemos que envolver los elementos React en un elemento superior para poder renderizar un solo elemento, ¿cómo podemos evitar este pequeño inconveniente?

    React posee un elemento que puede ser usado como elemento superior pero a la hora de transformarlo a elementos HTML y añadirlos al DOM, no crea un elemento innecesario. Este elemento se conoce como Fragment, por lo que de esta manera podemos hacer el ejemplo anterior de una forma mucho más limpia:

    import ReactDOM from "https://esm.sh/react-dom@18.2.0/client"
import React from "https://esm.sh/react@18.2.0"

const app = document.getElementById('app') // Obtenemos el elemento HTML que funcionará como raíz
const root = ReactDOM.createRoot(app) // Indicamos que elemento es la raíz
const button1 = React.createElement('button', {
    'data-id': 123
}, 'Button')
const button2 = React.createElement('button', {
    'data-id': 123
}, 'Button')
const button3 = React.createElement('button', {
    'data-id': 123
}, 'Button')
const wrapped = React.createElement(React.Fragment, null, [button1, button2, button3]);


root.render(wrapped)

    De esta manera, el árbol DOM queda más limpio y precavido de incidentes:

    html
    head
        title
        metas
    body
        div.root
           button
           button
           button

    Aún así, los ejemplos que hemos ido desarrollando usan una peculiaridad que queríamos evitar, la imperatividad. Para poder evitarla, será necesario hacer JSX y crearnos un proyecto que nos facilite el trabajo.

    Proyecto React¶

    Para poder hacer uso de la librería será necesario tener instalado NodeJS y un manejador de paquetes (npn, yarn, etc)

    Podemos crear react a través de comandos diferentes: npx create-react-app o haciendo uso de vite. La primera opción es la forma nativa y recomendada por React en su API, pero cierto es que es una forma más anticuada y mucho más lenta

    npx create-react-app example
cd example

    Para ejecutar el proyecto se usa el siguiente comando:

    npm start

    Vite ha creado un comando que hace que la creación de un proyecto sea más sencillo y más rápido:

    npm  create vite@latest
cd project

    Para poder ejecutar el proyecto creado:

    npm run dev

    Una vez ejecutado el navegador predeterminado deberá ser ejecutado con el proyecto en marcha. En caso de que no lo haga, se puede acceder a él a través de la URL http://localhost:PORT, donde el valor del puerto será por defecto 3000, en caso de usar el comando oficial, y 5173 si se usa el gestor de vite.

    Además al ejecutar el comando de creación, se habrá creado una estructura de proyecto similar a la siguiente (varia dependiendo de la versión del proyecto creado):

    Los ficheros y carpetas más importantes son:

    • Carpeta node_modules: Esta carpeta contiene todas las librerías de React y las que vayamos instalando a través de Node JS.
    • public: En esta carpeta aparecen los archivos públicos del proyecto, así como las imágenes, las plantillas html, etc.
    • src: Esta carpeta es la carpeta principal del proyecto y contiene todos los archivos necesarios para el desarrollo de nuestro proyecto, así como los componentes y el archivo principal (index.js para el comando oficial y main.jsx para vite).
