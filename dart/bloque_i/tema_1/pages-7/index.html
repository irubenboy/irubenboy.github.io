    1. Introducción¶

    Las librerías, o libraries, son una forma de estructurar un proyecto basado en modularidad, permitiendo así al desarrollador dividir el código en múltiples archvios.

    Se puede compartir parte de su código con otros desarrolladores.

    2. Libraries en Dart¶

    Al igual que en muchos otros lenguajes de programación, Dart usa librerías para permitir al desarrollador la posibilidad de estructurar el código (modularizar).

    En Dart, además, tiene otro importante rol, determinar que es visible o no para otras libraries.

    Para importar una librería, hacemos uso de la palabra reservada import seguida del archivo en la cual se encuentra nuestra librería.

    import 'my_library.dart';

    Cualquier código tipeado en la librería my_library.dartpuede ser usada ahora en el fichero que hayamos hecho la importación.

    En ocasiones, no es necesario importar todo el código de la librería, ya sea porque no necesitemos todo el código, para evitar conflitos de nombrados de dos librerías diferentes, o simplemente no lo usemos.

    Se puede usar la palabra reservada show, para mostrar el código que necesitaremos.

    import 'my_library.dart' show MathUtils; // Solo accederá la clase MathUtils de la librería

    La palabra hide sirve para ocultar el código que no vamos a utilizar.

    import 'my_library.dart' hide MathUtils; // Accederá toda la librería excepto la clase MathUtils

    3. Prefijos¶

    En ocasiones tenemos el mismo contenido en dos librerías diferentes, eso puede dar conflicto a la hora de desarrollar.

    Una de las soluciones a este problema, sería ocultar la clase del mismo nombre que no se vaya a usar, de esta forma no dará lugar a ningún tipo de conflicto, por ejemplo, tenemos una librería llamada lib_A.dart que contiene la clase User, por otro lado, tenemos otra librería llamada lib_B.dart, que también contiene la clase User, y la clase Math.

    import 'lib_A.dart'; // De esta libreria voy a usar la clase User
import 'lib_B.dart' hide User; // Solo se va a usar la clase Math para que no dé conflicto se oculta la clase User

    La desventaja de esto, es que en ocasiones es necesario usar ambas clases de ambas librerías, por lo que no se puede ocultar ninguna.

    En Dart, a diferencia de otros lenguajes, no existen namespace ni nada similar para identificar el contexto de una librería. En su caso, para evitar dicha ambigüedad, Dart propone como solución los prefijos.

    Para definir un prefijo se usa la palabra reservada as después del import.

    import 'lib_A.dart' as libA;
import 'lib_B.dart' as libB;
void main(){
    libA.User userA = libA.User(); // Usuario de la librería A
    libB.User userB = libB.User(); // Usuario de la librería B
}

    4. Rutas¶

    Para poder importar la librería debemos tener en cuenta, su ubicación. Podemos acceder a ella de estas maneras:

    • Rutas relativas: import 'directory/library.dart'
    • Rutas absolutas: import 'file:///c://directory/library.dart' (DEPENDE DEL SO)
    • Desde una URL: import 'http://www.IES.com/directory/library.dart'
    • Desde un paquete: import 'package:my_package/directory/library.dart'

    5. Packages¶

    El ecosistema de Dart usa paquetes para administrar software compartido, como bibliotecas o herramientas. La principal ventaja de usar y crear paquetes es que el código puede ser reusada y compartido.

    El manejo y configuración de paquetes se realiza a través de la herramienta pub, desde la consola del sistema operativo.

    En Dart, nos podemos encontrar con dos tipos de packages:

    • Application Packages: No están pensados para ser compartidos, aunque pueden contener dependencias. Por ejemplo: una aplicación.
    • Library Packages: Contiene código útil para otros. Se puede usar como dependencia y tener dependencias de otros.

    Haciendo uso de la herramienta pub que debe ser instalada a través del SDK de Dart podemos generar un proyecto.

    • pubspec.yaml -> Describe el paquete para el repositorio público.
    • lib -> Este es el lugar donde se ubican las libraries. Todo lo que pongamos en este directorio estará públicamente disponible para otros paquetes. Se conoce también como el “package public API”

    6. Fichero Pubspec¶

    El fichero pubspec sirve para configurar y definir los aspectos más relevantes de un paquete Dart. Está basado en yaml, un formato estándar usado sobretodo en archivo de configuración. Su estructura es fácil de leer y de seguir.

    Info

    Para más información https://dart.dev/tools/pub/pubspec

    Pubspec file
    name: dart_package
description: A sample command-line application.
# version: 1.0.0
# homepage: https://www.example.com
environment:
 sdk: '>=2.8.1 <3.0.0'
#dependencies:
#  path: ^1.7.0
dev_dependencies:
pedantic: ^1.9.0
test: ^1.14.4

    7. Dependencias de terceros¶

    Durante el desarrollo de una aplicación, podemos encontrarnos con la necesidad de de usar paquetes de terceros. Para ello, será necesario incluir dicho paquete en mi proyecto y luego importarlo donde sea necesario para poder usarlo.

    Para poder incluirlo, no es necesario descargarlo y añadirlo al proyecto, simplemente bastará con definir la dependencia en nuestro fichero de configuración pubspec e invocar el comando dart pub get desde la carpeta que contiene dicho archivo en la consola del sistema operativo que estemos usando para el desarrollo,

    name: dart_package
description: A sample command-line application.
# version: 1.0.0
# homepage: https://www.example.com
environment:
 sdk: '>=2.8.1 <3.0.0'
dependencies:
 email_validator: '^2.0.1' # Añadimos el email validator
dev_dependencies:
pedantic: ^1.9.0
test: ^1.14.4

    cd path
dart pub get

    Una vez realizado los pasos anteriores, la dependencia se cargará en el sistema y en el proyecto se ha generado un fichero (en caso de que no existiera) donde se registra la ubicación de este.

    Al no incluir dicha dependencia en nuestro workspace estamos aislando nuestro código del exterior.

    Resumen

    Para añadir una nueva dependencia a mi proyecto tenemos que añadir en el fichero pubspec dentro del apartado dependencies lo siguiente:

    dependencies:
<package>: <constraints>

    Info

    Encontrar paquetes: https://pub.dev/

    8. Constraints¶

    A la hora de añadir una dependencia, podemos definir las siguientes restricciones:

    • Vacío: Si no indicamos nada, no restringimos ninguna. Normalmente se usaría la versión más actual. email_validator:. Equivalente a email_validator: any
    • Versión en concreto: Indicamos la versión exacta. email_validator: '1.0.5'
    • Versión mínima: Indicamos que nos valdría cualquier versión desde la indicada. email_validator: '>1.0.5'. También se puede usar >=
    • Versión máxima: Al revés que la anterior, acepta como máximo la que aceptemos. email_validator: '<=1.0.5'
    • Rango: Indicamos un rango. email_validator: '>1.0 <1.0.5'
    • Rango semántico: Como mínimo la indicada y como máximo la siguiente salto. email_validator: '^1.0.0'. Equivalente a email_validator: ‘>=1.0.0 <2.0.0’. email_validator: ‘^1.1.0’. Equivalente a email_validator: ‘>=1.1.0 <1.2.0’

    Info

    Más información en https://dart.dev/tools/pub/versioning Además de poder restringir por versión también podemos indicarle a nuestro archivo de configuración restricciones de la fuente desde donde se obtiene la dependencia.

    Ejemplo de ello lo podemos ver en la siguiente URL: https://dart.dev/tools/pub/dependencies#dependency-source
