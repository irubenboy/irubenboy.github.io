    1 Instalación del entorno de desarrollo¶

    ¿Qué es Android?¶

    Android es un sistema operativo móvil basado en Linux, diseñado para ser utilizado en dispositivos móviles como teléfonos inteligentes (smart phones), tabletas (tablets), Google TV, relojes de pulsera (smart watch) y otros dispositivos.

    Acompañando al núcleo Linux del sistema operativo, Android incorpora una serie de capas de software (middleware) que hacen de intermediarias entre el núcleo y otras aplicaciones, hardware, redes, etc, aportando distintas funcionalidades al dispositivo físico.

    Android, al contrario que el sistema operativo para dispositivos móviles iOS, se desarrolla de forma abierta y se puede acceder a su código fuente. El que se tenga acceso al código fuente no significa que se pueda tener siempre la última versión de Android en un determinado móvil, ya que el código para soportar el hardware (controladores) de cada fabricante normalmente no es público, por lo que no dispondríamos del firmware para poder hacerlo funcionar en dicho terminal. Por otro lado, las nuevas versiones de Android suelen requerir más recursos, por lo que los modelos de dispositivos más antiguos pueden quedar descartados por razones de memoria (RAM), velocidad de procesador, etc.

    Las aplicaciones Android se desarrollan habitualmente en los lenguajes de programación Java o Kotlin, haciendo uso del Android Software Development Kit (Android SDK), el kit de desarrollo de software proporcionado por Google, y cuya descarga es gratuita. También tenemos disponible un kit de desarrollo nativo para aplicaciones o extensiones en C o C++, aunque sólo se usa en casos muy específicos.

    Google ha desarrollado su propia tienda online de aplicaciones para dispositivos Android, denominada Google Play. Esta web permite a los usuarios navegar y adquirir aplicaciones publicadas por los desarrolladores, ya sean de pago o gratuitas, de manera que Google retribuye a los desarrolladores con un porcentaje del precio de las aplicaciones.

    Por otra parte, los usuarios pueden instalar aplicaciones desde otras tiendas virtuales (tales como Amazon AppStore) o directamente en el dispositivo, si se dispone del archivo APK (paquete de instalación) de la aplicación.

    Un archivo con extensión .apk es un paquete para el sistema operativo Android. Este formato es una variante del formato JAR de Java y se usa para distribuir e instalar componentes empaquetados para la plataforma Android para smartphones y tablets.

    Historia¶

    El nombre Android (androide en castellano) hace referencia a la novela de Philip K. Dick llamada "¿Sueñan los androides con ovejas eléctricas?", que posteriormente fue adaptada al cine como "Blade Runner".

    En julio de 2005, Google adquirió Android Inc., una pequeña compañía de Palo Alto, California fundada en 2003. Entre los cofundadores de Android que se fueron a trabajar a Google están Andy Rubin, Rich Miner, Nick Sears, y Chris White.

    En Google, el equipo liderado por Rubin desarrolló una plataforma para dispositivos móviles basada en el kernel de Linux que fue promocionado a fabricantes de dispositivos y operadores con la promesa de proveer un sistema flexible y actualizable. Google pretendía crear una alianza con fabricantes de hardware y software para que cooperaran entre sí en el desarrollo de dispositivos móviles.

    El 5 de noviembre de 2007 la Open Handset Alliance (OHA, Alianza de Dispositivos móviles Abiertos), un consorcio de varias compañías, vio la luz con el fin de desarrollar estándares abiertos para dispositivos móviles.

    El primer producto de la OHA fue Android, una plataforma para dispositivos móviles construida sobre la versión 2.6 del kernel de Linux.

    El primer teléfono disponible en el mercado para ejecutar Android fue el HTC Dream, dado a conocer al público el 22 de octubre de 2008.

    A principios de 2010 Google colaboró con HTC para lanzar su producto estrella en dispositivos Android, el Nexus One (cuyo nombre es un guiño a la película Blade Runner), como primer dispositivo Android "puro".

    Web oficial para desarrolladores Android

    La web oficial para desarrolladores Android es https://developer.android.com/index.php, en la que podemos encontrar toda la documentación oficial.

Generalmente cada actualización del sistema operativo Android es desarrollada bajo un nombre en código de un elemento relacionado con postres o dulces. En cada versión el postre elegido empieza por una letra distinta siguiendo el orden alfabético.

    Cuando creemos un nuevo proyecto en Android Studio, éste nos permitirá acceder a una estadística sobre la distribución de uso de las distintas versiones de Android entre los usuario de Google Play.

    Google publica con cierta frecuencia la distribución de uso de las distintas versiones de Android.

    Esta información de distribución es muy importante para nosotros como desarrolladores porque nos permite conocer, entre otras cosas, con qué versiones de la API de Android nos merece la pena que sean compatibles nuestras aplicaciones, esto es, la versión mínima de la API Android para la que deberíamos desarrollar nuestras aplicaciones.

    Android ha sido criticado muchas veces por la fragmentación que sufren sus terminales. La fragmentación hace referencia a las distintas versiones de Android que hoy en día se pueden encontrar en el mercado. Las causas principales de esta fragmentación son las siguientes:

    • Android es un sistema abierto, y como tal, los fabricantes pueden personalizar la versión oficial de Android a los dispositivos que desarrollan, lo que implica que la misma versión del sistema operativo no luzca exactamente igual en distintos dispositivos.
    • El hecho de que muchos fabricantes instalan en sus dispositivos versiones personalizadas de Android, significa en la práctica, que las actualizaciones que realiza Google en la versión oficial de Android no son trasladadas inmediatamente a los dispositivos de los usuarios, ya que antes debe pasar por el proceso de personalización o adaptación que hacen los fabricantes. En algunos casos, dichas actualizaciones nunca llegan a producirse.
    • En el mercado actual conviven dispositivos provenientes de diferentes versiones oficiales de Android, como podemos apreciar en el gráfico anterior.

    Todo ello hace que la fragmentación sea un problema real para los desarrolladores de aplicaciones para Android, ya que tienen que desarrollar sus aplicaciones para multitud de dispositivos diferentes, con hardware diferente y con versiones del sistema operativo diferentes, lo que puede no ser rentable.

    Sin embargo, la fragmentación, como toda diversidad, también tiene su lado positivo para los desarrolladores, como la aparición de tiendas online (markets) alternativos que crean competencia a Google Play de Google y provocan variedad y abarcamiento de mercado, por lo que más personas descargan aplicaciones.

    Con objeto de mejorar los problemas de fragmentación, Google lleva a cabo distintas estrategias:

    • Algunas de las novedades son incorporadas a la Google Play Services, una librería propietaria de Google, que se incluye en todos los dispositivos certificados por Google, y que es compatible con versiones de Android a partir de la API 8.
    • Proporciona a los desarrolladores la librería de soporte Android Support Library, de manera que puedan utilizar las novedades de las nuevas versiones de Android al desarrollar sus aplicaciones manteniendo la compatibilidad con dispositivos con versiones de Android antiguas.

    Entorno de Desarrollo¶

    Para poder desarrollar aplicaciones Android es necesario que configuremos nuestro equipo de desarrollo con los siguientes elementos:

    • Sistema operativo: Podemos usar sistemas operativos Windows (XP o superior), en Linux (de 32 bits) y en Mac OS X (10.5.8 o superior sobre plataforma x86).
    • Lenguaje: Como se va a programar usando Java o Kotlin, es imprescindible tener instalado el JDK (Java Development Kit) de Oracle. Se recomienda usar la versión 1.8 de la JDK (Java 8).
    • Entorno integrado de desarrollo (IDE): Con objeto de hacer más asequible la programación para esta plataforma, utilizaremos el IDE Android Studio.
    • Kit de desarrollo en Android: Deberemos tener también una copia del Android SDK. Este paquete es un conjunto de herramientas que nos harán más fácil programar para Android y gestionar nuestros dispositivos virtuales. Se instala automáticamente al instalar Android Studio y podemos actualizarlo directamente desde el propio IDE.
    • Dispositivos de prueba: Podremos crear distintos dispositivos virtuales (emuladores), sobre los que simular el comportamiento de las aplicaciones que estamos desarrollando en diferentes tipos de dispositivos, con distintos tamaño y resoluciones de pantalla. Podremos crear dichos dispositivos virtuales desde el propio IDE Android Studio. Además, en vez de usar dispositivos virtuales, podemos usar dispositivos físicos reales Android en los que probar nuestras aplicaciones.

    Android Studio¶

    IntelliJ IDEA, también conocido como IntelliJ o IDEA, es un entorno de desarrollo de aplicaciones basadas en Java creado por la empresa JetBrains. Incluye plugins para Java y otros lenguajes y frameworks, entre ellos Android.

    Web oficial de IntelliJ IDEA

    https://www.jetbrains.com/es-es/idea/

    Dispone de las algunas características muy interesantes para nosotros como desarrolladores:

    • Editor de código con coloreado de sintaxis, auto-completado inteligente y refactorización de clases, diseños y estilos.
    • Diseño de interfaces de usuario mediante drag-and-drop.
    • Acceso al log del sistema (logcat), ejecución en modo debug

    Aunque se trata de un producto comercial, existe una versión community, de código abierto, que puede usarse de forma gratuita, pero que no incluye todas las funcionalidades de la versión comercial.

    En mayo de 2013, Google presenta Android Studio, un nuevo entorno de desarrollo gratuito basado en la versión community de IntelliJ IDEA, en el que se integran todas las herramientas específicas necesarias para el desarrollo y depuración de aplicaciones Android. Además de las funcionalidad aportadas por el propio IntelliJ, Android Studio proporciona:

    • Soporte para la generación de aplicaciones basadas en Gradle.
    • Refactorización y corrección rápida específicos para Android.
    • Herramienta Lint para mejorar el rendimiento, la usabilidad y la compatibilidad de nuestras aplicaciones.
    • Funcionalidad de firma de aplicaciones.
    • Asistentes basados en plantillas para crear diseños y componentes habituales.
    • Editor de layouts enriquecido, con funcionalidades avanzadas, como la posibilidad de añadir componentes a la interfaz de usuario simplemente arrastrando y soltando sobre ella, vista previa del layout en distintas configuraciones de pantalla, etc.
    • Herramientas de monitorización de uso de memoria, red, etc., por parte de nuestras aplicaciones.

    Con la instalación de Android Studio se instalan también automáticamente varios componentes importantes del Android SDK, permitiéndonos añadir componentes o actualizarlos más adelante a través del propio IDE. Existen versiones para Windows, Mac y Linux.

    Web oficial de Android Studio

    https://developer.android.com/studio

    Emulador¶

    El SDK de Android incluye un emulador de dispositivos virtuales móviles que puede ser ejecutado en nuestro propio ordenador, lo que nos permite hacer prototipos, desarrollar y probar las aplicaciones Android que desarrollemos sin tener que usar un dispositivo móvil real.

    El emulador proporciona una copia virtual de las características hardware y software de un dispositivo móvil real, excepto por el hecho de que no puede realizar llamadas telefónicas reales. Utiliza una ventana de nuestro sistema operativo para simular la pantalla del dispositivo móvil. Para manejar el emulador y su dispositivo móvil virtual dispondremos de una serie de teclas de navegación y de controles que podremos usar a través del ratón o del teclado de nuestro ordenador, que nos permitirán generar distintos eventos en nuestro dispositivo virtual para comprobar cómo se comporta ante ellos la aplicación que estamos desarrollando.

    Con objeto de permitirnos modelar y probar nuestra aplicación de una manera sencilla, el emulador utiliza las configuraciones pre-diseñadas en uno o más AVDs (Android Virtual Devices, Dispositivos Virtuales Android), que nos permiten definir características hardware específicas del dispositivo móvil que queremos emular y sobre el que queremos realizar las pruebas. Podemos crear tantos AVDs como estimemos oportuno.

    Con la instalación de Android Studio se nos instala también un dispositivo virtual por defecto, que podemos usar para probar nuestras aplicaciones, aunque a través del AVD Manager podemos crear tantos dispositivos virtuales como creemos oportuno.

    Cada AVD que definamos puede tener las características de algún dispositivo Android real existente en el mercado, o características completamente personalizadas por nosotros.

    Además, cuando creemos un AVD debemos indicar la imagen del sistema operativo Android que queremos cargar en el dispositivo virtual. De esta manera, podremos crear dispositivos virtuales con distintas características, como por ejemplo distintos tamaños de pantalla, y con distintas versiones de del sistema operativo Android, para poder comprobar cómo se comportan nuestras aplicaciones.

    Cuando queramos ejecutar la aplicación que estemos desarrollando en ese momento, tan solo bastará con tener un dispositivo virtual o real seleccionado en el combo box correspondiente, y pulsar sobre el icono correspondiente en la barra de herramientas de Android Studio.

    Si queremos ejecutar la aplicación en varios dispositivos a la vez, debemos desplegar la combo box de dispositivos y elegir la opción Select multiple devices..., tras lo que nos aparecerá una ventana de selección de dispositivos, tal y como se muestra en la imagen, donde podemos seleccionar uno o varios:

    Ejecución en dispositivo físico¶

    Aunque podemos usar el emulador proporcionado por el Android SDK para probar y depurar nuestras aplicaciones, siempre será recomendable realizar nuestras pruebas en distintos dispositivos físicos para poder comprobar cómo se comporta nuestra aplicación en realidad. De esta manera podremos analizar su velocidad de ejecución, su comportamiento sobre distinto hardware, cómo reacciona ante diferentes eventos que acontezcan en el dispositivos, etc.

    De hecho, algunas funcionalidad de nuestras aplicaciones sólo podremos probarlas en dispositivos físicos, ya que el emulador no dispone de ellas, como por ejemplo el acelerómetro.

    Para poder probar nuestras aplicaciones en un dispositivo físico debemos cumplir con los siguientes requisitos:

    • Tener instalado en el equipo de desarrollo el driver correspondiente a nuestro dispositivo físico para el sistema operativo del equipo de desarrollo. Este driver es suministrado por el fabricante del dispositivo, normalmente dentro de un pack de software del dispositivo para PC.
    • Habilitar la opción de depuración en el dispositivo físico. Para ello abrir la aplicación de configuración en nuestro dispositivo Android llamada "Ajustes" y seleccionar el menú "Opciones de Desarrollo", activando la opción "Depuración USB". A partir de la versión Kit Kat de Android el menú "Opciones de Desarrollo" viene deshabilitado por defecto. Para habilitarlo bastará con entrar en el menú "Información del dispositivo" y pulsar siete veces sobre el campo "Número de Compilación".
    • Tener conectado nuestro dispositivo al PC mediante un cable USB.

    También tenemos disponible un asistente de conexión, accesible desde Tools > Connection Assistant. Tan sólo deberemos seguir las instrucciones de conexión.

    Drivers USB

    En la web oficial para desarrolladores Android puede encontrar más información sobre los drivers USB y una lista con las páginas de descarga de los drivers de los principales fabricantes (https://developer.android.com/studio/run/oem-usb.php#Drivers).

    Existen aplicaciones para nuestro equipo de desarrollo que permiten mostrar en él la pantalla de nuestro dispositivo físico en el que estamos probando la aplicación que estamos desarrollando, permitiéndonos interactuar con él. De esta manera no es necesario manipular directamente el dispositivo al realizar las pruebas, además de poder introducir los datos directamente desde el teclado del equipo de desarrollo. Estas aplicaciones también suelen ser utilizadas para realizar demostraciones.
