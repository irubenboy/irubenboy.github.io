    6 Recursos de valores¶

    Introducción¶

    Como hemos dicho antes, algunos de los recursos de nuestro programa son nombrados explícitamente y no se toma para ello el nombre del archivo en el que se encuentran. Este es el caso de los conocidos como recursos de valores, que son almacenados en distintos ficheros XML dentro de la carpeta res/values/ de nuestro módulo.

    Recurso de valor

    Un recurso de valor corresponde a un nombre que asociamos a un determinado literal, ya sea de cadena, entero, de dimensión, etc.

    Estos ficheros XML incluirán en su interior la definición de recursos de valores. A diferencia del resto de carpetas, en las que cada fichero corresponde a un recurso, los ficheros contenidos en esta carpeta describen varios recursos, que son nombrados explícitamente.

    El contenido de cada fichero deberá comenzar con la etiqueta <resources> (y terminar con </resources>), y cada hijo de dicha etiqueta será considerado un recurso de constante. Así en un mismo archivo podemos definir múltiples recursos correspondientes, por ejemplo, a constantes de cadena.

    De hecho, podemos crear tantos ficheros de recursos de valores constantes como queramos, nombrarlos a nuestro gusto e incluso mezclar recursos de valores constantes de diferentes tipos en el mismo fichero.

    Sin embargo, para una mayor claridad es recomendable colocar todos los recursos de valores del mismo tipo en un mismo fichero.

    Normalmente se usan nombres estándar a la hora de nombrar los ficheros de recursos de constantes, aunque no es obligatorio usarlos:

    Nombres estándar de ficheros de recursos de valor

    • res/values/strings.xml: Recursos de cadena y de arrays de de cadena.
    • res/values/integers.xml: Recursos de entero.
    • res/values/colors.xml: Recursos de color.
    • res/values/dimens.xml: Recursos de dimensión.
    • res/values/styles.xml`: Recursos de estilo y de tema.

    Como ejemplo, veamos un posible contenido del archivo strings.xml:

    <resources>
    <string name="app_name">HelloWorld</string>
</resources>

    En este caso estamos definiendo un recurso de contante de cadena cuyo nombre será app_name y cuyo valor será HelloWorld.

    Todas estas etiquetas XML usan el atributo name para que indiquemos el nombre que queremos darle al recurso.

    Podemos definir distintos tipos de recursos de valores, entre los que destacamos:

    • Recursos de cadena
    • Recursos de enteros
    • Recursos de color
    • Recursos de dimensión
    • Recursos de estilo

    Recursos de cadena¶

    <string name="titulo">Mi Aplicación</string>

    El valor puede contener las etiquetas HTML <b> (negrita), <i> (cursiva), o <u> (subrayado). Si el valor debe contener los caracteres <, >, o &, el valor debe incluirse dentro de <![CDATA[valor]]>

    Cómo hacer referencia a ellos

    • Desde código Java o Kotlin: R.string.titulo
    • Desde código XML: @string/titulo

    Recurso de Entero¶

    <integer name="columns">2</string>

    Cómo hacer referencia a ellos

    • Desde código Java o Kotlin: R.integer.titulo
    • Desde código XML: @integer/titulo

    Recurso de color¶

    <color name="fondo">#AA8862</color>

    Cómo hacer referencia a ellos

    • Desde código Java o Kotlin: R.color.fondo
    • Desde código XML: @color/fondo

    Recurso de dimensión¶

    <dimen name="margen_arriba">10dp</dimen>

    Contiene un valor junto con su unidad.

    Cómo hacer referencia a ellos

    • Desde código Java o Kotlin: R.dimen.margen_arriba
    • Desde código XML: @dimen/margen_arriba

    Warning

    Debemos evitar usar la unidad px (píxeles) para establecer distancias o tamaños en nuestros layouts. El motivo es que existen pantallas con el mismo tamaño pero que tienen distintas densidades de píxeles, por lo que en este caso, un determinado tamaño en píxeles corresponde a diferentes tamaños físicos reales dependiendo de la densidad de la pantalla en cuestión.

    Por lo tanto, a la hora de especificar dimensiones debemos emplear siempre la unidad dp (o dip).

    dp

    Un dp es un pixel independiente de la densidad (density independent pixel), y corresponde al tamaño físico de un pixel en una resolución de 160 puntos por pulgada.

    Para el tamaño del texto, a la hora de especificar una dimensión, usaremos la unidad sp, en vez de dp.

    sp

    Un sp es un pixel escalado (scaled pixel), y corresponde al valor de un dp pero escalado atendiendo al tamaño de texto especificado en las preferencias del dispositivo

    Recurso de Estilo¶

    <style name="AppTheme" parent="Theme.AppCompat.Light.DarkActionBar">
    <item name="colorPrimary">@color/colorPrimary</item>
    <item name="colorPrimaryDark">@color/colorPrimaryDark</item>
    <item name="colorAccent">@color/colorAccent</item>
</style>

    Los veremos más adelante en un apartado específico.

    Cómo hacer referencia a ellos

    • Desde código Java o Kotlin: R.style.mi_estilo
    • Desde código XML: @style/mi_estilo

    Recursos de valor en nuestro proyecto¶

    Para practicar el uso de los recursos de valor, vamos a modificar la especificación XML de layout de la interfaz de usuario de nuestra actividad, main_activity.xml para que en vez de usar el literal de cadena "Hello World!", dicho literal sea definido como un recurso de constante de cadena de caracteres

    <LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/llRoot"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical" >
    <TextView
        android:id="@+id/txtHello"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_margin="16dp"
        android:text="Hello World!" />
</LinearLayout>

    El primer paso consiste en incluir en el archivo res/values/strings.xml la definición de un nuevo recurso de cadena al que le asignaremos el nombre de main_txtHello, con lo que quedaría

    <resources>
    <string name="app_name">HelloWorld</string>
    <string name="main_txtHello">Hello World!</string>
</resources>

    Para indicar el valor del margen con el que el TextView se separa de los bordes de su padre, definiremos y usaremos un recurso de valor de dimensión. El primer paso será definir dicho recurso en algún archivo de recursos de valor, en este caso crearemos el archivo res/values/dimens.xml:

    <resources>
    <dimen name="main_margin">16dp</dimen>
</resources>

    Después deberemos modificar la propiedad android:text del TextView para que su valor haga referencia al recurso de cadena, usando el formato de referencia que hemos visto anteriormente. Por otra parte, modificaremos el atributo android:layout_margin del TextView, cuyo valor estableceremos al valor del recurso de valor de dimensión que hemos definido:

    <LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/llRoot"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical" >
    <TextView
        android:id="@+id/txtHello"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_margin="@dimen/main_margin"
        android:text="@string/main_txtHello" />
</LinearLayout>

    Sugerencia

    Se recomienda siempre definir recursos de valor en vez de usar literales, ya que esto nos va a permitir usar diferentes valores para distintas configuraciones del dispositivo, como veremos más adelante.
