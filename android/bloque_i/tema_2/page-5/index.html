    5 Entorno de Android¶

    Proyecto Login¶

    En este proyecto desarrollaremos la típica pantalla de login en la que el usuario introduce el nombre de usuario y la contraseña y pulsa un botón para hacer login.

    Crea un nuevo proyecto usando el asistente.

    Primero, activa ViewBinding en el archivo build.gradle de tu módulo:

    android {
    // ... 
    viewBinding {
        enabled=true
    }
}

    Luego, modifica el layout generado por el asistente para la actividad de manera que pase a llamar res/layout/main_activity.xml y su contenido sea el siguiente:

    <LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/llRoot"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical"
    android:padding="@dimen/main_margin">

    <ImageView
        android:id="@+id/imgLogo"
        android:layout_width="match_parent"
        android:layout_height="200dp"
        android:src="@drawable/logo_transparent"
        android:scaleType="centerInside"
        android:contentDescription="@string/main_imgLogo"/>

    <TextView
        android:id="@+id/lblUsername"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="@string/main_lblUsername"/>

    <EditText
        android:id="@+id/txtUsername"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:hint="@string/main_txtUsername"
        android:inputType="text"/>

    <TextView
        android:id="@+id/lblPassword"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="@string/main_lblPassword"
        android:layout_marginTop="@dimen/main_margin"/>

    <EditText
        android:id="@+id/txtPassword"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:hint="@string/main_txtPassword"
        android:inputType="textPassword"/>

    <Button
        android:id="@+id/btnLogin"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:text="@string/main_btnLogin"
        android:layout_marginTop="@dimen/main_margin"/>

</LinearLayout>

    Creamos también los siguientes recursos de cadena en el archivo res/values/strings.xml:

    <resources>
    <string name="app_name">Login</string>
    <string name="main_lblUsername">Username</string>
    <string name="main_txtUsername">(username)</string>
    <string name="main_lblPassword">Password</string>
    <string name="main_btnLogin">Login</string>
    <string name="main_txtPassword">(password)</string>
    <string name="main_imgLogo">Logo</string>
</resources>

    y los siguientes recursos de dimensión en el archivo res/values/dimens.xml:

    <resources>
    <dimen name="main_margin">16dp</dimen>
</resources>

    Nuestra clase MainActivity contendrá inicialmente el siguiente código:

    public class MainActivity extends AppCompatActivity {

    private MainActivityBinding binding;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        binding = MainActivityBinding.inflate(getLayoutInflater());
        setContentView(binding.getRoot());
        setupViews();
    }

    private void setupViews() {
        binding.btnLogin.setOnClickListener(v -> btnLoginOnClick());
    }

    private void btnLoginOnClick() {
        String username = binding.txtUsername.getText().toString();
        String password = binding.txtPassword.getText().toString();
        if (isValidForm(username, password)) {
            login(username, password);
        }
    }

    private boolean isValidForm(String username, String password) {
        return isValidUserName(username) && isValidPassword(password);
    }

    private boolean isValidUserName(String username) {
        return !TextUtils.isEmpty(username);
    }

    private boolean isValidPassword(String password) {
        return !TextUtils.isEmpty(password);
    }

    private void login(String username, String password) {
        // TODO: Code to do login
        // ...
    }

}

    Como vemos, sólo se llama al método login(username, password) si el formulario es válido, es decir si se han introducido los dos datos, ya que ambos son obligatorios.

    Otro aspecto curiosos es que hemos usado el método estático TextUtils.isEmpty(string) en vez de simplemente usar el método string.isEmpty(). El motivo de ello es que en el primer caso si la cadena es null se retornará true, mientras que en el segundo caso se producirá un error en ejecución NullPointerException.

    Nos queda pendiente de hacer el código que simule el establecimiento del login. En nuestro caso consistirá en mostrar un mensaje informativo temporal al usuario, como veremos en el siguiente apartado.

    Toast¶

    Toda aplicación interactiva requiere de algún mecanismo para notificar al usuario cualquier tipo de información o evento que sea relevante. Android nos proporciona varias maneras de llevar a cabo esta notificación. En este apartado estudiaremos los llamados toasts.

    Un toast (brindis en español) es un mensaje emergente que se muestra en pantalla durante unos segundos y desaparece automáticamente transcurrido dicho tiempo, sin ser necesario ningún tipo de actuación por parte del usuario. La principal ventaja de los toast es que no interfieren en las acciones que esté realizando el usuario, ya que no requieren de su intervención, de manera que no debe dejar de hacer lo que estaba haciendo para atender al mensaje. De hecho, los toasts ni siquiera reciben el foco mientras son mostrados.

    Sugerencia

    Los toasts son muy adecuados para mostrar mensajes rápidos y sencillos al usuario, ya que no son molestos, pero no deberían ser usados para hacer notificaciones importantes, ya que puede que el usuario no les preste demasiada atención antes de que dejen de mostrarse.

    Para crear los toast haremos uso de la clase Toast y, en concreto, de su método estático makeText(), que recibirá como parámetros el contexto en el que se debe mostrar, el texto del mensaje y su duración, para la que emplearemos las constantes definidas en la clase Toast denominadas LENGTH_SHORT (duración corta) y LENGTH_LONG (duración larga). El método makeText() retorna el objeto Toast correspondiente, que podremos mostrar en pantalla mediante su método show(). Veámoslo en un ejemplo:

    Toast.makeText(context, message, Toast.LENGTH_SHORT).show();

    Por defecto los toast aparecen en la parte inferior de la pantalla en un rectángulo gris semitransparente.

    Warning

    Debemos tener cuidado de no olvidar llamar al método show() del objeto Toast que hayamos creado, o de lo contrario el mensaje no se mostrará.

    Como vemos para poder crear un toast debemos pasar como argumento el contexto. Pero ¿qué es eso del contexto? Lo estudiamos a continuación.

    Context¶

    La clase Context corresponde a una clase abstracta, cuya implementación es proporcionada por el sistema operativo Android.

    Context

    El contexto (context) corresponde a un objeto que proporciona información global acerca del entorno en el que se está ejecutando la aplicación.

    A través del contexto tendremos acceso a recursos y clases específicas de la aplicación, así como a métodos para realizar operaciones a nivel global, como iniciar componentes de la misma, ya sean actividades, servicios, etc.

    De esta manera, los componentes pueden determinar en qué aplicación se encuentran, acceder a los elementos globales, como los recursos o los servicios del sistema, y comunicarse con el resto de componentes de la aplicación.

    Sin embargo, no todas las implementaciones de Context proporcionadas por el sistema son iguales. Dependiendo del componente del que se trate, el contexto al que tendremos acceso variará ligeramente.

    Por ahora nos basta con saber que cada actividad tiene su propio contexto. De hecho, la clase Activity extiende de Context, por los que cualquier actividad es de por sí un contexto.

    Más adelante veremos que hay un contexto global a la aplicación, conocido como contexto de aplicación, que será más adecuado en algunas circunstancias que ya estudiaremos.

    Warning

    Debemos usar el contexto de la actividad sólo en aquellos casos en las que la operación correspondiente esté relacionada con la interfaz de usuario. Para el resto de operaciones se recomienda usa el contexto global de la aplicación, o de lo contrario se pueden producir memory leaks (memoria desperdiciada).

    Una vez que sabemos lo que es el contexto podemos modificar nuestro Proyecto Login para que muestre un toast con los datos de conexión cuando el usuario pulse sobre el botón de login. Usaremos this como contexto, porque en este caso this hace referencia a la actividad, que implementa Context.

    public class MainActivity extends AppCompatActivity {

        // ...

    private void login(String username, String password) {
        Toast.makeText(this, 
            getString(R.string.main_connecting, username, password),
            Toast.LENGTH_SHORT).show();
    }

}

    También tendremos que añadir el siguiente recurso de cadena al archivo res/values/strings.xml:

    <string name="main_connecting">%1$s - %2$s Connecting</string>
