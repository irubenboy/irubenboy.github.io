    1 Introducción a las vistas básicas¶

    Text View¶

    El vista más simple que existe en Android es la etiqueta (label en inglés), que recibe el nombre de TextView. Una etiqueta es un fragmento de texto que, por defecto, NO puede ser editado directamente por el usuario. Se utilizan generalmente para dar información al usuario sobre un widget adyacente situado a su lado. Por ejemplo, es muy habitual encontrar una etiqueta seguida de un cuadro de texto, de manera que mediante la etiqueta se informa al usuario sobre el dato que debe introducir en el mismo (por ejemplo el nombre del cliente).

    Clase TextView

    Si queremos incluir un TextView en nuestro archivo de layout e indicar en tiempo de compilación algunas de sus propiedades, tendremos que incluir en él una especificación como la que sigue:

    <TextView
   android:id="@+id/lblName"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:text="@string/main_lblName"
   android:textColor="@color/primaryText"
   android:textStyle="bold" />

    Los atributos XML más utilizados de los TextView son:

    • android:gravity: Alineación del texto dentro del rectángulo correspondiente a la vista. Los valores más habituales son start, center y end. No tiene sentido definirlo si la anchura de la vista es wrap_content.
    • android:text: Texto que muestra.
    • android:textColor: Color del texto.
    • android:textAllCaps: Texto todo en mayúsculas.
    • android:capitalize: Pone en mayúsculas de manera automática el texto, ya sea la primera letra de la frase, la primera letra de cada palabra o todos los caracteres.
    • android:hint: Consejo (tooltip o hint) cuando no se tiene ningún texto.
    • android:textSize: Tamaño de la letra.
    • android:textStyle: Formato del texto, ya sea negrita (bold), cursiva (italic) o ambas (bold_italic).
    • android:ellipsize: Posición por la que se trunca el texto cuando no cabe en la vista, añadiendo ... (start, end, middle)
    • android:selectAllOnFocus: Recibe un booleano para indicar si queremos que se seleccione automáticamente el texto de la vista cuando éste gane el foco. Esta propiedad se usara sobre todo en los EditText que veremos a continuación.

    También disponemos de un atributo específico para el editor visual de layouts llamado tools:text, que se aplica a cualquier vista que pueda mostrar texto, para indicar el texto que debe mostrar el editor visual de layouts, aunque no se haya especificado la propiedad android:text para la vista.

    Sugerencia

    A la hora de mostrar un texto, podemos usar las colecciones de datos de ejemplo proporcionados por Android Studio. Por ejemplo para mostrar como texto un párrafo de lorem ipsum podemos usar tools:text="@tools:sample/lorem" o para mostrar un nombre tools:text="@tools:sample/full_name".

    Por otra parte, tenemos la posibilidad de asociar un determinado drawable (por ejemplo un pequeño icono a un TextView, de manera que se muestre a su lado (a su izquierda, derecha, arriba o abajo) a una determinada distancia. Para ello usaremos los siguientes atributos XML:

    • android:drawableLeft, android:drawableRight, android:drawableTop y android:drawableBottom: Drawable que debe mostrarse a su izquierda, derecha, arriba y abajo, respectivamente.
    • android:drawablePadding: Espacio (distancia) entre el texto y el drawable.

    Sugerencia

    No existe ninguna propiedad para establecer el tamaño del drawable, por lo que se recomienda que el tamaño de éste sea acorde al del texto contenido en el TextView.

    Desde código Java o Kotlin los métodos más usados de TextView serán setText(charSequence) para establecer el texto que debe mostrar y getText(), que retorna un objeto CharSequence con el texto mostrado. CharSequence es una interfaz implementada por String. Si queremos convertirlo a cadena podemos usar el método toString() del objeto CharSequence.

    String name = binding.lblName.getText().toString();
binding.lblName.setText(name.toUpperCase());

    Por otro lado, la guía de diseño material design establece una serie de diseños tipográficos estándar para el texto:

    Los tamaños de dichos diseños tipográficos están incluidos en la librería de compatibilidad support library en forma de recursos de dimensión, así que podemos establecer el atributo android:textSize="@dimen/text_size_display_4_material" (el resto de recursos son similares sustituyendo display_4 por el nombre del diseño correspondiente en la imagen anterior).

    El problema es que los diseños anteriores no sólo modifican el tamaño del texto, sino otros elementos, como si el texto debe aparecer todo en mayúsculas. Por tanto lo más sencillo es establecer a la vez varias propiedades del texto a través del atributo style, asignándole como valor alguno de los recursos de estilo definidos en la librería de compatibilidad. Por ejemplo: style="@style/Base.TextAppearance.AppCompat.Display4" (el resto de recursos son similares sustituyendo Display4 por el nombre del diseño correspondiente en la imagen anterior).

    Warning

    El atributo style se usa sin el namespace android:. El motivo es que se trata de un atributo genérico, típico de los lenguajes de marcas, como por ejemplo HTML, y no algo específico de Android, por lo que los desarrolladores del framework decidieron no incluirlo en el namespace android:.

    <TextView
   android:id="@+id/lblName"       
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:text="@string/main_lblName"
   style="@style/Base.TextAppearance.AppCompat.Headline" />

    Button¶

    Un botón (button) es un zona de pantalla que puede ser pulsada por el usuario para indicar que se empiece a llevar a cabo una acción. Android proporciona distintos tipos de vistas con funcionalidad de botón. El tipo más simple y genérico es la clase Button, que es una subclase de TextView, por lo que hereda sus propiedades y métodos.

    La guía de diseño material design establece distintos estilos visuales para los botones. El estilo por defecto es el conocido como raised button (botón elevado):

    En el siguiente ejemplo se incluye un objeto Button en un fichero de layout XML para el que se usa el estilo por defecto raised button:

    <Button 
    android:id="@+id/btnAccept"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content" 
    android:text="@string/main_btnAccept"/>

    También podemos hacer que el botón incluya un icono, haciendo uso de los atributos android:drawableLeft y android:drawablePadding, heredados de la clase TextView.
