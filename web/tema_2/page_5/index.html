    5 Cascada y colisión¶

    Introducción¶

    En las hojas de estilos complejas, es habitual que varias reglas CSS se apliquen a un mismo elemento HTML. El problema radica en que puede que varias reglas asignen un valor distinto a la misma propiedad a un determinado elemento, es decir, que contengan declaraciones contradictorias. A este hecho se le conoce como colisión de estilos.

    Veamos el siguiente ejemplo, en el que hay dos reglas que incluyen declaraciones contradictorias que producen una colisión de estilos en todos los elementos <p>:

    p {
  color: red;
}

p {
  color: blue;
}

    ¿De qué color se muestra cada uno de los párrafos del documento HTML? ¿Qué declaración prevalece sobre cuál?

    Debemos tener en cuenta que la resolución de los conflictos se tiene que realizar individualmente para cada etiqueta del documento HTML, atendiendo a las declaraciones que se le deben aplicar. Por ejemplo si ampliamos el ejemplo anterior de la siguiente manera:

    p {
  color: red;
}

p {
  color: blue;
}

p.code {
  color: green;
}

    ¿De qué color se muestra un párrafo de la clase code? ¿Y un párrafo que NO sea de la clase code? La resolución de las colisiones será distinta en cada caso.

    CSS tiene un mecanismo de resolución de colisiones muy complejo, conocido como cascada, y que tiene en cuenta varios factores o conceptos:

    • Importancia de las declaraciones.
    • Especificidad de los selectores de las reglas que contienen las declaraciones.
    • Orden de los archivos CSS y de las declaraciones dentro de ellos.

    Si dos declaraciones tienen la misma importancia, la especificidad de las reglas decidirá cuál se debe aplicar. Si las reglas tienen la misma especificidad, el orden de las fuentes controla el resultado.

    Importancia de las declaraciones¶

    La importancia de una declaración CSS depende de dónde se ha especificado. Las declaraciones contradictorias se aplicarán en el orden siguiente (de menor a mayor importancia), de manera que las últimas prevalecerán sobre las anteriores:

    1. Hoja de estilos de agente de usuario.
    2. Declaraciones normales en hojas de estilo de usuario.
    3. Declaraciones normales en hojas de estilo de autor.
    4. Declaraciones importantes en hojas de estilo de autor (diseñador).
    5. Declaraciones importantes en hojas de estilo de usuario.

    Una hoja de estilos de agente de usuario es la hoja de estilo integrada del navegador. Cada navegador tiene sus propias reglas sobre cómo mostrar varios elementos de HTML si el usuario o diseñador de la página no especifica ningún estilo. Por ejemplo, los enlaces no visitados suelen ser azules y estar subrayados.

    Una hoja de estilos de usuario es una hoja de estilo que ha especificado el usuario. No todos los navegadores son compatibles con las hojas de estilo de usuario, pero pueden ser muy prácticas, sobre todo para los usuarios con problemas de accesibilidad. Por ejemplo, una persona disléxica puede tener una hoja de estilo de usuario que especifique determinadas fuentes y colores que le faciliten la lectura.

    Cuando hablamos de "hojas de estilo", normalmente hacemos referencia a una hoja de estilo de autor. Es la hoja de estilos que ha creado o enlazado el autor (diseñador) del documento.

    Las declaraciones importantes son aquellas que van seguidas de una directiva !important. Por ejemplo:

    div{
    color: red !important;
}

div{
    color: blue;
}

    Como se puede observar, las declaraciones importantes de una hoja de estilo de usuario tienen prioridad sobre todas las demás, lo cual es lógico.

    NO se recomienda usar !important habitualmente, de hecho se considera una mala práctica, y sólo deberemos recurrir a ella cuando no tengamos otra opción.

    Especificidad¶

    La especificidad se considerarse una medida de cómo de específico es el selector de una regla. La especificidad de un selector se calcula obteniendo el valor de especificidad mediante los siguientes elementos:

    • número de estilos aplicados directamente mediante un atributo style. Corresponderá a las unidades de millar del valor de especificidad.
    • número de selectores de id #. Corresponderá a las centenas del valor de especificidad.
    • número de selectores de clase .. Corresponderá a las decenas del valor de especificidad.
    • número de selectores de tipo y de pseudoelementos. Corresponderá a las unidades del valor de especificidad.
    • El selector universal * es ignorado.

    Veamos unos ejemplos:

    div { ... }                     /* Especificidad: 0001 */
div div { ... }                 /* Especificidad: 0002 */
#pagina div { ... }             /* Especificidad: 0101 */
#pagina div:hover { ... }       /* Especificidad: 0111 */
#pagina div:hover a { ... }     /* Especificidad: 0112 */
#pagina .sel:hover > a { ...}   /* Especificidad: 0121 */

    Si se produce una colisión de estilos entre dos o más declaraciones que deben aplicarse a un determinado elemento, y todas las declaraciones tienen la misma importancia, prevalecerá la declaración contenida en la regla con el selector con valor de especificidad más alto. Por ejemplo en:

    p {
  color: blue;  /* Especificidad 0001 */
}

p.code {
  color: green; /* Especificidad 0011 */
}

    se producirá una colisión de estilos en cada párrafo de la clase code. Ambas declaraciones tienen la misma importancia, ya que ambas son declaraciones normales en hojas de estilo de autor. Sin embargo, la segunda declaración posee un valor de especificidad mayor que la primera, por lo que prevalecerá frente a ésta, de manera que los párrafos de la clase code se mostrarán en verde.

    Info

    También tenemos disponible una herramienta online para el cálculo de la especificidad de un selector.

    Orden de inclusión¶

    Si dos declaraciones afectan al mismo elemento, tienen la misma importancia y la misma especificidad, la colisión de estilos se resolverá atendiendo al orden de inclusión de las declaraciones en el documento. A continuación se muestran las distintas opciones ordenadas de menor a mayor precedencia:

    • Archivos CSS externos vinculados a través de la etiqueta <link>. Si vinculamos varios archivos CSS, las declaraciones definidas en los archivos vinculados posteriormente tiene mayor precedencia que los vinculados anteriormente.
    • Reglas CSS internas definidas a través de la etiqueta <style>.
    • Declaraciones CSS embebido en el elemento HTML mediante el atributo style.

    Dentro de cada uno de estos apartados, las declaraciones posteriores prevalecerán frente a las definidas con anterioridad.

    En el siguiente ejemplo:

    p {
  color: red;
}

p {
  color: blue;
}

    se genera una colisión de estilos en todas los párrafos, ya que se le deben aplicar dos declaraciones contradictorias. Ambas declaraciones tienen la misma importancia, ya que ambas son declaraciones normales en hojas de estilos de autor. También tienen la misma especificidad, en concreto 0001, por lo que el sistema debe usar el orden de inclusión para resolverlo. Dado que ambas declaraciones están definidas en el mismo archivo CSS, la colisión se resuelve simplemente haciendo prevalecer la definida en último lugar, por lo que los párrafos se mostrarán de color azul.

    Por todo lo indicado anteriormente, es muy importante el orden en el que se vinculan en el documento mediante la etiqueta <link> los archivos externos CSS. Por ejemplo, si vinculamos el archivos CSS correspondiente a algún framework CSS, como Bootstrap, y después queremos sobrescribir algún estilo concerto, deberemos vincular nuestro propio archivos CSS después de vincular el de Bootstrap. Por ejemplo:

    <head>
  <!-- ... -->
  <link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.7/css/bootstrap.min.css" />
  <link rel="stylesheet" href="css/estilos.css" />
  <!-- ... -->
</head>

    ​ Por otra parte, también es muy importante el orden en el que se definen las reglas dentro de un archivo CSS. De hecho, dentro de un archivo CSS, se recomienda definir primero las reglas más generales y después las más específicas, para que éstas sobrescriban los estilos definidos con anterioridad.

    Resumen¶

    El método seguido por CSS para resolver las colisiones de estilos se muestra a continuación:

    1. Determinar todas las declaraciones que se aplican al elemento para el medio CSS seleccionado.
    2. Ordenar las declaraciones según su importancia, atendiendo a su origen (hoja de estilo del navegador, de usuario o de autor) y si se ha marcado con la palabra clave !important.
    3. En caso de igual importancia, ordenar las declaraciones según la especificidad del selector de la regla en la que se encuentre. Cuanto mayor sea dicho valor mayor prioridad tendrá.
    4. Si después de aplicar las normas anteriores existen dos o más declaraciones con la misma prioridad, se comprueba si la declaración ha sido definida mediante una etiqueta <style>, que prevalecerá frente a las definidas en archivos externos CSS vinculados.
    5. Si después de aplicar las normas anteriores no se ha resuelto la colisión, se aplica la que se definió en último lugar, teniendo en cuenta en orden en el que se han vinculado los archivos CSS externos y la posición de la declaración dentro de su archivos CSS.
