    5 Tablas¶

    Introducción¶

    Desde sus primeras versiones, HTML incluyó el soporte para crear tablas de datos en las páginas web. Además de ser sencillo, el modelo definido por HTML es muy flexible y bastante completo. Las tablas en HTML utilizan los mismos conceptos de filas, columnas, cabeceras y títulos que los que se utilizan en cualquier otro entorno de publicación de documentos:

    Las tablas de HTML puede contener elementos simples, agrupaciones de filas y de columnas, cabeceras y pies de tabla, subdivisiones, cabeceras múltiples y otros elementos complejos.

    Tablas básicas¶

    Las tablas más sencillas de HTML se definen con las etiquetas: <table>, <tr>, <td> y <th>. La etiqueta <table> contiene todas las filas de la tabla. Cada fila es representa por una etiqueta <tr> (table row). Cada fila contiene una serie de celdas, representadas por la etiqueta <td> (del inglés table data cell) o <th> (del inglés table header cell) si se trata de una celda que actúa como cabecera. Por ejemplo:

    <table>
  <tr>
    <th>Curso</th>
    <th>Horas</th>
    <th>Horario</th>
  </tr>
  <tr>
    <td>CSS</td>
    <td>20</td>
    <td>16:00 - 20:00</td>
  </tr>
  <tr>
    <td>HTML</td>
    <td>20</td>
    <td>16:00 - 20:00</td>
  </tr>
  <tr>
    <td>Dreamweaver</td>
    <td>60</td>
    <td>16:00 - 20:00</td>
  </tr>
</table>

    El atributo más utilizado de la etiqueta <th> es scope, que permite indicar si la celda es cabecera de la fila (<th scope="row">) o de la columna <th scope="col">.

    Uno de los aspectos más curiosos de los tablas en HTML es que NO se define el concepto de columna, sino que el número de columnas de la tabla es calculado como el número máximo de celdas contenido por alguna fila de la tabla. Así, si una tabla contiene 2 filas, la primera fila contiene 2 celdas y la segunda 3 celdas, la tabla se mostrará con 3 columnas, ya que es el máximo de nº de celdas. Pero ¿cómo se pintan las celdas de la primera fila si sólo hay 2? Pues de izquierda a derecha, es decir, que la primera celda ocupará la columna 1 y la segunda celda la columna 2.

    Pero, ¿y si en una fila quiero dejar la primera columna sin ocupar y ocupar la segunda y la tercera? La respuestas es que NO puede hacerse, tendremos que definir para dicha columna una celda vacía. Posteriormente, mediante CSS podremos hacer que dicha celda vacía no sea visible.

    Por otra parte, HTML define la etiqueta <caption> para establecer la leyenda o título de una tabla. La etiqueta debe colocarse inmediatamente después de la etiqueta <table> y cada tabla sólo puede incluir una etiqueta <caption>.

    Si la tabla está contenida dentro de una etiqueta <figure>, que veremos cuando hablemos de las imágenes, se recomienda usar <figcaption> en vez de S.

    Fusión de filas y columnas¶

    Las tablas complejas suelen disponer de una estructura irregular que junta varias columnas para formar una columna ancha o une varias filas para formar una fila más alta que las demás. Para fusionar filas o columnas, se utilizan en los elementos <td> y <th> los siguientes atributos:

    • rowspan="num_filas": Hace que el elemento ocupe el número de filas especificado, extendiéndose hacia abajo, ocupando las celdas correspondientes.
    • colspan="num_columnas": Hace que el elemento ocupe el número de columnas especificado, extendiéndose hacia la derecha, ocupando las celdas correspondientes. A la hora de determinar en qué fila o "columna" definir un elemento <td> o <th> que vaya a ocupar más de una celda gracias a rowspan o colspan, la recomendación es siempre hacerlo donde debería estar situada la celda de más arriba a la izquierda de las que vaya a ocupar el <td> o <th>.

    Tablas avanzadas¶

    Algunas tablas complejas están formadas por más elementos que filas y celdas de datos. Así, es común que las tablas más avanzadas dispongan de una sección de cabecera, una sección de pie y varias secciones de datos. Además, también es posible agrupar varias columnas de forma lógica para poder aplicar estilos similares a un determinado grupo de columnas.

    Las partes que componen las tablas complejas se definen mediante las etiquetas <thead>, correspondiente a la cabecera de tabla, <tbody>, correspondiente a cada sección de datos de la tabla, y <tfoot>, correspondiente al pie de la tabla.

    Cada tabla puede contener solamente una cabecera y un pie, pero puede incluir un número ilimitado de secciones de datos, siempre y cuando estén situados consecutivamente. Si se define una cabecera y/o un pie, las etiquetas <tbody> y <tfoot> deben colocarse después de la etiqueta <thead>. Por su parte, la etiqueta <thead> debe situarse después de la etiqueta <caption>.

    La etiqueta <tbody> permite realizar agrupaciones de filas, pero en ocasiones se necesitan agrupar columnas. Aunque su uso no es muy común, HTML define dos etiquetas similares para marcar columnas: <col> y <colgroup>. La etiqueta <colgroup> (de column group) se emplea para agrupar de forma estructural varias columnas de la tabla. La forma habitual de indicar el número de columnas que abarca la agrupación es utilizar el atributo span, que establece el número de columnas de cada agrupación. Otra opción es en vez de usar el atributo span, incluir una o más etiquetas <col> que representen las columnas incluidas en dicho grupo de columnas. La etiqueta <colgroup> debe estar situada después de la etiqueta <caption> pero antes de <thead>.

    Estilizando tablas¶

    Las tablas tienen una serie de características muy concretas en lo relativo a su estilo:

    • Todas las celdas de la misma fila tienen la misma altura, independientemente de la altura de su contenido individual
    • Todas la celdas que pertenecen a la misma columna tienen la misma anchura, también independientemente de su contenido individual.
    • Las celdas adyacentes pueden compartir un borde, incluso si ambas poseen border con un estilo muy diferente.
    • NO es posible establecer márgenes en los elementos internos de una tabla, como <tr>, <td> o <th>, aunque sí en el <caption>.
    • Los únicos atributos CSS que tienen efecto sobre las columnas de una tabla son:
      • border: (Sólo si border-collapse tiene el valor collapse).
      • background: Sólo afectará a celdas transparentes cuya fila también sea transparente.
      • width: Corresponderá a la anchura mínima de las celdas de la columna, pero que podrá ser mayor si el contenido de alguna celda lo fuerza.
      • visibility: Sólo serán válidos el valor collapse, para que la columna se colapse y no se tenga en cuenta para la anchura de la tabla, y el valor hidden, para que no se muestre.
    • CSS define un mecanismo para insertar objetos anónimos que suplan los que falten en la tabla. Por ejemplo, si no definimos una fila con <tr> pero sí incluimos etiquetas <td>, CSS incluirá por nosotros la fila faltante.
    • CSS define seis capas en las que es dibujada la tabla correspondientes a los siguientes elementos indicados desde el inferior hasta el superior: tabla, grupo de columnas, columnas, grupo de filas, filas y celdas. Así por ejemplo, el background de una fila se mostrará encima del background de una columna.
    • El elemento correspondiente a la etiqueta <caption> hereda las propiedades heredables de la etiqueta <table>, como por ejemplo la propiedad color. Su anchura también corresponde a la de la tabla.
    • Cuando se define una altura para la tabla, ésta es tratada como altura mínima, porque si el contenido de las celdas así lo fuerza la altura de la tabla puede llegar a ser mayor. En realidad NO se recomienda definir la altura de las tablas, sino que ésta se calcule automáticamente.

    table-layout¶

    Si queremos especificar cómo se debe calcular el ancho de las columnas de la tabla podemos usar la propiedad CSS table-layout:

    Propiedad table-layout
    Valores auto | fixed
    Se aplica a Tablas
    Valor inicial auto
    Heredada No
    Descripción Define el mecanismo de fusión de los bordes de las celdas adyacentes de una tabla

    Valores posibles:

    • auto: La anchura de la tabla y de sus celdas es la necesaria para mostrar su contenido, por lo que unas columnas pueden ser mucho más anchas que otras, dependiendo del contenido que tengan. Además, la tabla puede no llegar a ocupar el 100% del espacio disponible.
    • fixed: La anchura de la tabla y de sus columnas se establecen en base a la anchura explícitamente especificada para la tabla (es necesario indicar la propiedad width para la tabla) y al reparto de dicho espacio realizado entre las celdas de la primera fila de la tabla. Si no se especifica explícitamente la anchura en ninguna de dichas celdas, se reparte proporcionalmente la anchura de la tabla entre las celdas. Si para alguna celda se indica una determinada anchura, ésta es respectada, y la anchura restante en la tabla es repartida propocionalmente entre las celdas para las que no se ha especificado anchura alguna.

    border-collapse y border-spacing¶

    CSS define dos modelos diferentes para el tratamiento de los bordes de las celdas. La propiedad que permite seleccionar el modelo de bordes es border-collapse:

    Propiedad border-collapse
    Valores collapse | separated | inherit
    Se aplica a Tablas
    Valor inicial separate
    Heredada Sí
    Descripción Define el mecanismo de fusión de los bordes de las celdas adyacentes de una tabla

    El modelo collapse fusiona de forma automática los bordes de las celdas adyacentes, mientras que el modelo separate fuerza a que cada celda muestre sus cuatro bordes. Por defecto, los navegadores utilizan el modelo separate, aunque en general los diseñadores prefieren el modelo collapse porque estéticamente resulta más atractivo. Veamos un ejemplo:

    table {
  border-collapse: collapse;
}

    Si se opta por el modelo separate (que es el que se aplica si no se indica lo contrario) se puede utilizar la propiedad border-spacing para controlar la separación entre los bordes de cada celda.

    Propiedad border-spacing
    Valores unidad_medida unidad_medida?
    Se aplica a Tablas
    Valor inicial 0
    Heredada Sí
    Descripción Establece la separación entre los bordes de las celdas adyacentes de una tabla

    Si solamente se indica como valor una medida, se asigna ese valor como separación horizontal y vertical. Si se indican dos medidas, la primera es la separación horizontal y la segunda es la separación vertical entre celdas.

    NO debemos pensar que establecer border-spacing:0 produce el mismo efecto visual que border-collapse: collapse, ya que lo que ocurrirá es que se mostrarán los bordes con una anchura doble (correspondiente a los dos bordes pegados sin separación).

    Si optamos por el modo collapse, debemos tener en cuenta lo siguiente:

    Si se especifica un padding para la tabla éste NO se aplica. Lo que sí pueden tener es margin. De cada dos bordes adyacentes siempre prevalece el más interesante. Para determinarlo se siguen una serie de reglas de precedencia, indicadas a continuación de más a menos prioridad: border-style: hidden, más ancho, double, solid, dashed, dotted, etc, color celda, color fila, etc., el más alejado a la esquina superior izquierda.

    caption-side¶

    Como hemos visto, el título de las tablas se establece mediante el elemento <caption>, que por defecto se muestra encima de los contenidos de la tabla. La propiedad caption-side permite controlar la posición del título de la tabla.

    Propiedad caption-side
    Valores top | bottom | inherit
    Se aplica a Tablas
    Valor inicial top
    Heredada Sí
    Descripción Establece la posición del título de la tabla

    El valor bottom indica que el título de la tabla se debe mostrar debajo de los contenidos de la tabla. Si también se quiere modificar la alineación horizontal del título, debe utilizarse la propiedad text-align. Por ejemplo:

    caption {
  caption-side: bottom;
  text-align: center;
}

    empty-cells¶

    La propiedad empty-cells permite establecer el tratamiento que deben recibir las celdas vacías de una tabla. Una celda vacía es aquella que no tiene ningún contenido, ni siquiera un espacio en blanco o un &nbsp.

    Propiedad empty-cells
    Valores show | hide | inherit
    Se aplica a Celdas de una tabla
    Valor inicial show
    Heredada Sí
    Descripción Define el mecanismo utilizado para el tratamiento de las celdas vacías de una tabla

    El valor hide indica que las celdas vacías no debe mostrar ni borde ni fondo. Por ejemplo:

    table {
  empty-cells: hide;
}

    Un aspecto muy importante de esta propiedad es que sólo se aplica cuando el modelo de bordes de la tabla es de tipo separate. Si queremos conseguir el mismo efecto en tablas donde hayamos establecido border-collapse: collapse, tendremos que hacerlo explícitamente haciendo uso de la pseudoclase :empty (que selecciona un elemento vacío), de la siguiente forma:

    th:empty, td:empty {
  border: none;
  background-color: transparent;
}

    Alineación vertical del contenido de las celdas¶

    Por defecto, el contenido de las celdas está centrado vertical atendiendo a la altura de la misma, que a su vez es calculada por defecto con la altura correspondiente a la celda más alta de la fila. Si queremos variar este comportamiento podemos usar la propiedad vertical-align (que estudiaremos más en profundidad cuando hablemos del posicionamiento estático o normal) y asignarle el valor top, middle (valor por defecto) o bottom. Por ejemplo:

    tbody td {
  vertical-align: top;
}

    Tabla rayada¶

    Un efecto visual muy extendido en las tablas es hacer que las filas pares tengan un color de fondo distinto que las filas impares. Para conseguir este efecto se hace uso de el selector de pseudoclase estructural tr:nth-child(2n), que seleccionará sólo las filas pares, y/o tr:nth-child(2n+1), que seleccionará sólo las filas impares. Por ejemplo:

    tbody tr:nth-child(2n) {
  background-color: #CCC;
}

    Scroll horizontal¶

    Las tablas de datos son, por su naturaleza, uno de los elementos que peor se adaptan a los distintos tamaños de pantalla. Por ese motivo, si se visualiza nuestra tabla en un viewport estrecho es muy probable que la tabla no puede ser visualizada en toda su anchura. La solución más simple consiste en hacer que en ese caso se muestre una barra de scroll que permita al usuario desplazarse hacia los datos de la tabla que hayan quedado ocultos. Para conseguir este efecto, tendremos que incluir nuestro elemento <table> dentro de una etiqueta <div> que actúe como contenedor de la tabla, y establecer la propiedad overflow-x: auto para dicho contenedor. Así el marcado HTML seguiría el siguiente esquema:

    <div class="contenedor-tabla">
  <table>
  ...
  </table>
</div>

    .contenedor-tabla {
  overflow-x: auto;
}

    Hover sobre la tabla¶

    Otro de los efectos visuales más habituales en las tablas es hacer que conforme se pasa el ratón por una fila, ésta cambie de color de fondo (y puede que de color del texto). Para conseguirlo basta con usar la pseudoclase :hover sobre el elemento adecuado. Por ejemplo:

    tbody tr:hover {
  background-color: purple;
  color: white;
}

    Cuando usamos esta técnica con tablas donde hay celdas que tiene definido rowspan se nos presenta un problema, y es que cuando nos situamos sobre una celda que está contenida en la fila siguiente a una fila que contiene celdas con rowspan, dichas celdas no cambiarán su fondo, ya que no se encuentran en la misma fila que la celda sobre la que nos encontramos, y sin embargo, visualmente parece que dichas celdas pertenecen a ambas filas.

    Para solucionar este inconveniente, agruparemos dichas filas en un elemento <tbody> y aplicaremos, por ejemplo, la siguiente regla CSS:

    /* Selecciona las celdas que tienen atributo rowspan
dentro de un tbody sobre el que estamos situados, y
las filas sobre las que nos encontramos */
tbody:hover td[rowspan], tbody tr:hover {
  background-color: purple;
  color: white;
}
