    3 Listas¶

    Introducción¶

    En ocasiones, es posible agrupar determinadas palabras o frases en un conjunto de elementos que tienen más significado de forma conjunta. El menú de navegación de un sitio web por ejemplo está formado por un grupo de palabras. Aunque cada palabra por separado tiene sentido, de forma conjunta constituyen el menú de navegación de la página, por lo que su significado conjunto es mayor que por separado.

    El lenguaje HTML define tres tipos diferentes de listas para agrupar los elementos: listas no ordenadas (se trata de una colección simple de elementos en la que no importa su orden), listas ordenadas (similar a la anterior, pero los elementos están numerados y por tanto, importa su orden) y listas de definición (un conjunto de términos y definiciones similar a un diccionario).

    Etiquetas ul¶

    Las listas no ordenadas son las más sencillas y las que más se utilizan. Una lista no ordenada es un conjunto de elementos relacionados entre sí pero para los que no se indica un orden o secuencia determinados. La etiqueta <ul>, de ***u***nordered ***l***ist, encierra todos los elementos de la lista y la etiqueta <li>, de ***l***ist ***i**tem cada uno de sus elementos. Veamos un ejemplo:

    <ul>
  <li>Inicio</li>
  <li>Noticias</li>
  <li>Artículos</li>
  <li>Contacto</li>
</ul>

    El navegador por defecto muestra los elementos de la lista tabulados y con una pequeña viñeta formada por un círculo negro. Como ya se sabe, el aspecto con el que se muestran los elementos de las listas se puede modificar mediante las hojas de estilos CSS.

    Las etiquetas <ul> corresponden a elementos en bloque.

    Etiqueta ol¶

    Las listas ordenadas son casi idénticas a las listas no ordenadas, salvo que en este caso los elementos relacionados se muestran siguiendo un orden determinado. Cuando se crea por ejemplo una lista con las instrucciones de un producto, es importante el orden en el que se realiza cada paso. Cuando se muestra un índice o tabla de contenidos en un libro, es importante el orden de cada elemento del índice.

    En todos estos casos, la lista más adecuada es la lista ordenada, que se define mediante la etiqueta <ol>. Los elementos de la lista se definen mediante la etiqueta <li>, la misma que se utiliza en las listas no ordenadas. Veamos un ejemplo:

    <ol>
  <li>Enchufar correctamente</li>
  <li>Comprobar conexiones</li>
  <li>Encender el aparato</li>
</ol>

    El navegador muestra la lista de forma muy parecida a las listas no ordenadas, salvo que en este caso no se emplean viñetas gráficas en los elementos, sino que se numeran de forma consecutiva. El tipo de numeración empleada también se puede modificar aplicando hojas de estilos CSS a los elementos de la lista.

    En la etiqueta <ol> podemos usar los siguientes atributos:

    • reversed: Para que se numeren en orden de mayor a menor.
    • start="numero": Para indicar el valor inicial.
    • type="tipo_numeracion": Para el indicar el tipo de numeración (decimal, latin, greek, etc.). Sus valores son similares a los del atributo CSS list-style-type que veremos a continuación. Por su parte, cuando usamos la etiqueta <li> dentro de un elemento <ol> podemos establecer para las etiqueta <li> el atributo value="valor" para indicar el valor de orden que debe mostrar el <ol> para dicho elemento.

    Los elementos <ol> son bastante menos usados que los elementos <ul> dado que mediante CSS es posible hacer que un <ul> luzca como un <ol>. Además la etiqueta <ul> es usada normalmente para representar los menús.

    Las etiquetas <ol> y <li> corresponden a elementos en bloque.

    Etiquetas dl, dt y dd¶

    La etiqueta <dl> permite crear listas de definición (definition list, dl), que no es más que una lista de pares término-definición. Así, una etiqueta <dl> contendrá una serie de pares de etiquetas <dt> , que sirve para indicar el término (definition term, dt), y <dd> , que sirve para indicar el significado o definición (definition definition, dd).

    Veamos un ejemplo:

    <h3>Siglas de partidos políticos</h3>
<dl>
  <dt>PSOE</dt>
  <dd>Partido Socialista Obrero Español</dd>
  <dt>PP</dt>
  <dd>Partido Popular</dd>
  <dt>IU</dt>
  <dd>Izquierda Unida</dd>
</dl>

    Estilizado básico de listas¶

    list-style-type¶

    CSS define varias propiedades para controlar el tipo de viñeta que muestran las listas, además de poder controlar la posición de la propia viñeta. La propiedad básica es la que controla el tipo de viñeta que se muestra y que se denomina list-style-type.

    Propiedad list-style-type
    Valores disc | circle | square | decimal | decimal-leading-zero | lower-roman | upper-roman | lower-greek | lower-latin | upper-latin | armenian | georgian | lower-alpha | upper-alpha | none | inherit
    Se aplica a Elementos de una lista
    Valor inicial disc
    Heredada Sí
    Descripción Permite establecer el tipo de viñeta mostrada para una lista

    En primer lugar, el valor none permite mostrar una lista en la que sus elementos no contienen viñetas, números o letras. Se trata de un valor muy utilizado, ya que es imprescindible para los menús de navegación creados con listas, como se verá más adelante.

    El resto de valores de la propiedad list-style-type se dividen en tres tipos: gráficos, numéricos y alfabéticos. Veamos los más habituales:

    • Gráficos: disc (disco relleno), circle (círculo) y square (cuadrado).
    • Numéricos: decimal (numérico decimal), decimal-leading-zero (numérico decimal con 0 inicial para conservar el mismo número de dígitos), lower-roman (números romanos en minúscula), upper-roman (números romanos en mayúscula).
    • Alfanuméricos: lower-latin (letras en minúscula), upper-latin (letras en mayúscula). Por ejemplo:
    ul {
  list-style-type: square;
}
ol {
  list-style-type: decimal-leading-zero;
}

    list-item-position¶

    Por otra parte, la propiedad list-item-position permite controlar la colocación de las viñetas.

    Propiedad list-item-position
    Valores inside | outside | inherit
    Se aplica a Elementos de una lista
    Valor inicial outside
    Heredada Sí
    Descripción Permite establecer la posición de la viñeta de cada elemento de una lista

    Por defecto, la viñeta se encuentra fuera del margen en el que se muestra cada elemento de la lista, correspondiente a valor outside. Si establecemos para la lista esta propiedad al valor inside la viñeta se encuentra dentro de la primera línea de cada elemento. Veamos una demostración:

    list-style-image¶

    Cuando se requiere personalizar el aspecto de las viñetas, se debe emplear la propiedad list-style-image, que permite mostrar una imagen propia en vez de una viñeta automática.

    Propiedad list-style-image
    Valores url | none | inherit
    Se aplica a Elementos de una lista
    Valor inicial none
    Heredada Sí
    Descripción Permite reemplazar las viñetas automáticas por una imagen personalizada

    Las imágenes personalizadas se indican mediante la URL de la imagen. Si no se encuentra la imagen o no se puede cargar, se muestra la viñeta automática correspondiente (salvo que explícitamente se haya eliminado mediante la propiedad list-style-type). Por ejemplo:

    list-style¶

    Como es habitual, CSS define una propiedad de tipo shorthand que permite establecer todas las propiedades de una lista de forma directa. La propiedad se denomina list-style.

    Propiedad list-style
    Valores (list-style-type list-style-position list-style-image) | inherit
    Se aplica a Elementos de una lista
    Valor inicial -
    Heredada Sí
    Descripción Propiedad que permite establecer de forma simultánea todas las opciones de una lista

    El orden en el que se indican los valores de la propiedad es indiferente. Por ejemplo:

    ul {
  list-style: url("imagenes/cuadrado_rojo.gif") square;
}

    Estilizando menú de navegación¶

    Como ya hemos visto anteriormente, los menús de navegación se suelen crear con las etiquetas <nav>, <ul>, <li> y <a> con una estructura parecida a la siguiente:

    <nav>
    <ul>
      <li><a href="#">Item 1</a></li>
      <li><a href="#">Item 2</a></li>
      <li><a href="#">Item 3</a></li>
    </ul>
</nav>

    Para aplicar un estilo de menú a esta lista desordenada debemos tener en cuenta una serie de aspectos:

    • La lista desordenada <ul> no debe mostrar ninguna viñeta, por lo que le estableceremos list-style-type: none;. Además, debemos tener en cuenta que el navegador por defecto aplica un determinado y margen a las listas, por lo que deberemos resetear dicho comportamiento. Por ejemplo:

      nav ul {
  margin: 0; /* reset styles */
  padding: 0; /* reset styles */
  list-style-type: none;
  ...
}

    • Las etiqueta <a> (enlaces) dentro de la lista no deben tener ninguna línea decorativa y deben ocupar todo el elemento <li>, por lo que deberemos convertirlos en bloque (ya que por defecto los <a> son elementos en línea). Por ejemplo:

      css nav ul li a { display: block; text-decoration: none; ... }
