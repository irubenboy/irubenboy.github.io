    4 Generador de contenido con CSS¶

    Introducción¶

    En algunas ocasiones, el desarrollador quiere que el navegador muestre contenido que en realidad no se encuentra en el documento HTML. Por ejemplo, supongamos que queremos que el navegador inserte la palabra "Figura " delante del contenido del pie de cada imagen del documento HTML, o que añada la palabra "Tema " antes de cada elemento <h1>.

    Otro ejemplo característico es el caso de las listas numeradas, es decir, de los elementos <li> contenidos dentro de un elemento <ol>. El navegador añadirá el número ordinal correspondiente a cada ítem de la lista, aunque en realidad dicho número no se encuentre dentro del documento HTML. En este caso, dicho comportamiento está definido por el valor list-item de la propiedad display, que es asignado a los elementos <li> en la hoja de estilos por defecto del navegador.

    Para el resto de los ejemplos planteados, CSS nos proporciona la propiedad content, que permite establecer el contenido que se quiere añadir. El problema es que no queremos modificar el contenido de elemento, por lo que no nos basta con seleccionar dicho elemento y aplicarle un valor determinado a su propiedad content, sino que lo queremos es poder añadir un contenido antes o después del elemento. Y ahí es donde entran en juego los pseudoelementos ::before y ::after.

    Pseudoelementos ::before y ::after¶

    Los pseudo-elementos :before y :after permiten seleccionar, respectivamente, la "posición" anterior y posterior de un elemento. Una vez seleccionada la "posición", normalmente se usa la propiedad content para establecer el contenido de dicha "posición", de manera que dicho contenido se añade antes o después del elemento. De esta forma, desde CSS podemos generar contenido en nuestra página que no se encuentra realmente en el documento HTML.

    Los pseudo-elementos ::before y ::after se utilizan en combinación con la propiedad content de CSS para añadir contenidos antes o después del contenido original de un elemento. Por ejemplo, las siguientes reglas CSS añaden el texto "Nota: " delante de los párrafos de la clase nota y el carácter . detrás de cada párrafo de la página:

    p.nota::before { content: "Nota: "; }
p::after   { content: "."; }

    Los pseudoelementos ::before y ::after heredan todas las propiedades heredables desde el elemento al que están asociados. Las propiedades que no son heredables, reciben el valor por defecto correspondiente. En otras palabras, los contenidos insertados dinámicamente en un elemento son a todos los efectos parte de ese mismo elemento, por lo que heredan el valor de todas sus propiedades CSS heredables.

    Por otra parte, estos pseudoelementos afectan a las elementos de su entorno exactamente igual que si fueran elementos reales insertados dentro del elemento asociado, teniendo en cuenta que, por defecto, son considerados elementos en línea.

    content¶

    La propiedad content se emplea para generar nuevo contenido de forma dinámica e insertarlo en la página HTML. El estándar CSS indica que la propiedad content sólo puede utilizarse en los pseudo-elementos ::before y ::after.

    Propiedad content
    Valores Combinación de counter() counters() attr() open-quote close-quote no-open-quote no-close-quote | normal | none | inherit
    Se aplica a Solamente a los pseudo-elementos ::before y ::after
    Valor inicial normal
    Descripción Establece el contenido before y after de un elemento

    Los valores normal y none tienen el mismo efecto, hacen que el pseudoelemento no se genere, por lo que si queremos que se genere aunque no muestre ningún texto tendremos que establecerle como valor una cadena vacía "".

    Si establecemos como valor una cadena de texto, éste tiene que ir entre comillas simples o dobles y sólo permite incluir texto básico. Si se incluye alguna etiqueta HTML en la cadena de texto, el navegador muestra la etiqueta tal y como está escrita, ya que no la interpreta. Para incluir un salto de línea en el contenido generado, se utiliza el carácter especial \A. Para incluir una comilla doble o simple usar \" o \'. Veamos un ejemplo:

    También podemos establecer como valor una URI, que normalmente suele corresponder a la URL de una imagen que se quiere añadir de forma dinámica al contenido. Por ejemplo:

    En otras ocasiones, lo que queremos añadir antes o después del contenido del elemento es el valor de alguno de sus atributos, para lo que podemos usar como valor de la propiedad content la función especial attr(nombre_atributo), que retornará el valor de dicho atributo en el elemento. Desgraciadamente esta función especial sólo se puede utilizar dentro de la propiedad content.

    Por ejemplo, con la siguiente regla hacemos que se añada automáticamente a los elementos <abbr> el valor del atributo title (en el que se indica el significado de la abreviatura o siglas correspondientes).

    abbr::after {
  content: " (" attr(title) ")"
}

    Veamos una demostración en la que se añade al contenido el valor del atributo href de los enlaces:

    Otros valores posibles están relacionados con las comillas y la propiedad quotes que veremos más adelante. Por ejemplo, open-quote y close-quote indican que se debe mostrar, respectivamente, una comilla de apertura o una de cierre. El tipo de comillas a mostrar en cada caso se establece mediante la propiedad quotes. El siguiente ejemplo añade automáticamente las comillas de apertura y cierre a los elementos <blockquote>:

    blockquote::before {
  content: open-quote;
}
blockquote::after {
  content: close-quote;
}

    Por su parte, los valores no-open-quote y no-close-quote no escriben contenido, sino que incrementan o decrementan, respectivamente, el nivel de comillas anidadas. Lo entenderemos mejor que veamos la propiedad quotes.

    También tenemos otros valores relacionados con los contadores, como counter(contador) y counters(contador, separador). El primero representa el valor de un contador en el elemento, y el segundo, la concatenación del valor de un determinado contador para todos los antepasados del elemento y el propio elemento, separados por la cadena indicada como separador. Por ejemplo:

    h1::before {
  content: "Capítulo " counter(numero_capitulo);
}

    Veremos más ejemplos de estas funciones cuando hablemos de las propiedades counter-reset y counter-increment.

    quotes¶

    La propiedad quotes permite establecer de forma explícita los caracteres que se utilizan al añadir comillas en la propiedad content. Sólo tiene sentido cuando se utiliza junto con la propiedad content y los pseudoelementos ::after y ::before.

    Propiedad quotes
    Valores | none | inherit
    Se aplica a Solamente a los pseudo-elementos ::before y ::after
    Valor inicial Depende de cada navegador
    Heredada Sí
    Descripción Establece los caracteres utilizados para mostrar las comillas dentro de la propiedad content

    La propiedad quotes permite indicar un número ilimitado de pares de comillas. Cuando sólo se define un par, se muestran siempre que se utilice open-quote y close-quote dentro del valor de la propiedad content. Cuando se definen varios pares, se van alternando cada vez que se muestran unas comillas dentro de otro elemento que ya tiene comillas. Cada comilla se separa de las demás mediante un espacio en blanco. Por ejemplo:

    Contadores¶

    Los contadores son variables numéricas que podemos definir en nuestro CSS para posteriormente incluir su valor en la propiedad content de algún pseudoelemento que hayamos seleccionado con ::before o ::after.

    Para definir un contador se usa la propiedad counter-reset

    Propiedad counter-reset
    Valores none | inherit
    Se aplica a Todos los elementos
    Valor inicial none
    Descripción Crea e inicializa el valor de una variable contador

    En el caso más sencillo, la propiedad counter-reset recibe el nombre de un contador seguido, opcionalmente, por un número entero que indica el valor al que se inicializa el contador. Si no se indica el valor, se asume el valor 0. Si el contador no existía ya en nuestro CSS, se crea. Si ya existía se reinicializa. Por ejemplo:

    body {
  counter-reset: numero_capitulo 0;
}

    La regla CSS anterior hace que se inicialice a 0 un contador llamado numero_capitulo cuando el navegador encuentre el elemento <body>. Si dicho contador no existía, se crea.

    El contador creado será utilizado posteriormente en las propiedades content y counter-increment.

    Como comentamos anteriormente, para usar el valor de un contador dentro de la propiedad content se usa la función counter(nombre_contador, estilo). Además el contador a mostrar, podemos indicar, opcionalmente, el estilo con el que se muestra el valor del contador. La lista de estilos permitidos son los mismos que los de la propiedad list-style-type. El estilo por defecto es decimal.

    body {
  counter-reset: numero_capitulo 0;
}
h1::before {
  content: "Capítulo " counter(numero_capitulo);
  counter-increment: numero_capitulo 1;
}

    La propiedad counter-increment sirve para incrementar el valor de un contador la cantidad indicada. Un aspecto a tener en cuenta es que cuando se inicializa y se actualiza el valor de un contador en un mismo elemento, primero se inicializa su valor y después se actualiza según la propiedad counter-increment.

    Propiedad counter-increment
    Valores |none | inherit
    Se aplica a Todos los elementos
    Valor inicial none
    Descripción Incrementa el valor de una variable contador

    Si no se indica el incremento, los navegadores suponen que vale 1.

    Las propiedades counter-reset y counter-increment también permiten indicar varios contadores para inicializarlos o incrementarlos de forma simultánea. Por ejemplo:

    h1 {
  counter-reset: numero_seccion 0 numero_tabla -1 numero_imagen -1;
}

    La regla CSS anterior inicializa a 0 el contador numero_seccion y a -1 los contadores numero_tabla y numero_imagen cada vez que se encuentra un elemento <h1> en la página.

    Uno de los aspectos más complejos de los contadores es que éstos tienen en cuenta el anidamiento de los elementos HTML. Por ejemplo, si tenemos las siguientes reglas CSS:

    ol {
  counter-reset: elemento;
  list-style-type: none;
}
li::before {
  content: counter(elemento) ". ";
  counter-increment: elemento;
}

    y el siguiente código HTML en el que se anidan elementos <ol> dentro de los elementos <li>:

    <ol>
  <li>Elemento</li>
  <li>Elemento</li>
  <li>Elemento
    <ol>
        <li>Elemento</li>
        <li>Elemento
          <ol>
            <li>Elemento</li>
            <li>Elemento</li>
            <li>Elemento</li>
          </ol>
        </li>
        <li>Elemento</li>
    </ol>
  </li>
  <li>Elemento</li>
</ol>

    el navegador crea tres contadores diferentes con el mismo nombre (elemento), uno por cada vez que se encuentra un elemento <ul>. La función counter() sólo mostrará el valor del contador correspondiente a dicho nivel de anidación, por lo que dentro de cada <ul> se muestra la numeración normal (1, 2 y 3).

    Pero ¿Y si en el ejemplo anterior queremos mostrar la concatenación de los valores de los tres contadores anidados con el mismo nombre, por ejemplo 1.1.1 , 1.1.2, etc.? No podremos utilizar la función counter(), sino que utilizaremos la función counters(nombre_contador, cadena_separacion, estilo), que recibe el nombre del contador, y la cadena que se debe emplear como separador entre los distintos valores a la hora de realizar la concatenación y, opcionalmente, el estilo con el que deben mostrarse los contadores. counters() obtendrá los distintos contadores anidados con dicho nombre los mostrará en orden, con el estilo que se le haya indicado, escribiendo entre ellos la cadena de separación indicada.

    Así si variamos la regla CSS en el ejemplo anterior:

    ol {
  counter-reset: elemento;
  list-style-type: none;
}
li::before {
  content: counters(elemento, '.') ".- ";
  counter-increment: elemento;
}

    los elementos <li> se mostrarán con la numeración 1.1.1., 1.1.2. y así sucesivamente.

    El motivo para este comportamiento es que los elementos <li> anidados se ven afectados por varios contadores llamados elemento. La función counter() sólo muestra el valor del contador que afecta más directamente al elemento, mientras que la función counters() muestra todos los contadores empezando desde el más externo hasta llegar al más interno.
