    1 Introducción al modelo de cajas¶

    Introducción¶

    El modelo de cajas (box model) es seguramente la característica más importante del lenguaje de hojas de estilos CSS, ya que condiciona el diseño de todas las páginas web. El modelo de cajas es el comportamiento de CSS que hace que todos los elementos de las páginas se representen mediante cajas rectangulares.

    Las cajas de una página se crean automáticamente. Cada vez que se inserta una etiqueta HTML, se crea una nueva caja rectangular que encierra los contenidos de ese elemento. La siguiente imagen muestra las tres cajas rectangulares que crean las tres etiquetas HTML que incluye la página:

    Las cajas de las páginas no son visibles a simple vista porque inicialmente no muestran ningún color de fondo ni ningún borde.

    Los navegadores crean y colocan las cajas de forma automática, pero CSS permite modificar todas sus características. Cada una de las cajas está formada por seis partes, tal y como muestra la siguiente imagen:

    Las partes que componen cada caja y su orden de visualización desde el punto de vista del usuario son las siguientes:

    • Contenido (content): se trata del contenido HTML del elemento (las palabras de un párrafo, una imagen, el texto de una lista de elementos, etc.)
    • Relleno (padding): espacio libre opcional existente entre el contenido y el borde.
    • Borde (border): línea que encierra completamente el contenido y su relleno.
    • Imagen de fondo (background image): imagen que se muestra por detrás del contenido y el espacio de relleno.
    • Color de fondo (background color): color que se muestra por detrás del contenido y el espacio de relleno.
    • Margen (margin): separación opcional existente entre la caja y el resto de cajas adyacentes. El relleno y el margen son transparentes, por lo que en el espacio ocupado por el relleno se muestra el color o imagen de fondo (si están definidos) y en el espacio ocupado por el margen se muestra el color o imagen de fondo de su elemento padre (si están definidos). Si ningún elemento padre tiene definido un color o imagen de fondo, se muestra el color o imagen de fondo de la propia página (si están definidos).

    Si una caja define tanto un color como una imagen de fondo, la imagen tiene más prioridad y es la que se visualiza. No obstante, si la imagen de fondo no cubre totalmente la caja del elemento o si la imagen tiene zonas transparentes, también se visualiza el color de fondo. Combinando imágenes transparentes y colores de fondo se pueden lograr efectos gráficos muy interesantes.

    Ejercicio

    Realiza una página web que muestre una etiqueta padre div que contenga en su interior otra etiqueta div. Haz que tanto el padre como el hijo tengan una anchura de 200px. El div hijo debe mostrar el texto Pulsa. El color de fondo del div padre debe ser distinto del div hijo. Agrega un relleno de 10px al div padre. Centra horizontalmente el texto en el div hijo.

    Hazte las siguientes preguntas:

    ¿Por qué el espacio entre el borde y el contenido del div hijo se ve del color de fondo de dicha caja? ¿Qué anchura real tiene el div hijo? ¿Qué anchura real tiene el div padre? ¿Qué altura tiene el div hijo? ¿Qué altura tiene el div padre?

    Realiza el mismo ejercicio anterior, pero asegurando que la anchura real del div padre sea 200px

    Modelo de cálculo de anchura y altura¶

    La propiedad box-sizing se usa para establecer la forma de calcular la anchura y la altura de la caja que representa a un elemento.

    Propiedad box-sizing
    Valores content-box | border-box | inherit
    Se aplica a Todos los elementos
    Valor inicial content-box
    Descripción Establece la forma de calcular el alto y el ancho de los elementos

    Podemos especificar uno de los siguientes valores:

    • content-box: Determina que las propiedades width y height, ancho y alto respectivamente, corresponden a la anchura y altura solo del contenido de la caja, pero no incluyen el relleno (padding), el borde (border) o el margen (margin). Por tanto la anchura real de la caja será width + padding + borde y su altura real será height + padding + borde. Por ejemplo, si especificamos la siguiente regla CSS:

      .box{
    width: 300px;
    height: 200px;
    border: 2px solid black;
    padding: 10px;
}

      cualquier elemento de la clase box tendrá:

      • una anchura de caja de 324px, calculada mediante la siguiente operación: 2px (border-left) + 10px (padding-left) + 300px (width) + 10px (padding-right) + 2px (border-right)
      • una altura de caja de 224px, calculada mediante la siguiente operación: 2px (border-top) + 10px (padding-top) + 200px (height) + 10px (padding-bottom) + 2px (border-bottom).

    • border-box: Los valores especificados para las propiedades width y height corresponden al tamaño de la propia caja, no al tamaño del contenido, y por tanto incluyen el relleno (padding) y el borde (border), pero no el margen (margin). Así, si especificamos la siguiente regla CSS:

      .box {
  box-sizing: border-box;
  width: 300px;
  height: 200px;
  border: 2px solid black;
  padding: 10px;
}

      cualquier elemento de la clase box tendrá:

      • Una anchura de caja de 300px, de manera que la anchura del contenido será de 276px, calculada mediante la siguiente operación:300px - 2px (border-left) - 10px (padding-left) - 10px (padding-right) - 2px (border-right)
      • Una altura de caja de 200px, de manera que la altura del contenido será de 176px, calculada mediante la siguiente operación: 200px - 2px (border-top) - 10px (padding-top) - 10px (padding-bottom) - 2px (border-bottom).

    El problema del valor content-box es que el diseñador tendrá que hacer cálculos matemáticos para asegurarse de que las cajas caben en la posición deseada, dado que el tamaño real de la caja es mayor que el especificado en la propiedades width y height. Entonces ¿por qué es el valor por defecto? El motivo es la compatibilidad con versiones anteriores de CSS. La propiedad box-sizing fue añadida posteriormente en la especificación CSS 3 y en CSS 2 todas las cajas utilizaban la forma de cálculo descrita en el valor content-box.

    Si queremos resetear el comportamiento por defecto para todas las cajas, usaremos las siguientes reglas para hacer que el elemento raíz use el valor border-box y el resto de elementos hereden este comportamiento:

    html {
  box-sizing: border-box;
}

*, *:before, *:after {
  box-sizing: inherit;
}
    Ejercicio

    Realiza el mismo ejercicio anterior, pero sin tener que hacer ningún cálculo matemático

    Anchura¶

    La propiedad width establece la anchura del elemento. Si el valor de la propiedad box-sizing tiene el valor content-box, el valor indicado para width hace referencia a la anchura del contenido, no de la caja en sí. Si, por el contrario, la propiedad box-sizing tiene el valor border-box, el valor indicado para width hace referencia a la anchura de la caja, incluyendo el contenido, el relleno (padding) y el borde.

    Propiedad width
    Valores unidad de medida | porcentaje | auto
    Se aplica a Todos los elementos, salvo los elementos que no sean imágenes, las filas de las tablas y los grupos de filas de tablas
    Valor inicial auto
    Descripción Estable la anchura de un elemento

    La propiedad width no admite valores negativos y los valores en porcentaje se calculan a partir de la anchura de su elemento padre. El valor inherit indica que la anchura del elemento se hereda de su elemento padre. El valor auto, que es el que se utiliza si no se establece de forma explícita un valor a esta propiedad, indica que el navegador debe calcular automáticamente la anchura del elemento, teniendo en cuenta sus contenidos, si se trata de un elemento en bloque o en línea y el sitio disponible en la página.

    Es muy importante resaltar que la propiedad width NO tendrá efecto sobre los elementos en línea, excepto la etiqueta <img>, así como las filas de las tablas.

    Algunas veces querremos establecer que la anchura de un elemento sea variable y se adapte a la anchura de la ventana de visualización pero respetando ciertos límites, es decir, que no sea tan estrecho como para que no se puedan mostrar correctamente su contenido y tampoco sea tan ancho como para que el contenido quede mal estéticamente. Para estos casos, podemos establecer un valor relativo para la propiedad width, por ejemplo un porcentaje o un valor expresado en vw, en conjunción con las propiedades min-width y/o max-width, que permiten establecer, respectivamente, la anchura mínima y máxima del elemento. El valor asignado a la propiedad box-sizing afecta a estas propiedades de manera similar a como afecta a width.

    Propiedad min-width
    Valores unidad de medida | porcentaje
    Se aplica a Todos los elementos salvo filas y grupos de filas de tablas
    Valor inicial 0
    Descripción Permite definir la anchura mínima de un elemento
    Propiedad max-width
    Valores unidad de medida | porcentaje | inherit
    Se aplica a Todos los elementos salvo filas y grupos de filas de tablas
    Valor inicial none
    Descripción Permite definir la anchura máxima de un elemento

    Si especificamos para la propiedad width un valor absoluto, por ejemplo en px, el valor establecido para las propiedades min-width o max-width no tendrá ningún efecto, ya que se ha establecido una anchura fija para el elemento. De igual forma, si especificamos el valor de width en porcentaje y el de min-width o max-width también en porcentaje, NO se producirá el resultado esperado. Lo adecuado es definir el valor de width como un porcentaje y el de min-width y/o max-width como un valor absoluto, por ejemplo en px.

    No debemos olvidar que las propiedades min-width y max-width no son heredadas por defecto, ya que sus valores iniciales no son inherit.

    Altura¶

Si el valor de la propiedad box-sizing tiene el valor content-box, el valor indicado para height hace referencia a la altura del contenido, no de la caja en sí. Si, por el contrario, la propiedad box-sizing tiene el valor border-box, el valor indicado para height hace referencia a la altura de la caja, incluyendo el contenido, el relleno (padding) y el borde.

    Valores unidad de medida | porcentaje | auto
    Se aplica a Todos los elementos, salvo los elementos que no sean imágenes, las filas de las tablas y los grupos de filas de tablas
    Valor inicial auto
    Descripción Estable la altura de un elemento

    Al igual que sucede con width, la propiedad height no admite valores negativos. Si se indica un porcentaje, se toma como referencia la altura del elemento padre. Si el elemento padre no tiene una altura definida explícitamente, se asigna el valor auto a la altura.

    Desbordamiento del contenido¶

    Normalmente, los contenidos de un elemento se pueden mostrar en el espacio reservado para ese elemento. Sin embargo, en algunas ocasiones el contenido de un elemento no cabe en el espacio reservado para ese elemento, en cuyo caso se dice que el contenido se desborda.

    La situación más habitual en la que el contenido sobresale de su espacio reservado es cuando se establece la anchura y/o altura de un elemento mediante la propiedad width y/o height. , aunque existen otras situaciones en las que también se produce, como por ejemplo con líneas de texto muy largas contenidas dentro de un elemento <pre>, que hacen que la página entera sea demasiado ancha.

    No debemos olvidar que por defecto el texto se moverá a una nueva línea de forma natural cuando se llegue al final del elemento, por lo que en raras ocasiones el texto producirá desbordamiento de anchura. Sin embargo, si hemos cambiado el valor de la propiedad white-space, como ocurre con las etiquetas <pre>, el texto si podrá producir desbordamiento de anchura.

    Respecto a la altura, si no hemos especificado la altura de la caja mediante height o max-height, el fluir del texto haga que la caja sea simplemente más alta, ya que el valor por defecto para la propiedad height es auto, de manera que la altura de la caja es calculada en base su contenido, imposibilitando el desbordamiento de altura. Sin embargo, si hemos establecido una altura fija para la caja es perfectamente posible que si el texto es extenso se produzca un desbordamiento en altura.

    CSS define las propiedades overflow-x, overflow-y y overflow para controlar la forma en la que se visualizan los contenidos que sobresalen de sus elementos, respectivamente de forma horizontal, vertical y en ambos sentidos.

    Propiedades overflow, overflow-x, overflow-y
    Valores visible | hidden | scroll | | auto | inherit
    Se aplica a Todos los elementos
    Valor inicial visible
    Descripción Define la forma en que se visualiza el contenido que sobresale de un elemento, debido a que no cabe en él (en el eje x, en el eje y o en cualquiera de sus ejes)

    Analicemos los posibles valores:

    • visible: el contenido no se corta y se muestra sobresaliendo de la zona reservada para visualizar el elemento. Este es el comportamiento por defecto.
    • hidden: el contenido sobrante se oculta y sólo se visualiza la parte del contenido que cabe dentro de la zona reservada para el elemento.
    • scroll: solamente se visualiza el contenido que cabe dentro de la zona reservada para el elemento, pero también se muestran barras de scroll que permiten visualizar el resto del contenido.
    • auto: el comportamiento depende del navegador, aunque normalmente es el mismo que el valor scroll.
