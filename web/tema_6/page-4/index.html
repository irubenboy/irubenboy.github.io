    4 MediaQueries¶

    Introducción¶

    CSS 3 añadió la posibilidad de crear reglas CSS que sólo se aplicaran en determinadas condiciones, como por ejemplo que la página web se vaya a imprimir, que el dispositivo en el que se vaya a ver tenga una determinada orientación, o que pantalla tenga como mínimo una determinada anchura, etc. De esta manera CSS 3 permite adaptar la representación del contenido a las características del dispositivo en el que se va a visualizar, sin tener que cambiar el contenido en sí.

    Para conseguir esta adaptación, tendremos que usar las llamadas media queries (consultas de medios), que corresponden a la agrupación de una serie de reglas CSS que sólo se usarán cuando nos encontremos en el tipo de dispositivo especificado en la media query y se cumplan las condiciones establecidas en la misma.

    @media <consulta1>, <consulta2> {
​
  /* Reglas CSS a aplicar si se cumple la expresión booleana
  ...
​    */
}

    Cada <consulta> estará formada por:

    • Un media type (tipo de medio): Indica el tipo de dispositivo para el que queremos que se apliquen las reglas CSS contenidas en la media query. Los valores más habituales son: screen, print o tv. Podemos usar el valor all para indicar todos los tipos de medios.
    • Una o más condiciones de características del dispositivo, unidas mediante el operador lógico and. Cada condición tiene el formato (función: valor). Por ejemplo (min-width: 700px) especifica la condición de que el ancho mínimo debe ser de 700px.
    • Podemos usar el operador lógico not para negar toda la consulta. El resultado de la consulta es verdadera si la condición se evalúa a verdadera, atendiendo al tipo de medio especificado, al valor de verdadero o falso retornados por las condiciones y a los operadores lógicos indicados.

    Si especificamos en la media query varias condiciones separadas por , es cómo si aplicáramos un operador or, es decir que las reglas CSS incluidas dentro de la media query se aplicarán si la consulta1 se evalúa a verdadero o si la consulta2 se evalúa a verdadero.

    Veamos ejemplos:

    /* Cualquier tipo de medio cuya anchura sea como mínimo 700px. */
@media (min-width: 700px) {
  /* Reglas CSS.
  ...
    */
}
​
/* Cualquier tipo de medio cuya anchura sea como mínimo 700px y tenga orientación apaisada */
@media (min-width: 700px) and (orientation: landscape) {
  /* Reglas CSS.
  ...
    */
}
​
/* Cualquier medio que no sea una pantalla a color o un medio impreso con color. El not se aplica a not(screen and (color)) */
@media not screen and (color), print and (color) {
  /* Reglas CSS.
  ...
    */
}
​
/* Pantallas con un ancho entre 500px y 800px */
@media screen and (min-width: 500px) and (max-width: 800px) {
  /* Reglas CSS.
  ...
    */
}

    Funciones multimedia¶

    Para especificar las condiciones de las media queries tenemos disponibles distintas funciones multimedia, entre las que destacamos:

    • min-width, max-width: Describe el ancho mínimo / máximo de la superficie a renderizar en el dispositivo de salida. Por ejemplo (min-width: 700px) establece la condición de que el ancho mínimo de renderizado en el dispositivo de salida sea de 700px.
    • orientation: Indica cuando el dispositivo esta en modo apaisado (valor landscape) en el que el ancho de la pantalla es mayor al alto, o modo retrato (valor portrait), en el que el alto de la pantalla es mayor al ancho. Un aspecto que debemos tener en cuenta cuando usamos media queries es que si tenemos varias se aplicarán todas las reglas de todas las media queries que se evalúen a verdadero. Por este motivo debemos elegir bien el orden en el que se establecen las media queries, para hacer que las reglas de las posteriores sobrescriban los comportamientos establecidos por las reglas de las anteriores.

    Técnicas de diseño y media queries¶

    A la hora de diseñar una página responsive podemos seguir dos técnicas de diseño distintas relacionadas con los distintos tipos de dispositivos donde se visualizará nuestra página web: mobile-first (primero el móvil) o desktop-first (primero la pantalla de escritorio).

    La técnica mobile-first consiste en diseñar la web pensando en su apariencia en un dispositivo móvil, tomándolo como diseño base, y posteriormente modificar su apariencia cuando nos encontremos en una tablet o en una pantalla de escritorio, haciendo uso de media queries específicas para la anchura mínima de estos dispositivos.

    Así el código CSS quedaría algo parecido a:

    /* Estilos normales (para móviles en vertical) */
body { /*..*/. }
p    { /*..*/. }
/*..*/.
​
/* Móviles en horizontal o tablets en vertical */
@media (min-width: 768px) {
  /* Reglas CSS que sobrescriben determinados comportamientos anteriores */
  p {
    /*..*/.
  }
  /*..*/.
 }
​
/* Tablets en horizontal y pantallas de escritorio normales */
@media (min-width: 1024px) {
  /* Reglas CSS que sobrescriben determinados comportamientos anteriores */
  p {
    /*..*/.
  }
  /*..*/.
 }
​
/* Pantallas de escritorios muy anchas */
@media (min-width: 1200px) {
  /* Reglas CSS que sobrescriben determinados comportamientos anteriores */
  p {
    /*..*/.
  }
  /*..*/.
 }

    Por su parte la técnica desktop-first consiste en diseñar la web pensando en su apariencia en una pantalla de escritorio, tomándolo como diseño base, y posteriormente modificar su apariencia cuando nos encontremos en una tablet o en un dispositivo móvil, haciendo uso de media queries específicas para la anchura máxima de estos dispositivos.

    Así el código CSS quedaría algo parecido a:

p    { /*...*/ }
/*...*/

@media (min-width: 768px) and (max-width: 1199px) {

  p {
    /*...*/
  }
  /*...*/
 }
​
/* Móviles en horizontal o tablets en vertical */
@media (max-width: 767px) {
  /* Reglas CSS que sobrescriben determinados comportamientos anteriores */
  p {
    /*...*/
  }
  /*...*/
 }
​
/* Móviles en vertical */
@media (max-width: 480px) {
  /* Reglas CSS que sobrescriben determinados comportamientos anteriores */
  p {
    /*...*/
  }
  /*...*/
 }

    Mobile-first vs Desktop-first¶

    El uso de la técnica mobile-first tiene una serie de ventajas:

    • Está 100% optimizado para la experiencia en móviles.
    • Reduce la web a lo absolutamente esencial.
    • Prioriza el contenido frente a los aspectos estéticos, que puede ser lo deseado.

    Sin embargo, también tiene una serie de inconvenientes:

    • La versión para dispositivos de escritorio puede resultar demasiado simple o vacía.
    • Puede ser más complicado su diseño y desarrollo.
    • Hay menos libertad creativa, por lo que puede ser más complicado crear webs que se distingan del resto.
    • Los clientes están acostumbrados a ver como prototipo una versión de escritorio.
    • Puede que los usuarios no visiten habitualmente tu web desde el móvil.

    Entonces ¿qué opción tomamos? Pues depende del proyecto en cuestión, pero independientemente de la opción seleccionada, a la hora de hacer nuestro diseño siempre deberemos tener en cuenta que deberá adaptarse a distintos tamaños de pantalla.

    Breakpoints¶

    Un breakpoint (punto de ruptura) es una cantidad, correspondiente a un ancho concreto, en el que cambiamos el diseño de nuestra web mediante unas reglas CSS específicas que deben aplicarse sólo cuando el viewport tenga al menos esa anchura, en el caso de mobile-first, o hasta dicha anchura, en el caso de desktop-first.

    Un aspecto muy importante del diseño responsive es saber elegir bien nuestros breakpoints. En principio tenemos tres opciones:

    • Elegir como breakpoints anchuras de viewport de los modelos de dispositivos más habituales, como los del iPhone, iPad, etc.
    • Elegir como breakpoints anchuras de viewport en los que se agrupen los tamaños más habituales, para lo que es necesario conocer las estadísticas de tamaño de pantalla. Se recomienda leer el siguiente artículo .
    • Elegir como breakpoints anchuras de viewport para las que nuestro diseño no se muestre adecuado. Este es el enfoque que se recomienda, aunque no siempre es el más usado.
