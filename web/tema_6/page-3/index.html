    3 Formularios¶

    Introducción¶

    HTML es un lenguaje de marcado cuyo propósito principal consiste en estructurar los contenidos de los documentos y páginas web. Sin embargo, HTML también permite crear formularios para que los usuarios interactúen con las aplicaciones web.

    Etiqueta form¶

    Para crear un formulario en HTML debemos crear una etiqueta <form>, que tendrá como hijos todos los controles del formulario (cuadros de texto, botones, etc.).

    La etiqueta <form> se trata de un elemento en bloque, que puede recibir los siguientes atributos:

    • action: URL del destinatario de los datos que se van a enviar desde el formulario, y que se encargará de procesarlos.
    • method: Método HTTP empleado al enviar el formulario, Puede ser GET o POST.
    • enctype: Tipo de codificación empleada al enviar el formulario al destinatario. Sólo se usa este atributo cuando se pretende enviar al destinatario un archivo desde el formulario. Los valores posibles son: application/x-www-form-urlencoded (valor por defecto), multipart/form-data (para envío de ficheros) y text/plain.
    • autocomplete: Indica que el navegador puede autocompletar el campos del formulario. Puede tomar el valor on (activado) u off (desactivado).
    • novalidate: Indica que el navegador NO debe validar los datos del formulario antes de enviarlos. Por defecto si son validados (cuando este atributo no está presente).

    La mayoría de formularios utilizan sólo los atributos action y method. El atributo action indica la URL de la aplicación del servidor que se encarga de procesar los datos introducidos por los usuarios. Esta aplicación también se encarga de generar la respuesta que muestra el navegador.

    El atributo method establece la forma en la que se envían los datos del formulario al servidor. Este atributo hace referencia al método HTTP, por lo que no es algo propio de HTML. Los dos valores que se utilizan en los formularios son GET y POST.

    Al margen de otras diferencias técnicas, el método POST permite el envío de mucha más información que el método GET. En general, el método GET admite como máximo el envío de unos 500 bytes de información. La otra gran limitación del método GET es que no permite el envío de archivos adjuntos con el formulario. Además, los datos enviados mediante GET se ven en la barra de direcciones del navegador (se añaden al final de la URL de la página), mientras que los datos enviados mediante POST no se pueden ver tan fácilmente.

    Si no sabes que método elegir para un formulario, existe una regla general que dice que el método GET se debe utilizar en los formularios que no modifican la información (por ejemplo en un formulario de búsqueda). Por su parte, el método POST se debería utilizar cuando el formulario modifica la información original (insertar, modificar o borrar alguna información).

    El ejemplo más común de formulario con método GET es el de los buscadores. Si realizas una búsqueda con tu buscador favorito, verás que las palabras que has introducido en tu búsqueda aparecen como parte de la URL de la página de resultados.

    Veamos un ejemplo:

    <form action="http://www.informaticasaladillo.es/mostrar.php" method="post" id="formulario">
  <label for="nombre">Escriba su nombre completo:</label>
  <input type="text" id="nombre" name="fullname" value="Baldomero" />
  <input type="submit" value="Enviar" />
</form>

    Etiqueta label¶

    HTML incluye una etiqueta denominada <label> que se utiliza para establecer el título de cada campo del formulario. El único atributo que suele utilizarse con la etiqueta es for, que indica el identificador (atributo id) del campo de formulario del que sirve como título. Así, en el ejemplo anterior, el label sirve como título del campo de formulario con id="nombre".

    Las principales ventajas de utilizar <label> frente a texto simple son:

    • El código HTML está mejor estructurado y se mejora su accesibilidad.
    • Al pinchar sobre el texto del <label>, es como si hiciéramos click sobre el campo asociado al que sirve de título, recibiendo automáticamente el foco. Debemos tener en cuenta que se trata de un elemento en línea.

    A la hora de modificarlo con CSS, puede ser usado con las propiedades de textos y de elementos en línea.

    Etiqueta input¶

    Los elementos de formulario como botones y cuadros de texto también se denominan campos de formulario y controles de formulario. La mayoría de controles se crean con la etiqueta <input>, por lo que su definición formal y su lista de atributos es muy extensa.

    El atributo más importante es type, que indica el tipo de control de formulario del que se trata el elemento. Analicemos en los siguientes sub-apartados los distintos valores que puede tomar este atributo.

    Además, todo campo de formulario debe tener un nombre, representado por el atributo name. El valor de dicho atributo será enviado al destinatario del formulario, como identificador del valor introducido en el campo, de manera que el destinatario pueda referenciarlo para procesarlo. De hecho, si un campo no incluye el atributo name, sus datos no se envían al servidor.

    El atributo que representa el valor inicial del campo de formulario es value, que usaremos cuando queramos que aunque el usuario no haya introducido aún nada el campo muestre de inicio un valor.

    Otros atributos que podemos usar con todos los tipos de campos de formulario son:

    • alt: Texto descriptivo del campo (para el uso de personas con discapacidad).
    • form: Recibe el id del elemento <form> al que pertenece el campo. Este atributo permite situar los elementos <input> en cualquier parte del documento, incluso no siendo hijo de ningún <form> y luego indicar cuál a qué formulario pertenece.
    • tabindex: Permite indicar el orden de tabulación del elemento de formulario. Por defecto el orden de tabulación corresponde al orden en el que aparecen en el documento HTML.

    Debemos tener en cuenta que la etiqueta <input> es autonconclusa, es decir, que no lleva etiqueta de cierre, por lo que la escribiremos como <input />. Por otra parte, no debemos olvidar que la etiqueta <input> es un elemento en línea.

    Campos de texto¶

    HTML nos proporciona una serie de valores para el atributo type que nos permiten crear campos de formulario para introducir texto:

    • text: Para introducir un texto genérico, por ejemplo un nombre.
    • password: Para introducir una contraseña.
    • email: Para introducir un email.
    • tel: Para introducir una teléfono.
    • url: Para introducir una URL.
    • search: Para introducir una cadena de búsqueda.

    Dependiendo del tipo de campo de texto del que se trate el navegador realizará por defecto unas determinada comprobaciones de validación sobre el valor introducido. Por ejemplo, si se trata de un <input type="email"/>, el navegador antes de enviar el formulario comprobará que el valor introducido en dicho campo contiene al menos una @ y un .. Otra de las funcionalidad de los tipos, es que en dispositivos móviles adecúan el teclado virtual al valor posiblemente introducido. Por ejemplo, si el tipo es email, en el teclado aparecerá de forma más accesible los caracteres de email (@, .).

    Hay varios atributos de la etiqueta <input> que podemos usar exclusivamente con los campos de formulario textuales:

    • minlength: Número mínimo de caracteres que se pueden introducir en el campo.
    • maxlength: Número máximo de caracteres que se pueden introducir en el campo.
    • pattern: Expresión regular con la que se va a comparar el valor introducido para comprobar si es válido.
    • size: Indica la anchura del campo de formulario en número de caracteres. Si no se especifica, el navegador muestra el cuadro de texto con un tamaño predeterminado.
    • placeholder: Indica el texto que aparece en el campo de formulario como consejo (hint) para que el usuario sepa qué dato debe introducir en dicho campo, y que sólo está visible cuando el campo está vacío. Veamos ejemplos de campos de formulario textuales
    <input type="text" id="nombre" name="fullname"
    maxlength="50" size="30" placeholder="(nombre)" />
<input type="tel" id="telefono" name="phonenumber"
    maxlength="9" size="11" placeholder="(teléfono)" pattern="[0-9]{9}" />

    Campos de fecha y hora¶

    HTML nos proporciona una serie de valores para el atributo type que nos permiten crear campos de formulario para introducir datos relacionados con las fechas:

    • date: Para introducir una fecha. Formato YYYY-MM-DD, por ejemplo 2016-06-23.
    • time: Para introducir una hora. Formato HH:MM, por ejemplo 21:45.
    • month: Para introducir un mes de un año. Formato YYYY-MM, por ejemplo 2011-11.
    • week: Para introducir un número de semana de un año. Formato YYYY-Wnum_semana, por ejemplo 2014-W43.

    A este tipo de campos podemos aplicarles una serie de atributos específicos:

    • min: Valor mínimo que se puede introducir en el campo.
    • max: Valor máximo que se puede introducir en el campo. Veamos ejemplos de campos de formulario para fecha y hora:
    <input type="date" id="fecha" name="birthdate" value="2011-11-24" pattern="[0-9]{4}-[0-9]{2}-[0-9]{2}" title="YYYY-MM-DD" placeholder="YYYY-MM-DD" />
<input type="time" id="hora" name="time" value="21:52" min="20:00" max="22:00" pattern="[0-9]{2}:[0-9]{2}" title="HH:MM" placeholder="HH:MM" />
<input type="month" id="mes" name="month" value="2011-11" pattern="[0-9]{4}-[0-9]{2}" title="YYYY-MM" placeholder="YYYY-MM" />
<input type="week" id="semana" name="week" value="2011-W36" pattern="[0-9]{4}-W[0-9]{2}" title="YYYY-Wnumero_semana" placeholder="YYYY-Wnumero_semana" />

    Campos numéricos¶

    HTML nos proporciona una serie de valores para el atributo type que nos permiten crear campos de formulario para introducir valores numéricos:

    • number: Para introducir un valor numérico.
    • range: Para seleccionar un valor numérico dentro de un rango con un slider.

    Los siguientes atributos son característicos de este tipo de campos de formulario:

    • min: Valor mínimo que se puede introducir en el campo.
    • max: Valor máximo que se puede introducir en el campo.
    • step: Cantidad que se debe incrementar / decrementar el valor del campo cuando el usuario pulse los botones de incremento / decrementos mostrados dentro del campo de tipo number o mueva el slider del campo tipo range. Su valor por defecto es 1. Podemos asignarle el valor step="any" para indicar que el incremento/decremento puede ser cualquier valor, incluso aunque no sea entero.

    Si el usuario introduce manualmente un valor que no es válido en relación al atributo step, al pulsar sobre alguno de los botones de incremento o decremento se realizará un redondeo al siguiente / anterior valor válido.

    En los campos range si no se proporciona un valor inicial, se usará el valor correspondiente a (min+max)/2.

    Veamos ejemplos de campos de formulario numéricos:

    <input type="number" id="hermanos" name="siblings" min="1" max="50" value="1" />
<input type="range" id="salario" name="earnings" min="600" max="2000" step="200" value="1000">

    Campos checkbox¶

    El valor checkbox del atributo type nos permite crear un campos checkbox o casillas de verificación. Este tipo de controles permite al usuario seleccionar y deseleccionar opciones individualmente. Aunque en ocasiones se muestran varios checkbox juntos, cada uno de ellos es completamente independiente del resto. Por este motivo, se utilizan cuando el usuario puede activar y desactivar varias opciones relacionadas pero no excluyentes.

    Cuando un checkbox es seleccionado y se envía el formulario al destinatario el valor enviado para dicho campo corresponde al valor que hayamos establecido para su atributo value. Si no especifica valor para dicho atributo se enviará el valor on.

    Si se quiere mostrar un checkbox seleccionado por defecto, se utiliza el atributo checked.

    Veamos un ejemplo:

    <input type="checkbox" id="ejecutivo" name="executive" value="ejecutivo" checked="checked" /> Ejecutivo
<input type="checkbox" id="ingeniero" name="engineer" type="checkbox" value="ingeniero" /> Ingeniero
<input type="checkbox" id="operador" name="operator" type="checkbox" value="operador" /> Operador

    Campos radiobutton¶

    Los controles de tipo radiobutton son similares a los controles de tipo checkbox, pero presentan una diferencia muy importante: son mutuamente excluyentes. Los radiobutton se utilizan cuando el usuario solamente puede escoger una opción de entre varias que se le presentan. Cada vez que se selecciona una opción, automáticamente se deselecciona la opción que estuviera seleccionaba anteriormente.

    El valor del atributo type para estos controles de formulario es radio. El atributo name se emplea para indicar los radiobutton que están relacionados. Por lo tanto, cuando varios radiobutton tienen el mismo valor en su atributo name, el navegador sabe que están relacionados (agrupados), de manera que sólo permite tener seleccionado uno de ellos, así que cuando se selecciona un radiobutton con un determinado valor de name, el navegador deselecciona automáticamente el resto de radiobuttons que tengan el mismo valor en el atributo name.

    Cuando se envía el formulario al destinatario, para cada grupo de radiobuttons (aquellas que comparten el valor del atributo name) se envía un único valor, el correspondiente al atributo value del radiobutton seleccionado. Si no especifica valor para dicho atributo en el radiobutton seleccionado, se enviará el valor on, que NO nos permitirá distinguir realmente qué opción ha sido la seleccionado, por lo que deberíamos de especificar siempre el valor value para todos los radiobuttons.

    Si se quiere mostrar como seleccionado por defecto un determinado radiobutton (dentro de un grupo), estableceremos para dicho campo el atributo checked.

    Veamos un ejemplo:

    <input type="radio" id="hombre" name="sexo" value="hombre" />
<input type="radio" id="mujer" name="sexo" value="mujer" checked="checked" />

    Botones¶

    HTML nos proporciona una serie de valores para el atributo type que nos permiten crear botones en nuestro formulario con distintas funcionalidades:

    • submit: Crea un botón de envío, es decir, aquel que al pulsarlo hace que los datos introducidos en el formulario sean enviados al destinatario.
    • reset: Crea un botón que al pulsarlo borra todos los datos introducidos por el usuario dejando a los campos en su estado inicial.
    • button: Crea un botón genérico que no tiene una funcionalidad específica, sino que debe ser indicada mediante programación.
    • image: Crea un botón de envío (similar al valor submit), pero con la apariencia de una imagen. Para indicar la imagen a mostrar se usa el atributo src que recibirá la URL de la imagen.

    En este tipo de controles, excepto en el tipo image, el valor del atributo value es el texto que muestra el botón. Si no se establece el atributo value, el navegador muestra un texto predefinido. Veamos unos ejemplos

    <input type="submit" value="Enviar al destinatario" />
<input type="reset" value="Limpiar" />
<input type="image" src="imagenes/aceptar.png" />
<input type="button" value="Avisar" onclick="alert('Ha pulsado el botón');"/>

    Campos de selección de color¶

    El valor color del atributo type permite crear campos de formulario para que el usuario seleccione un color. Normalmente el navegador se encargará de mostrar un botón que al ser pulsado abra una ventana de selección de color.

    Si queremos especificar un valor inicial para el campo asignaremos al atributo value un valor que represente un color en formato RGB hexadecimal de 6 dígitos, por ejemplo #FF0000, es el único formato soportado. El valor por defecto es #000000.

    Veamos un ejemplo:

    <input type="color" id="color-letra" name="font-color" value="#FF0000" 
pattern="#[a-f0-9]{6}" title="Color RGB hexadecimal" placeholder="#000000"/>

    Campos de selección de archivos¶

    Los formularios también permiten adjuntar archivos para ser enviados al destinatario. El valor del atributo type para este control de formulario es file. El navegador se encarga de mostrar un cuadro de texto donde aparece el nombre del archivo seleccionado y un botón que permite navegar por los directorios y archivos del ordenador del usuario. Dicha visualización depende del navegador y del sistema operativo del cliente, y prácticamente NO se puede estilizar (aunque existen diferentes técnicas).

    Si se incluye un control para adjuntar archivos, es obligatorio añadir el atributo enctype en la etiqueta <form> del formulario con el valor multipart/form-data, así como establecer el método de envío method="POST". Por tanto al definir el formulario usaremos la siguiente estructura:

    <form action="..." method="POST" enctype="multipart/form-data">
  ...
</form>

    HTML nos proporciona una serie de atributos específicos para este tipo de campo de formulario:

    • multiple: Permite seleccionar más de un archivo para ser enviados al destinatario.
    • accept: Indica los tipos de fichero que es destinatario puede recibir. Corresponde una lista de extensiones o tipos MIME separados por ,. Por ejemplo: .jpg, .png, .doc, audio/*.
    • capture: Indica si se prefiere en vez de la selección del archivo la captura directa, por ejemplo la captura en ese momento de una foto, un video, u un audio. Veamos un ejemplo
    <form action="http://www.informaticasaladillo.es/mostrar.php" method="post" id="formulario" enctype="multipart/form-data">
  <input type="file" id="fotos" name="photos" multiple="multiple" accept="image/*" capture="capture" />
  ...
</form>

    Campos ocultos¶

    Los campos ocultos se emplean para añadir información oculta en el formulario. El valor del atributo type para este control de formulario es hidden. Los campos ocultos NO se muestran por pantalla, de forma que el usuario desconoce que el formulario los incluye. Normalmente los campos ocultos se utilizan para incluir información que necesita el destinatario pero que no es necesario o no es posible que la establezca el usuario, como por ejemplo el número de cliente del cliente cuyos datos quieren ser actualizados a través del formulario.

    Veamos un ejemplo:

    <input type="hidden" name="id_clilente" value="45678" />

    Etiqueta textarea¶

    La etiqueta <textarea> permite crear muchos tipos diferentes de campos de formulario. Sin embargo, existen otros tipos de campos que se crean con etiquetas diferentes. Un ejemplo de ello es la etiqueta <textarea>, que permite crear un campo de texto en el que introducir una gran cantidad de texto.

    A diferencia de <input>, la etiqueta <textarea> necesita NO es autoconclusiva, por lo que tendremos que usar la etiqueta de cierre </textarea>. El valor inicial del campo corresponderá al texto contenido entre la etiqueta de apertura y cierre.

    La etiqueta <textarea> crea por defecto un elemento en línea, al igual que <input>.

    Los atributos más utilizados en las etiquetas <textarea> son los que controlan su anchura y altura visual:

    • cols: Establece el número de caracteres que se podrán escribir como máximo en cada fila.
    • rows: Establece el número de filas de texto que serán visibles. No debemos olvidar que estos atributos sirven para indicar al navegador cómo debe dibujar el campo <textarea>, pero NO limitan el número de caracteres que se pueden introducir en él, algo que deberemos hacer con el atributo maxlength.

    Los siguientes atributos que explicamos cuando hablamos de la etiqueta <input> también son aplicables a la etiqueta <textarea>: minlength, maxlength y placeholder.

    Veamos un ejemplo:

    <textarea cols="80" rows="8" maxlength="200" placeholder="Indica cualquier observación digna de mención"></textarea>

    Etiquetas select, option y optgroup¶

    HTML nos proporciona la etiqueta <select> (selección) para crear campos de formulario correspondientes a listas de valores de entre los que el usuario puede seleccionar uno (o varios). Cada uno de los valores está representado por una etiqueta <option> (opción).

    La etiqueta <select> tiene una serie de atributos específicos que determinan su estilo visual y su funcionamiento:

    • size: Número de opciones que son visibles en la lista en un momento dado. Determina la altura del campo. Su valor por defecto es 0, en cuyo caso la lista como un cuadro combinado (combo box).
    • multiple: Indica si es posible o no seleccionar más de una opción dentro de la lista. Es decir hace que la lista sea multiselección.

    Por su parte, la etiqueta <option> tiene los siguientes atributos:

    • value: Corresponde al valor que se envía al destinatario cuando se selecciona dicha opción en el campo. Es un atributo obligatorio.
    • selected: Indica si dicha opción aparece seleccionada inicialmente en el campo. Si no se ha especificado el atributo multiple en la etiqueta <select> en la que se encuentra la opción, sólo debemos establecer el atributo selected como máximo a una opción. Cuando la lista se muestra como un cuadro combinado (combo box) es muy habitual que la primera opción sea simplemente informativa de que se debe seleccionar un elemento. A dicha opción deberemos asignarle value="" para que si se selecciona no se envíe ningún valor al destinatario y para que si el campo es obligatorio no sea válido mientras esté seleccionada dicha opción.

    Además HTML nos permite agrupar las opciones en diferentes grupos y mostrar un texto que describa el grupo. Para ello usaremos la etiqueta <optgroup>, que contendrá en su interior la distintas etiquetas <option> correspondientes a las opciones incluidas en dicho grupo. NO debemos anidar <optgroup>.

    Para indicar el texto descriptivo del grupo se usa el atributo label de la etiqueta <optgroup>. Este texto descriptivo permite a los usuarios detectar más rápidamente la opción a seleccionar dentro de la lista. No debemos olvidar que el atributo label simplemente es un texto descriptivo del grupo, por lo que NO es una opción seleccionable de la lista.

    Veamos unos ejemplos:

    <select name="system" id="sistema" required="required">
  <option value="">- seleccione -</option>
  <option value="windows">Windows</option>
  <option value="linux">Linux</option>
  <option value="mac">MacOS</option>
</select>

</select>

    <option value="vlc">VLC</option>
  </optgroup>

    Etiqueta button¶

    Como ya hemos visto, para crear botones HTML nos proporciona los valores submit, reset, button e image para el atributo type de la etiqueta <input>. Sin embargo, tenemos una opción adicional más flexible. Se trata de la etiqueta <button>, que permite crear un botón sobre el que se puede hacer click. A diferencia de la etiqueta <input>, la etiqueta <button> NO es autoconclusa, por lo que tendremos que usar la estructura: <button>Pulsa aquí</button> ​ Como vemos, el botón mostrará el contenido situado entre la etiqueta de apertura y la de cierre. Y este hecho es precisamente lo que hace que la etiqueta <button> sea más flexible que <input>, ya que podremos usar otras etiquetas dentro del contenido del botón, como por ejemplo <em>, <strong> o incluso <img>, y hacer uso de los pseudoelementos :after y :before, algo que no podemos hacer con <input>.

    Para indicar la funcionalidad del botón podemos usar el atributo type con los valores submit, reset y button (con el mismo significado que en <input>). Además también podemos usar los atributos name, value y otros más.

    Veamos unos ejemplos:

    <button type="submit">Enviar al <strong>destinatario</strong></button>
<button type="reset">Reinicializar</button>
<button type="submit"><img src="http://www.myiconfinder.com/uploads/iconsets/24-24-f2e15c4b9e21e0d16045c1c982786a2f-accept.png" /></button>
<button type="button" onclick="alert('Ha pulsado el botón');">Avisar</button>

    Etiqueta datalist¶

    La etiqueta <datalist> permite agrupar valores predefinidos que faciliten el autocompletado de un campo <input>, de manera que conforme el usuario va escribiendo en el campo el navegador muestre una lista desplegable con los valores predefinidos.

    Cada valor predefinido está representado por una etiqueta <option> cuyo atributo value corresponderá al valor que será asignado al <input> si se selecciona dicha opción. El contenido de la etiqueta <option> será el valor predefinido propuesto.

    Para indicar al <input> la lista de valores predefinidos que debe usar asignaremos su propiedad list el id de la etiqueta <datalist> correspondientes.

    Veamos un ejemplo:

    <input type="text" id="poblacion" name="city" maxlength="50" size="30" placeholder="(población)" list="poblaciones" /><br />
<datalist id="poblaciones">
  <option value="Algeciras">Algeciras</option>
  <option value="La Línea">La Línea de la Concepción</option>
  <option value="San Roque">San Roque</option>
  <option value="Los Barrios">Los Barrios</option>
  <option value="Tarifa">Tarifa</option>
</datalist>

    Campos de salida¶

    La etiqueta <output> representa el resultado de un cálculo o de acción de usuario. No es por tanto un campo para introducir ningún dato, sino un campo de salida que muestra un resultado o dato. Su valor es establecido mediante código de programación.

    Se emplea, por ejemplo, acompañando un campo de tipo range para mostrar la representación numérica del valor seleccionado por el usuario.

    Veamos un ejemplo:

    <label for="salario">Salario</label>
<input type="range" id="salario" name="earnings" min="600" max="2000" step="200" value="1000"
  onchange="document.querySelector('#salario-output').value = value;">
<output for="salario" id="salario-output">1000</output>

    Igualmente, la etiqueta <progress> permite mostrar un valor numérico correspondiente al nivel de progreso dentro de un rango. Se trata de una etiqueta inline-block. Usaremos los atributos value, min y max que ya conocemos. Si no especificamos el atributo value se mostrará un progreso indeterminado.

    Finalmente, la etiqueta meter permite mostrar un valor numérico correspondiente al nivel de grado dentro de un rango. Se trata de una etiqueta inline-block. Usaremos los atributos value, min y max que ya conocemos. Por defecto value vale 0, min vale 0 y max vale 1. También podemos definir las siguientes propiedades:

    • low: Valor límite superior de lo que considera un nivel bajo. Si el valor está entre el min y low la barra se muestra de color amarillo.
    • high: Valor límite inferior de de los que se considera un nivel alto. Si el valor está entre high y max la barra se muestra de color rojo. Si el valor está comprendido entre low y high la barra se muestra de color verde.

    Etiquetas fieldset y legend¶

    La etiqueta <fieldset> permite crear un grupo de campos de formulario. Se trata de un elemento en bloque. El navegador muestra por defecto un borde resaltado para cada agrupación. Por su parte, la etiqueta <legend> se incluye dentro de cada etiqueta <fieldset> y establece el título que muestra el navegador para cada agrupación de elementos. Se trata de un elemento en línea.

    Veamos un ejemplo:

    <fieldset>
  <legend>Sexo</legend>
  <label for="hombre">Hombre</label><br />
  <input type="radio" id="hombre" name="sexo" value="hombre" />
  <label for="mujer">Mujer</label><br />
  <input type="radio" id="mujer" name="sexo" value="mujer" checked="checked" />
</fieldset>

    Atributos relacionados con el estado de los campos de formulario¶

    Las etiquetas <input>, <textarea> y <select> nos proporcionan una serie de atributos relacionados con el estado del campo de formulario:

    • disabled: El control aparece deshabilitado y su valor no se envía al destinatario.
    • readonly: El contenido del control no se puede modificar. Sólo válido en campos de formulario de tipo texto, fecha o number.
    • required: El usuario debe introducir algún dato en el campo antes de poder enviar el formulario al destinatario. Indica, por tanto, que el campo es obligatorio. NO se puede usar en campos de botones, de imagen o hidden.
    • autofocus: Indica que dicho campo debe recibir el foco cuando cargue la página. Sólo un campo del formulario debe usar este atributo. NO se puede usar como campos hidden. Veamos un ejemplo:

    Pseudoclases CSS¶

    CSS define una serie de selectores de pseudoclases específicos para seleccionar los campos de formulario que se encuentren en un determinado estado:

    • :enabled, :disabled: Selecciona los elementos que estén habilitados / deshabilitados. Están relacionados con el atributo disabled.
    • :checked, :indeterminate: Selecciona los elementos que estén chequeados / en estado indeterminado (ni chequeado ni no chequeado). Están relacionados con el atributo checked de los checkbox y los radiobutton.
    • :valid, :invalid: Selecciona los elementos con dato válido / inválido. Están relacionados con el proceso de validación.
    • :required, :optional: Selecciona los elementos obligatorios de rellenar / opcionales. Están relacionados con el atributo required.
    • :in-range, :out-of-range: Selecciona los elementos cuyo dato está dentro / fuera del rango especificado. Están relacionados con los atributo min y max.
    • :read-only, :read-write: Selecciona los elementos de sólo lectura no editables / editables. Están relacionados con el atributo readonly.
    • :placeholder-shown: Selecciona los elementos en los que está siendo mostrado el placeholder, es decir, aquellos para los que no se ha introducido valor y se ha especificado un placeholder. Está relacionado con el atributo placeholder. No es soportado aún por algunos navegadores.
    • :default: Selecciona el elemento por defecto del formulario (normalmente el botón de envío).

    Otras pseudoclases no son específicas de los campos de formulario, pero son muy usadas con ellos:

    • :focus: Selecciona el elemento que tiene el foco.
    • :active: Selecciona el elemento sobre el que se está actuando (por ejemplo un botón).

    Estas pseudoclases se puede concatenarse en la regla CSS para realizar una selección muy precisa. Por ejemplo :focus:invalid significa "el campo que tenga el foco y cuyo valor no sea válido".

    Estilizado de formularios¶

    Como hemos podido comprobar en los ejemplos anteriores, los formularios no son precisamente fáciles de estilizar, por varios motivos:

    En primer lugar, las etiquetas disponibles para crear campos de formulario son en su mayoría etiquetas en línea, de manera que, por ejemplo una etiqueta <input> se sitúe sin complicación detrás de la etiqueta <label> que la describe. El problema aparece cuando queremos mostrar sólo un par label-input por "fila". En dicho caso, tendremos que añadir alguna etiqueta que actúe como padre del par y que haga las veces de "fila", para lo que podremos recurrir a la etiqueta <fieldset>, a una etiqueta <div> o incluso a una etiqueta <li> (dentro una etiqueta <ul> que agrupe todas la filas).

    En segundo lugar, estilizar un <label> basándonos en el estado del correspondiente <input> es complicado. El motivo es que dado que el <label> está situado antes del <input> en el documento HTML, no podemos usar ninguno de los selectores de hermanos, por ejemplo el de hermano adyacente + o hermano posterior ~, ya que ambos hacen referencia a hermanos posteriores. No hay ningún selector para un hermano anterior. La solución a este problema pasa por situar en el documento HTML el <label> después del correspondiente <input>, para que podamos seleccionarlo usando alguno de los selectores mencionados anteriormente, y usar el atributo order de las cajas flexibles para hacer que el <label> se muestre antes del <input> (aunque en el documento se encuentre después). Pero para que esto sea posible, el par label-input debe estar contenido en una etiqueta padre que actúe como fila y a la que se le establezca el modo de formato visual display: flex; que disponga los hijos en forma de fila con flex-direction: row;.
