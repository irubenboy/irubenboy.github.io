    1 Modelo de formato visual¶

    Introducción¶

    Uno de los aspectos más importantes y complejos de CSS es el conocido como modelo de formato visual, que determina qué tipo de caja genera cada elemento de nuestro documentos HTML. Como ya hemos comentado, CSS asigna a cada tipo de etiqueta (como <p>, <a>, etc.) un determinado modelo de formato visual. Por ejemplo, como ya comentamos, a las etiquetas <p> se les asigna por defecto el modelo de formato visual conocido como elemento en bloque o block, mientras que a las etiquetas <a> se les asigna por defecto el modelo de formato visual conocido como elemento en línea o inline.

    display¶

    El modelo de formato visual está representado en CSS por la propiedad display, que nos permite asignar a un elemento un modelo distinto al que tiene por defecto. El valor de la propiedad display influye o condiciona al elemento en varios aspectos, tanto exteriormente como interiormente:

    • Respecto a sí mismo: Establece si el elemento admite o no una serie de propiedades como tamaños, márgenes, etc., y define cómo será calculado el valor de dichas propiedades si no son admitidas.
    • Respecto a sus hermanos: Determina el comportamiento y la relación del elemento con sus elementos hermanos, tanto los que le preceden como los que le sucede. Por ejemplo, si debe forzar un salto de línea en su dibujado o no.
    • Respecto a sus hijos: Establece determinadas características y comportamientos en sus elementos hijos, y habilita poder declarar una serie de propiedades en los hijos para controlarlos satisfactoriamente.

    En CSS 2.1 la propiedad display está definida de la siguiente manera:

    Propiedad display
    Valores inline |block | list-item | run-in | inline-block | table | inline-table | table-row-group | table-header-group | table-footer-group | table-row | table-column-group | table-column | table-cell | table-caption | none | inherit
    Valor inicial inline
    Se aplica a Todos los elementos
    Definición Establece el tipo de caja generada por un elemento

    Analicemos los valores más habituales:

    • none: hace que el elemento no genere ninguna caja para sí mismo ni para sus descendientes. El resultado es que el elemento desaparece por completo de la página y no ocupa sitio, por lo que los elementos adyacentes ocupan su lugar.
    • block: Hace que el elemento sea de bloque.
    • inline: Hace que el elemento sea en línea.
    • inline-block: Hace que el elemento se comporte como un elemento en bloque en su interior, pero de cara al exterior se comporte como un elemento en línea.

    También hay una serie de valores relacionados con el formato de tabla:

    • table: El elemento se comporta como una tabla. Es el valor asignado por el navegador a los elementos <tr>.
    • table-header-group: El elemento se comporta como un grupo de filas de encabezado de tabla. Es el valor asignado por el navegador a los elementos <thead>.
    • table-footer-group: El elemento se comporta como un grupo de filas de pie de tabla. Es el valor asignado por el navegador a los elementos <tfoot>.
    • table-row-group: El elemento se comporta como un grupo de filas de tabla. Es el valor asignado por el navegador a los elementos <tbody>.
    • table-column-group: El elemento se comporta como un grupo de columnas de tabla. Es el valor asignado por el navegador a los elementos <colgroup>.
    • table-row: El elemento se comporta como una fila de una tabla. Es el valor asignado por el navegador a los elementos <tr>.
    • table-column: El elemento se comporta como una columna de tabla. Es el valor asignado por el navegador a los elementos <col>.
    • table-caption: El elemento se comporta como un título de tabla. Es el valor asignado por el navegador a los elementos <caption>.
    • table-cell: El elemento se comporta como una celda de tabla. Es el valor asignado por el navegador a los elementos <td> y <th>.

    CSS 3 ha añadido dos nuevos valores muy interesantes que estudiaremos posteriormente:

    • flex: El elemento permite organizar sus hijos de forma flexible.
    • grid: El elemento permite organizar sus hijos en forma de cuadrícula.

    Finalmente disponemos de una serie de valores con poco uso:

    • list-item: El elemento se muestra como si fuera un elemento <li>. Se pueden utilizar con ellos las propiedades de listas, como list-style-type, list-style-image, list-style-position y list-style.
    • run-in: La caja generada por el elemento no pertenece al flujo del padre, sino que pasa a ser una caja en línea dentro de su hermano sucesor. Este valor no es soportado por todos los navegadores.

    Elementos con display: block¶

    El navegador asigna por defecto a los elementos correspondientes a una serie de etiquetas la propiedad display: block. Como ya hemos visto, estos elementos reciben el nombre de elementos en bloque.

    Las siguientes etiquetas crean por defecto elementos en bloque: <address>, <article>, <aside>, <audio>, <blockquote>, <canvas>, <dd>, <div>, <dl>, <fieldset>, <figcaption>, <figure>, <footer>, <form>, <h1>, <h2>, <h3>, <h4>, <h5>, <h6>, <header>, <hgroup>, <hr>, <li>, <main>, <nav>, <noscript>, <ol>, <output>, <p>, <pre>, <section>, <table>, <tfoot>, <ul>, <video>.

    Cualquier otra etiqueta distinta a las anteriores puede producir un elemento en bloque si establecemos para él la propiedad display: block.

    Los elementos con display:block tienen una serie de características:

    • Ocupan todo el espacio de su elemento padre (contenedor), creando así un bloque.
    • De forma predeterminada, comienzan en una nueva línea.
    • Pueden contener en su interior elementos en línea y otros elementos en bloque.​

    Elementos con display: inline¶

    El navegador asigna por defecto a los elementos correspondientes a una serie de etiquetas la propiedad display: inline. Como ya hemos visto, estos elementos reciben el nombre de elementos en línea.

    Las siguiente etiquetas crean por defecto elemento en línea: b, big, i, small, tt, abbr, acronym, cite, code, dfn, em, kbd, strong, samp, time, var, a, bdo, br, img, map, object, q, script, span, sub, sup, button, input, label, select, textarea.

    Los elementos con display: inline tienen una serie de características:

    Ocupan por defecto sólo el espacio necesario para mostrar sus contenidos, por lo que no son afectados por las propiedades width y height.

    De forma predeterminada, NO es obligatorio que comiencen en una nueva línea, sino que se sitúan al lado del elemento anterior el la línea actual.

    Si le especificamos margen vertical mediante las propiedades margin-top, margin-bottom o margin, este NO tiene efecto sobre la posición del elemento. Sin embargo, el margen horizontal, especificado mediante margin-left, margin-right o margin SI afecta al posicionamiento del elemento dentro de la línea en la que se sitúe.

    Si le especificamos relleno vertical mediante las propiedades padding-top, padding-bottom o padding, éste NO altera la posición del elemento en la línea en la que se sitúe, aunque sí qu exista, como puede comprobarse si cambiamos el color de fondo del elemento. Sin embargo, el relleno horizontal, especificado mediante padding-left, padding-right o padding SI afecta al posicionamiento del elemento dentro de la línea en la que se sitúe.

    La etiqueta <img> es una excepción a este respecto, ya que aunque se trata de un elemento en línea SI le afecta el relleno y el margen vertical a la hora de situarlo verticalmente.

    Pueden contener únicamente los datos y otros elementos en línea, pero NO elementos en bloque.

    Elemento con display: inline-block¶

    Un elemento con display: inline-block es similar a un elemento en línea, en el sentido de que se situará en la posición que le corresponde en la línea continuando el flujo natural del texto (en la línea base), pero a diferencia de éstos, podremos especificar para ellos una determinada anchura y altura, ya que las propiedades width y height SI les afectan.

    Si necesitamos ajustar la posición vertical respecto a otros elementos inline o inline-block podemos usar la propiedad vertical-align.

    Elemento con display: table¶

    Un elemento con display: table hace que dicho elemento se comporte como si correspondiera a una etiqueta <table>. Es parecido a un elemento en bloque, pero por defecto ocupa sólo en ancho que se necesite para mostrar el contenido.

    vertical-align¶

    En los elementos con display: inline o display: inline-block podemos especificar cómo queremos que se alineen verticalmente dentro de la propia línea. Para controlar este aspecto CSS nos proporciona la propiedad vertical-align.

    Propiedad vertical-align
    Valores baseline | sub | super | top | text-top | middle | bottom | text-bottom | porcentaje | unidad de medida | inherit
    Se aplica a Elementos en línea y celdas de tabla
    Valor inicial baseline
    Descripción Determina la alineación vertical del elemento en línea respecto a "línea" en la que se encuentra

    Además esta propiedad también puede ser usada en las celdas de las tablas para controlar la alineación vertical de su contenido, como podemos comprobar en el apartado sobre tablas.

    Aunque pueda parecer que esta propiedad podrá resolver todos nuestros problemas sobre alineación vertical, la realidad es que su funcionalidad está limitada a elementos en linea o inline-block, por lo que no es tan útil como podría parecer. En realidad existen distintas técnicas para alinear verticalmente los elementos hijos dentro de un elemento padre, de entre las que destacamos el uso de display: flex, como veremos dentro de poco.
