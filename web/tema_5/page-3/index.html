    3 Posicionamiento¶

    Introducción¶

    Cuando los navegadores descargan el contenido HTML y CSS de las páginas web, aplican un procesamiento muy complejo antes de mostrar las páginas en la pantalla del usuario (renderizado). Para cumplir con el modelo de cajas, los navegadores crean una caja para representar a cada elemento de la página HTML.

    Al conjunto de algoritmos que definen cómo se muestra cada elemento en pantalla se le conoce como layout model, y dichos algoritmos tienen en cuenta distintos factores a la hora de dibujar la caja correspondiente a un elemento:

    • Todas las propiedades de la caja relacionadas con el modelo de caja: width (anchura), height (altura), box-sizing, padding (relleno), border (borde). El modo de visualización del elemento, es decir, si es un elemento en bloque, en línea, etc., representado por la propiedad display.
    • El modo de posicionamiento de la caja, representado por las propiedades position y float.
    • Las relaciones entre elementos atendiendo a dónde aparecen en el archivo HTML (quienes son su padre, sus hermanos y sus hijos). Al árbol de nodos especificados en el archivo HTML se le conoce como DOM (Document Object Model, modelo de objetos del documento).
    • Otro tipo de información, como por ejemplo el tamaño de las imágenes y el tamaño de la ventana del navegador. Los navegadores crean y posicionan de forma automática todas las cajas que forman cada página HTML, atendiendo al conocido como flujo normal, que consiste en que éstos aparezcan de arriba a abajo y de izquierda a derecha atendiendo al orden en que aparecen en el documento HTML. Además, los elementos que aparecen después en el DOM se muestran en capas superiores a los anteriores y los hijos en capas por encima de las de los padres.

    No obstante, CSS permite al diseñador modificar la posición en la que se muestra cada caja. Utilizando las propiedades que proporciona CSS para alterar la posición de las cajas es posible realizar efectos muy avanzados y diseñar estructuras de páginas que de otra forma no serían posibles.

    Posicionamiento¶

    El estándar de CSS define los siguientes modos diferentes para posicionar una caja:

    • Estático o normal (static): se trata del posicionamiento que utilizan los navegadores por defecto si no se indica lo contrario.
    • Relativo (relative): variante del posicionamiento normal que consiste en posicionar una caja según el posicionamiento normal y después desplazarla respecto de su posición original.
    • Absoluto (absolute): la posición de una caja se establece de forma absoluta respecto de su elemento contenedor y el resto de elementos de la página ignoran la nueva posición del elemento.
    • Fijo (fixed): variante del posicionamiento absoluto que convierte una caja en un elemento inamovible, de forma que su posición en la pantalla siempre es la misma independientemente del resto de elementos e independientemente de si el usuario hace scroll en la ventana del navegador.

    Reglas CSS sobre el posicionamiento¶

    La propiedad CSS más importante para determinar el posicionamiento de las cajas es position, que permite establecer el modo de posicionamiento para la misma, excepto de en el caso del posicionamiento flotante, que se realiza mediante la propiedad float, que veremos más adelante.

    Propiedad position
    Valores static | relative | absolute | fixed | sticky | inherit
    Se aplica a Todos los elementos
    Valor inicial static
    Descripción Selecciona el posicionamiento con el que se mostrará el elemento

    Normalmente, cuando se posiciona una caja con el modo relativo, absoluto, fijo o pegajoso, también es necesario desplazarla respecto de su posición original o respecto de otro origen de coordenadas. CSS define cuatro propiedades llamadas top, right, bottom y left para controlar el desplazamiento de las cajas posicionadas.

    Propiedad top, right. bottom, left
    Valores unidad_medida | porcentaje | auto | inherit
    Se aplica a Todos los elementos posicionados
    Valor inicial auto
    Descripción Indican el desplazamiento horizontal y vertical del elemento respecto de su posición original

    Si el desplazamiento se indica en forma de porcentaje, en las propiedades right y left se refiere al porcentaje sobre la anchura,del elemento, y en las propiedades top y bottom al porcentaje sobre la altura del elemento.

    La interpretación de sus valores depende del tipo de posicionamiento establecido para el elemento:

    • En los posicionamientos estático (normal) y flotante, estas propiedades son ignoradas.
    • En el posicionamiento relativo cada una de estas propiedades indica el desplazamiento del elemento desde la posición original de su borde superior/derecho/inferior/izquierdo.
    • En los posicionamientos absoluto y fijo estas propiedades indican el desplazamiento del elemento respecto del borde superior/derecho/inferior/izquierdo de su primer elemento antecesor no posicionado de forma estática (normal), o el viewport si todos sus antecesores usan posicionamiento estático.

    Posicionamiento estático o normal¶

    El posicionamiento normal o estático es el modelo que utilizan por defecto los navegadores para mostrar los elementos de las páginas. En este modelo, sólo se tiene en cuenta si el elemento es de bloque o en línea, las propiedades del modelo de caja, como width, height, box-sizing, etc. y su contenido.

    Los elementos de bloque forman lo que CSS denomina contextos de formato de bloque. En este tipo de contextos, las cajas se muestran una debajo de otra comenzando desde el principio del elemento contenedor. La distancia entre las cajas se controla mediante los márgenes verticales.

    Si un elemento se encuentra dentro de otro, el elemento padre se llama elemento contenedor y determina tanto la posición como el tamaño de todas sus cajas interiores.

    Si un elemento no se encuentra dentro de un elemento contenedor, entonces su elemento contenedor es el elemento <body> de la página. Normalmente, la anchura de los elementos de bloque está limitada a la anchura de su elemento contenedor, aunque en algunos casos sus contenidos pueden desbordar el espacio disponible.

    Por su parte, los elementos en línea forman los contextos de formato en línea. En este tipo de contextos, las cajas se muestran una detrás de otra de forma horizontal comenzando desde la posición más a la izquierda de su elemento contenedor. La distancia entre las cajas se controla mediante los márgenes laterales.

    Si las cajas en línea ocupan más espacio del disponible en su propia línea, el resto de cajas se muestran en las líneas inferiores.

    Posicionamiento relativo¶

    El posicionamiento relativo desplaza una caja respecto de su posición original establecida mediante el posicionamiento normal. El desplazamiento de la caja se controla con las propiedades top, right, bottom y left.

    El valor de la propiedad top se interpreta como el desplazamiento entre el borde superior de la caja en su posición final y el borde superior de la misma caja en su posición original. De la misma forma, el valor de las propiedades left, right y bottom indican respectivamente el desplazamiento entre el borde izquierdo/derecho/inferior de la caja en su posición final y el borde izquierdo/derecho/inferior de la caja original.

    Por tanto, la propiedad top se emplea para mover las cajas de forma descendente, la propiedad bottom mueve las cajas de forma ascendente, la propiedad left se utiliza para desplazar las cajas hacia la derecha y la propiedad right mueve las cajas hacia la izquierda. Este comportamiento parece poco intuitivo y es causa de errores cuando se empiezan a diseñar páginas con CSS. También se puede decir, que cuando usamos algunas de esas propiedad, es la distancia que hay desde dicha perspectiva al elemento, por ejemplo, si usamos top el elemento se posiciona donde la distancia sea la indica con respecto la perspectiva top. Si se utilizan valores negativos en las propiedades top, right, bottom y left, su efecto es justamente el inverso.

    Si tanto left como right tienen valores distintos de auto, que es el valor por defecto, uno de los dos valores se tiene que ignorar porque son mutuamente excluyentes. Para determinar la propiedad que se tiene en cuenta, se considera el valor de la propiedad direction, que vimos en un apartado anterior, de manera que si direction tiene el valor ltr (valor por defecto) prevalecerá el valor de la propiedad left, y si direction tiene el valor rtl, prevalecerá el valor de right.

    El desplazamiento relativo de una caja NO afecta al resto de cajas adyacentes, que se muestran en la misma posición que si la caja desplazada no se hubiera movido de su posición original. El principal problema de posicionar elementos de forma relativa es que se pueden producir solapamientos con otros elementos de la página.

    z-index¶

    Cuando posicionamos elementos con un tipo de posicionamiento distinto del normal se pueden producir solapamientos con otros elementos de la página. CSS nos proporciona la propiedad z-index para controlar la posición de un elemento sobre el eje Z (profundidad). Esta aplicación sólo afecta a elementos cuyo posicionamiento sea distinto al posicionamiento estático o normal.

    Propiedad z-index
    Valores auto | número | inherit
    Se aplica a Elementos que han sido posicionados explícitamente
    Valor inicial auto
    Descripción Establece el nivel tridimensional en el que se muestra el elemento

    El valor más común de la propiedad z-index es un número entero, que puede ser positivo o negativo. Cuanto más alto sea el valor numérico, más cerca del usuario se muestra la caja.

    Si no especificamos la propiedad z-index para ningún elemento, éstos se apilan en el orden en el que aparecen definidos en el documento. Los elementos con posicionamiento distinto al posicionamiento estático siempre aparecían por encima de los elementos con posicionamiento estático. Si asignamos un valor negativo a la propiedad z-index conseguiremos que aparezca detrás.

    Un hijo puede estar detrás de su padre sólo si el padre no tiene un z-index declarado.

    Otro aspecto importante es que esta propiedad respeta la jerarquía de elementos. Esto es, si un elemento B se sitúa por encima de un elemento A, un elemento hijo de A, nunca podrá situarse por encima del elemento B, aunque establezcamos un valor mayor a su propiedad z-index.     Posicionamiento absoluto¶

    El posicionamiento absoluto se emplea para establecer de forma exacta la posición en la que se muestra la caja de un elemento. Su posición inicial es igual a su posición según su posicionamiento estático, pero podemos indicar la nueva posición de la caja mediante las propiedades top, right, bottom y left.

    Cuando una caja se posiciona de forma absoluta, el resto de elementos de la página S**Í se ven afectados y modifican su posición**. Al igual que en el posicionamiento relativo, cuando se posiciona de forma absoluta una caja es probable que se produzcan solapamientos con otras cajas.

    En la imagen anterior, la caja 2 está posicionada de forma absoluta, lo que provoca que el resto de elementos de la página modifiquen su posición. En concreto, la caja 3 deja su lugar original y pasa a ocupar el hueco dejado por la caja 2.

    La interpretación de los valores de estas propiedades es mucho más compleja que en el posicionamiento relativo, ya que en este caso depende del elemento que se tome como referencia. Para identificar dicho elemento:

    • Se recorren todos los elementos contenedores (antepasados) empezando por el más cercano a la caja, subiendo por la jerarquía de elementos.
    • El primer elemento contenedor que esté posicionado de cualquier forma diferente a position: static se elige como elemento de referencia.
    • Si ningún elemento contenedor se ha posicionado con un modo diferente a position: static, se elige como elemento de referencia el viewport (no confundir con el elemento <body>). Una vez determinada el elemento de referencia del posicionamiento absoluto, los valores de las propiedades top, right, bottom y left se interpretan como el desplazamiento desde el borde superior/derecho/inferior/izquierdo de la caja hasta el borde superior/derecho/inferior/izquierdo del elemento de referencia.

    Por tanto, si se quiere posicionar un elemento de forma absoluta respecto de su elemento contenedor, es imprescindible posicionar este último. Lo más habitual, en este caso, es posicionar el elemento contenedor con position: relative y no asignarle ningún valor a top, right, bottom y left, para que el contenedor no se vea desplazado.

    De esta manera cuando posicionemos un elemento hijo de este contenedor con posicionamiento absoluto se tomará como elemento de referencia su elemento contenedor (su padre).

    De esta forma el posicionamiento absoluto permite hacer que un elemento se visualice fuera de los límites de su padre, en unas coordenadas relativas a él, siempre y cuando establezcamos un posicionamiento distinto a estático para el padre.

    Otro aspecto muy interesante de los elementos posicionados de manera absoluta es que si NO se especifica explícitamente su tamaño mediante width y height, al establecerla un posicionamiento absoluto, su tamaño pasará a ser el estrictamente necesario para cubrir el contenido, incluso aunque la etiqueta corresponda a un elemento de bloque.     Posicionamiento fijo¶

    El estándar CSS considera que el posicionamiento fijo es un caso particular del posicionamiento absoluto, ya que sólo se diferencian en el comportamiento de las cajas posicionadas.

    Su posición inicial es igual a su posición según su posicionamiento estático, pero podemos indicar la nueva posición de la caja mediante las propiedades top, right, bottom y left.

    Cuando una caja se posiciona de forma fija, la forma de obtener el origen de coordenadas para interpretar su desplazamiento es idéntica al posicionamiento absoluto. De hecho, si el usuario no mueve la página HTML en la ventana del navegador, no existe ninguna diferencia entre estos dos modelos de posicionamiento.

    La principal característica de una caja posicionada de forma fija es que su posición es inamovible dentro de la ventana del navegador. El posicionamiento fijo hace que las cajas no modifiquen su posición ni aunque el usuario haga scroll en la página dentro del navegador. Casi siempre el elemento de referencia es el viewport. El significado de los valores de las propiedades top, right, bottom, left y z-index es similar que en el caso del posicionamiento absoluto.

    Posicionamiento pegajosos¶

    Este tipo de posicionamiento, hace que la caja se comporte como si tuviera el posicionamiento relativo dentro de su padre, hasta que se llega a un determinado umbral de desplazamiento dentro del viewport, superado el cual se comporta como si su posicionamiento fuera fijo.
